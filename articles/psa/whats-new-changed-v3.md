# <a name="whats-new-or-changed-in-project-service-automation-version-3"></a>มีอะไรใหม่หรือมีอะไรเปลี่ยนแปลงใน Project Service Automation รุ่น 3

[!include [banner](../includes/psa-now-project-operations.md)]

[!INCLUDE[cc-applies-to-psa-app-1x-2x](../includes/cc-applies-to-psa-app-1x-2x.md)]

[!include [rename-banner](~/includes/cc-data-platform-banner.md)]

หัวข้อนี้แสดงข้อมูลเกี่ยวกับการเปลี่ยนแปลงส่วนติดต่อผู้ใช้ (UI) ฟังก์ชันการทำงาน และคำศัพท์เฉพาะ ใน Project Service Automation ระหว่างรุ่น 2 หรือรุ่น 1 และรุ่น 3

## <a name="project-scheduling"></a>การจัดกำหนดการโครงการ
กำหนดการโครงการ ซึ่งเรียกว่าโครงสร้างการแบ่งงาน (WBS) ในเวอร์ชันก่อนหน้า ได้ถูกเปลี่ยนชื่อเป็น กำหนดการ และเข้าถึงได้โดยการคลิกแท็บ **กำหนดการ** 

![กำหนดการโครงการ](media/psa-schedule-01.png)

กำหนดการตอนนี้มีการแสดงสำหรับการโต้ตอบใหม่ที่ทั้งทันสมัยและเข้าถึงได้ อย่างไรก็ตาม เอ็นจิ้นการกำหนดเวลา Project Service Automation พื้นฐานไม่มีการเปลี่ยนแปลง ปุ่มควบคุมใน Ribbon ของกริดกำหนดการช่วยให้คุณสามารถโต้ตอบกับกำหนดการด้วยวิธีที่คล้ายกับ Project Service Automation รุ่นก่อนหน้านี้ การเปลี่ยนแปลงเพิ่มเติมใน กำหนดการ รวมถึง:

- **แผนภูมิ Gantt** แผนภูมิ Gantt ถูกนำออกไปแล้ว การแสดงภาพ Gantt แบบใหม่จะกลับมา ในการปรับปรุงในอนาคต
- **ส่วนหัวของคอลัมน์** - คุณสามารถซ่อนส่วนหัวของคอลัมน์ในกริดได้ โดยคลิกที่ตัวบ่งชี้ที่อยู่ถัดจากชื่อคอลัมน์ 
- **คอลัมน์** - คุณสามารถแสดงคอลัมน์ที่ซ่อนอยู่ **โดยคลิก** เพิ่มคอลัมน์ 
- **ประเภทธุรกรรม** - การค้นหา **ประเภทธุรกรรม** ได้ถูกเพิ่มลงในตารางกำหนดการ และจะแสดงตามค่าเริ่มต้น 
 
## <a name="project-templates"></a>แม่แบบโครงการ
มีการการเปลี่ยนแปลงต่อไปนี้กับฟังก์ชันเท็มเพลตโครงการ

### <a name="create-a-project-template"></a>สร้างเท็มเพลตโครงการ 
คุณสามารถสร้างแม่แบบโครงการในเวอร์ชัน 3 ที่คล้ายกับ Project Service Automation รุ่นก่อนหน้า แม่แบบจะมีได้เฉพาะกำหนดการ และกำหนดการสามารถรวมการมอบหมายงานได้ แต่ไม่จำเป็นต้องทำ ถ้ากำหนดการมีการมอบหมายงาน จะสามารถใช้ได้สำหรับทรัพยากรทั่วไปเท่านั้น คุณสามารถสร้างความต้องการทรัพยากรสำหรับทรัพยากรทั่วไป แต่ไม่สามารถจองกับทรัพยากรจริงในแม่แบบได้ คุณไม่สามารถจองทรัพยากรจริงให้กับทีมในแม่แบบได้ 

### <a name="create-a-template-from-an-existing-template"></a>สร้างแม่แบบจากแม่แบบที่มีอยู่
เมื่อคุณสร้างแม่แบบโครงการใหม่จากแม่แบบที่มีอยู่ใน Project Service Automation รุ่น 3 จะเกิดสิ่งต่อไปนี้ขึ้น: 

- กำหนดการของโครงการต้นทางจะถูกคัดลอกลงในแม่แบบ 
- ทรัพยากรทั่วไปจะถูกคัดลอกลงในทีม และการมอบหมายทรัพยากรทั่วไปใด ๆ จะถูกคัดลอกไป ข้อกำหนดสำหรับทรัพยากรทั่วไปจะไม่ถูกคัดลอก 

### <a name="create-a-template-from-an-existing-project"></a>สร้างแม่แบบจากโครงการที่มีอยู่
เมื่อคุณสร้างแม่แบบโครงการใหม่จากโครงการ จะเกิดสิ่งต่อไปนี้ขึ้น: 

- กำหนดการของโครงการต้นทางจะถูกคัดลอกลงในแม่แบบ 
- ทรัพยากรทั่วไปจะถูกคัดลอกลงในทีม และการมอบหมายทรัพยากรทั่วไปใด ๆ จะถูกเก็บไว้ ข้อกำหนดสำหรับทรัพยากรทั่วไปจะไม่ถูกคัดลอก    
- ทรัพยากรที่มีชื่อ ทั้งที่กำหนดหรือไม่กำหนดจะถูกลบออกจากทีม และแทนที่ด้วยทรัพยากรทั่วไป
- หากมีข้อมูลลูกค้า จะถูกลบออก 
- หากมีการอ้างอิงถึงใบเสนอราคาและสัญญา จะถูกนำออก 

### <a name="create-a-project-from-a-template"></a>สร้างโครงการจากเท็มเพลต
ใน Project Service Automation รุ่น 3 เมื่อคุณสร้างแม่แบบโครงการใหม่จากแม่แบบที่มีอยู่ จะเกิดสิ่งต่อไปนี้ขึ้น:

- กำหนดการ ทีม และการมอบหมาย จะถูกคัดลอกไปยังโครงการใหม่   
- วันเริ่มต้นคือวันที่คัดลอกหรือวันที่ที่ผู้ใช้เลือก   
- สำหรับสมาชิกในทีมทั่วไปที่มีความต้องการทรัพยากรในแม่แบบ ความต้องการจะไม่ถูกคัดลอกหรือสร้างขึ้นโดยอัตโนมัติ คุณจะต้องสร้างขึ้นเอง 

## <a name="copy-a-project"></a>คัดลอกโครงการ
ใน Project Service Automation รุ่น 3 เมื่อคุณคัดลอกแม่แบบโครงการใหม่จากแม่แบบที่มีอยู่ จะเกิดสิ่งต่อไปนี้ขึ้น: 

- วันที่เริ่มต้นที่ถูกประมาณจะถูกคัดลอก แต่สามารถเปลี่ยนได้  
- กำหนดการและงานของโครงการจะถูกคัดลอก 
- ทรัพยากรทั่วไปและการมอบหมายจะถูกคัดลอก ความต้องการทรัพยากรสำหรับทรัพยากรทั่วไปจะไม่ถูกคัดลอก คุณจะต้องสร้างขึ้นเองใหม่ 
- ทรัพยากรจริงและการมอบหมายจะไม่ถูกคัดลอก แต่จะถูกแทนที่ด้วยทรัพยากรทั่วไป 
- ค่าจริงจะไม่ถูกคัดลอกไปยังโครงการใหม่ 

## <a name="move-a-scheduled-project"></a>ย้ายโครงการที่จัดกำหนดการไว้
เมื่อคุณย้ายกำหนดการของโครงการที่มีอยู่ไปข้างหน้า สิ่งต่อไปนี้จะเกิดขึ้น: 

- วันที่ของงานจะถูกย้ายโดยอัตโนมัติ เพื่อให้สอดคล้องกับรอบระยะเวลาการเคลื่อนไหว 
- ทรัพยากรทั่วไปที่กำหนดยังคงถูกกำหนดไว้   
- หากถูกสร้างก่อนที่จะย้ายโครงการ ความต้องการสำหรับทรัพยากรทั่วไปจะยังคงเหมือนเดิมและไม่ได้สร้างขึ้นใหม่โดยอัตโนมัติ คุณจะต้องสร้างอีกครั้ง เพื่อแสดงงานมอบหมายใหม่เนื่องจากการเคลื่อนไหวของงาน 
- การเปลี่ยนงานมอบหมายเกี่ยวกับแปลงทรัพยากรจริง เพื่อให้สอดคล้องกับการย้ายวันที่ของงาน การจองทรัพยากรจริงจะไม่เปลี่ยนแปลง คุณจะต้องปรับเปลี่ยนการจอง โดยใช้มุมมองการกระทบยอด 
- ทรัพยากรทีมที่มีการจอง แต่ไม่มีการมอบหมายจะไม่เปลี่ยนแปลง 
- ค่าจริงจะไม่ย้าย 

## <a name="estimates"></a>การประมาณการ
การประมาณการได้ถูกแบ่งออกเป็นสองแท็บ **การกำหนดทรัพยากร** และ **การประเมิน** แท็บ **การกำหนดทรัพยากร** ประกอบด้วยการประเมินความพยายาม และแสดงการมอบหมายทรัพยากรสำหรับงานในมุมมองตามระยะเวลาได้ คุณสามารถแก้ไขการประมาณการตามสิ่งที่กลไกจัดการการจัดกำหนดการได้สร้างขึ้น

![แท็บการมอบหมายทรัพยากรแสดงการประมาณการความพยายาม และการกำหนดทรัพยากรสำหรับงาน](media/resource-assignments-tab-02.png)

แท็บการ **การประมาณการ** แสดงต้นทุนและยอดขายสำหรับการกำหนดทรัพยากร ยอดเงินเป็นแบบอ่านอย่างเดียว ขณะนี้การคำนวณต้นทุนและการกำหนดราคาขายจะถูกขับเคลื่อนจากการมอบหมายสมาชิกทีม ในกำหนดการ หมายความว่า ถ้าคุณมีงานโดยไม่มีการมอบหมายงาน งานจะแสดงภายใต้กลุ่ม ไม่ได้กำหนด นอกจากนี้ยังหมายความว่าหากไม่มี **บทบาท** ซึ่งเป็นมิติการกำหนดราคาเริ่มต้น จะไม่มีการประมาณต้นทุนหรือยอดขาย ถ้าคุณมีลูกค้า หรือสัญญา/ใบเสนอราคา ที่เชื่อมโยงกับโครงการ 

![แท็บการประมาณการที่แสดงต้นทุนและยอดขาย](media/estimates-tab-03.png)
  
ประเภทยังได้รับการสนับสนุนในงาน ในมุมมองกำหนดการ การจัดกลุ่มตามประเภทในมุมมองตามระยะเวลาของการประมาณการจะให้ประสบการณ์ที่ดีขึ้น โดยเฉพาะอย่างยิ่งเมื่อคุณมีการประมาณการค่าใช้จ่ายในโครงการของคุณ การประมาณการค่าใช้จ่ายจะถูกป้อนโดยใช้กริดบนแท็บที่แยกต่างหาก 

สามารถป้อนการประเมินค่าใช้จ่ายในกริดบนแท็บ **การประเมินค่าใช้จ่าย** 

![แท็บการประเมินค่าใช้จ่ายที่แสดงกริดการประเมินค่าใช้จ่าย](media/expense-estimates-tab-04.png)

## <a name="resource-management"></a>การจัดการทรัพยากร
ใน Project Service Automation รุ่นที่ 3 ที่มีไคลเอนต์รวม UI ใหม่ และการเปลี่ยนแปลงในความสัมพันธ์ระหว่างการจองและการมอบหมาย การสรรหาพนักงานโครงการที่มีทรัพยากรทั่วไป หรือทรัพยากรจริงได้เปลี่ยนแปลงไปจากรุ่น 2 และรุ่น 1 อย่างมาก อย่างไรก็ตาม แนวคิดของทรัพยากรที่จองได้ทั้ง **ทรัพยากรจริง** และ **ทรัพยากรทั่วไป** ยังคงเหมือนเดิม เช่นเดียวกับ สมาชิกในทีม ข้อกำหนด งานมอบหมาย และการจอง   

![การใช้ตัวเลือกทรัพยากร](media/resource-management-05.png)

### <a name="assign-a-real-bookable-resource"></a>กำหนดทรัพยากรที่สามารถจองได้จริง 
ใน Project Service Automation รุ่น 3 การจองและการมอบหมายงานจะไม่เชื่อมโยงกันอย่างแน่นหนาเหมือนกับ Project Service Automation รุ่นก่อนหน้า คุณสามารถใช้ตารางทีมเพื่อจองสมาชิกในทีม **จริง** คล้ายกับในตลาด

ในการใช้ตัวเลือกทรัพยากรบนกำหนดการ คุณสามารถเลือกสมาชิกทีมที่ถูกสร้างขึ้นในมุมมองของทีม และจากนั้นจึงกำหนดงานให้ คุณสามารถมอบหมายงานให้กับพวกเขาต่อไปได้แม้จะผ่านเวลาจองมาแล้ว ใช้แท็บ **การกระทบยอด** เพื่อทำให้สมาชิกในทีมที่มีความแตกต่างในการจองและการมอบหมายสอดคล้องกันมากขึ้น

ตัวเลือกทรัพยากรจะแสดงสมาชิกในทีมสำหรับโครงการ คุณยังสามารถใช้ตัวเลือกทรัพยากรเพื่อค้นหาและดูทรัพยากรอื่นที่สามารถจองได้ที่ไม่ได้เป็นส่วนหนึ่งของทีมโครงการ คุณสามารถกำหนดพวกเขาลงในงาน และพวกเขาจะกลายเป็นส่วนหนึ่งของทีมโครงการ คุณจะต้องจองโดยใช้ **ตารางกำหนดการ** หรือแท็บ **การกระทบยอด**

### <a name="assign-a-generic-bookable-resource-on-a-task-and-project-team-and-then-fulfill-with-a-real-resource-via-schedule-board"></a>กำหนดทรัพยากรทั่วไปที่สามารถจองไดเในงานและทีมโครงการ และจากนั้นทำตามโดยใช้ทรัพยากรจริงผ่านตารางกำหนดการ 
ใน Project Service Automation รุ่น 3 จะไม่มีการใช้ฟังก์ชันสร้างทีมสำหรับทรัพยากรทั่วไป แต่ คุณสามารถสร้างและกำหนดทรัพยากรทั่วไปจากกำหนดการโดยตรง โดยการพิมพ์ชื่อตำแหน่งของทรัพยากรทั่วไปในเซลล์ทรัพยากรของกำหนดการ หรือ คุณสามารถเลือกไอคอน ทรัพยากร ในเซลล์โดยใช้ตัวเลือกทรัพยากร แล้วป้อนชื่อของทรัพยากรทั่วไปที่คุณต้องการสร้าง การดำเนินการนี้จะเปิดแผงการสร้างด่วนที่ช่วยให้คุณสามารถตั้งค่าบทบาทและหน่วยงานของสมาชิกในทีมทรัพยากรทั่วไป หลังจากที่คุณสร้างทรัพยากร จะมีการกำหนดให้กับงานและคุณสามารถกำหนดทรัพยากรทั่วไปนั้นให้กับงานอื่น ๆ ในกำหนดการได้    
 
เมื่อคุณได้กำหนดทรัพยากรให้กับงานที่เหมาะสมทั้งหมดแล้ว คุณสามารถสร้างความต้องการทรัพยากรและทำให้เสร็จโดยการจองโดยตรงกับ **ตารางกำหนดการ** หรือโดยการส่งคำขอทรัพยากร นอกจากนี้คุณยังสามารถเพิ่มทรัพยากรทั่วไปลงในตารางสมาชิกทีมได้โดยตรง 

ทรัพยากรทั่วไปจะถูกเพิ่มไปยังทีมโครงการที่ไม่มีความต้องการทรัพยากร และมีวันเริ่มต้น/สิ้นสุดของโครงการจนกว่าความต้องการทรัพยากรจะถูกสร้างขึ้น หากต้องการสร้างความต้องการ ให้เลือกทรัพยากรทั่วไปและคลิก **สร้าง** ขณะนี้ลิงก์ความต้องการจะแสดงขึ้น และชั่วโมงที่ต้องใช้จะถูกเติมด้วยชั่วโมงที่กำหนด คุณสามางานคลิกลิงก์เพื่อเปิด และอัพเดตความต้องการได้
  
เมื่อการจองเสร็จสมบูรณ์ และเสร็จสิ้นโดยทรัพยากรที่มีชื่อ ทรัพยากรทั่วไปจะถูกแทนที่ด้วยทรัพยากรที่มีชื่อ และการมอบหมายในกำหนดการจะถูกปรับปรุงด้วยทรัพยากรที่มีชื่อ 

ขณะนี้ทรัพยากรที่นำเสนอสำหรับความต้องการ จะถูกเก็บไว้บนแท็บแทนที่จะเป็นส่วนที่แยกต่างหาก

### <a name="multiple-named-resources-fulfilling-a-generic-resource"></a>เติมเต็มทรัพยากรทั่วไป ด้วยหลายทรัพยากรที่มีชื่อ
เมื่อความต้องการสำเร็จกับหลายทรัพยากร ทรัพยากรทั่วไปยังคงอยู่ในทีมและกำหนดให้กับงาน สมาชิกในทีมที่มีชื่อที่ถูกจอง จะไม่ได้ถูกกำหนดให้เป็นส่วนหนึ่งของตำแหน่ง ผู้จัดการโครงการสามารถกำหนดงานตามความจำเป็นกับทรัพยากรจริง  มุมมอง **การกระทบยอด** แสดงการแบ่งการจองในหลายทรัพยากรไปเป็นการมอบหมายงานหลาย ๆ การมอบหมาย สิ่งนี้ไม่ได้ทำโดยอัตโนมัติ เพราะในสถานการณ์ที่ซับซ้อนมากขึ้น เช่น เมื่อคุณมีกลุ่มของงานที่ทำให้เกิดความต้องการขึ้น เจตนาของวิธีการที่ผู้จัดการโครงการต้องการที่จะกำหนด จะต้องมีการสันนิษฐาน เนื่องจากระบบไม่สามารถเข้าใจเจตนา ก็มีโอกาสที่สมมติฐานจะแตกต่างจากที่ตั้งใจ และผลลัพธ์ที่เกิดขึ้นอาจไม่ถูกต้องหรือคาดเดาไม่ได้ ผลลัพธ์ที่คาดการณ์ได้คือเมื่อทรัพยากรทั่วไปยังคงถูกกำหนดไว้จนกว่าผู้จัดการโครงการจะมอบหมายงาน โดยใช้มุมมอง **การกระทบยอด**

### <a name="reconciliation"></a>การกระทบยอด
แท็บ **การกระทบยอด** แสดงการจองและงานมอบหมายทั้งหมดสำหรับสมาชิกทีมโครงการแต่ละคน จะแสดงชั่วโมงในเซลล์ที่สามารถแสดงจุดของเวลาตั้งแต่เดือนไปจนถึงวัน มุมมองนี้ช่วยให้ผู้จัดการโครงการกระทบยอดการจองของสมาชิกในทีม และงานมอบหมายสำหรับทีมโครงการของพวกเขา การทำเช่นนี้จะเป็นประโยชน์เนื่องจากการจองและงานที่ได้รับมอบหมายไม่สัมพันธ์กันมากเกินไป ซึ่งช่วยให้มีความยืดหยุ่นมากขึ้นเมื่อวางแผนโครงการ 

![แท็บ การกระทบยอด แสดงการจองและงานมอบหมายทั้งหมดสำหรับสมาชิกทีมโครงการแต่ละคน](media/resource-reconciliation-tab-06.png)

สำหรับแต่ละทรัพยากร มุมมองจะใช้ความแตกต่างระหว่างการจองของสมาชิกในทีมและค่าสะสมของการกำหนดงานของพวกเขา และแสดงความแตกต่างสองข้อต่อไปนี้ที่อาจเกิดขึ้นกับการจองและงานมอบหมายในโครงการ: 

- **การจองไม่เพียงพอ** - การจองไม่เพียงพอเกิดขึ้นเมื่อทรัพยากรมีการมอบหมายมากกว่าการจอง เนื่องจากกำลังการผลิตนี้ไม่ได้รับการสงวนสิทธิ์ ผู้จัดการโครงการสามารถแก้ไขได้ โดยการขยายการจองของทรัพยากร เพื่อครอบคลุมความขาดแคลน 
- **การจองเกินจำนวน** - การจองเกินจำนวนจะเกิดขึ้นเมื่อมีการจองทรัพยากรไปยังโครงการ แต่ยังไม่ได้กำหนดให้กับงาน  นี่อาจเป็นที่ยอมรับได้ ตัวอย่างเช่น ทรัพยากรมีการจองก่อนที่การมอบหมายงานจะเกิดขึ้น อย่างไรก็ตาม ในกรณีอื่นๆทรัพยากรอาจไม่สามารถวางแผนจะกำหนดและ PM ควรพิจารณายกเลิกการจองของทรัพยากรเพื่อให้สามารถใช้สำหรับโครงการอื่น 

เมื่อมีการมอบหมายงานสำหรับทรัพยากรแต่ไม่มีการจอง (การจองไม่เพียงพอ) คุณสามารถเลือกการจองไม่เพียงพอรวม แล้วเลือก **ขยายการจอง** จากที่นี่ คุณสามารถดูการจองที่จำเป็นเพื่อแก้ไขปัญหาการขาดแคลนของทรัพยากรและความพร้อมใช้งาน 
 
## <a name="time-and-expense"></a>เวลาและค่าใช้จ่าย
ส่วนนี้แสดงข้อมูลเกี่ยวกับการเปลี่ยนแปลงใน เวลา ค่าใช้จ่าย และการอนุมัติของ Project Service Automation ในรุ่น 3 ในฐานะที่เป็นส่วนหนึ่งของโซลูชัน Dynamics 365 Project Service Automation คุณลักษณะ **รายการเวลา** ได้ถูกรีเฟรชเพื่อใช้ประโยชน์จากกรอบงานส่วนติดต่อแบบรวม นี่ทำให้เกิดส่วนติดต่อผู้ใช้ที่เป็นรูปแบบเดียวกันซึ่งสอดคล้องกัน และเป็นไปตามการดีไซน์ที่ตอบสนองต่อการดู ที่จะเหมาะสมกับขนาดหน้าจอหรืออุปกรณ์ใดๆ 

### <a name="landing-page"></a>เพจเริ่มต้น
ประสบการณ์การป้อนข้อมูลเวลาแบบกำหนดเองที่ไม่มีการขยายได้ถูกเลิกใช้ในรุ่นที่ 3 แต่มีตอนนี้มีการขยายได้และเข้าถึงได้ ของประสบการณ์ตารางดังเดิม คุณสามารถเข้าถึงฟังก์ชันการทำงานของรายการเวลา ได้โดยใช้แผนผังเว็บไซต์ทางด้านซ้าย ในการเปลี่ยนแปลงนี้ คุณจะไม่สามารถป้อนเวลาครั้งละหนึ่งสัปดาห์ได้อีกต่อไป แต่คุณจะต้องสร้างรายการเวลาสำหรับแต่ละวันในกริดแทน หลังจากที่มีการสร้างรายการเพียงไม่กี่ครั้ง ผู้ใช้สามารถสร้างรายการเวลาที่มีฟังก์ชัน **คัดลอก** ที่จะอธิบายในภายหลังในหัวข้อนี้ 

![หน้าเริ่มต้นของรายการเวลา](media/time-entry-landing-page-07.png)
 
### <a name="create-new-time-entries"></a>สร้างรายการเวลาใหม่ 
คลิก **สร้าง** ใน ribbon เพื่อเปิดหน้าสร้างด่วน สำหรับรายการเวลาที่คุณป้อนช่วงเวลาเป็นนาที ชั่วโมง หรือวัน โดยเพียงแค่เริ่มพิมพ์ h m หรือ d พร้อมกับปริมาณ  

![สร้างรายการเวลาด่วน](media/quick-create-time-entry-08.png)

ฟิลด์การค้นหาได้รับการสนับสนุนโดยมุมมองของระบบ ตัวอย่างเช่น หลังจากที่คุณป้อนข้อมูลโครงการฟิลด์ **งานโครงการ** ถูกตั้งค่าเป็นค่าเริ่มต้นให้กับมุมมอง **งานโครงการของฉันที่เปิดอยู่** ถ้าต้องการสร้างรายการเวลาสำหรับงานที่ไม่ได้กำหนดให้กับผู้ใช้ ให้คลิก **เปลี่ยนมุมมอง** ในกล่องโต้ตอบการค้นหา แล้วเลือกมุมมอง **งานโครงการที่ใช้งานอยู่ทั้งหมด** หลังจากที่มีการสร้างและแสดงรายการเวลาในกริด คุณสามารถแก้ไขค่าบรรทัดใด ๆ ได้โดยตรงในกริด  

### <a name="bulk-createcopy"></a>สร้าง/คัดลอกจำนวนมาก 
อย่างไรก็ตาม หลังจากที่มีการสร้างรายการบางส่วนแล้ว คุณสามารถฟังก์ชันคัดลอกเพื่อสร้างรายการได้หลายรายการในครั้งเดียว คลิก **คัดลอก** เพื่อเปิดกล่องโต้ตอบการ **คัดลอก** ใน **รอบระยะเวลาเริ่มต้น: วันที่เริ่มต้น** ให้ตั้งค่าช่วงวันที่ที่ต้องการคัดลอกมา ใน **รอบระยะเวลาสิ้นสุด: วันที่เริ่มต้น** ให้ระบุวันที่ที่ต้องสร้างรายการเวลา คลิก **คัดลอก** เพื่อคัดลอกรายการเวลาไปยังวันที่สอดคล้องกัน ของสัปดาห์ที่ระบุใน **รอบระยะเวลาสิ้นสุด** ตัวอย่างเช่น รายการเวลาสำหรับวันจันทร์ของสัปดาห์ที่ผ่านมาจะถูกคัดลอกไปยังวันจันทร์ของสัปดาห์ที่ได้ระบุไว้ใน **รอบระยะเวลาที่สิ้นสุด** 

![คัดลอกรายการเวลาหลายรายการ](media/bulk-copy-time-entry-09.png)
 
### <a name="import-data"></a>นำเข้าข้อมูล 
งานมอบหมายและแลกเปลี่ยนดำเนินการตามรูปแบบ UI เดียวกัน ซึ่งช่วยให้ผู้ใช้สามารถระบุช่วงวันที่จากการจองที่ต้องนำเข้า คุณจะต้องเลือกการจองที่ควรคัดลอกลงในรายการเวลา **แบบร่าง** อย่างชัดเจน ในรุ่น 3 คุณจะไม่สามารถดูรูปแบบของรายการเวลา **ที่แนะนำ** บนกริดและปฏิทินได้อีกต่อไป  

### <a name="change-in-calendar-control"></a>การเปลี่ยนแปลงในการควบคุมปฏิทิน
ในรุ่น 3 เราได้เลิกใช้การควบคุมปฏิทินแบบกำหนดเอง และมาใช้ UC Calendar เพื่อแสดงรายการเวลาสำหรับสัปดาห์ ด้วยปฏิทินนี้คุณสามารถดูวัน สัปดาห์ และเดือนได้ 

> [!NOTE]
> ข้อจำกัดในปฏิทินคือ การควบคุมนี้ไม่สนับสนุนการดำเนินการในรายการปฏิทินแต่ละรายการ ตัวอย่างเช่น คุณจะไม่สามารถเลือกรายการปฏิทินหนึ่ง หรือหลายรายการ แล้วทำการส่งหรือลบรายการเหล่านั้น การคลิกที่รายการปฏิทินจะเปิดหน้า **เอนทิตีรายการเวลา** สำหรับการดำเนินการเพิ่มเติม 

### <a name="extensibility"></a>ความสามารถในการเพิ่ม
**รวบรวมข้อมูลบนฟิลด์แบบกำหนดเองในเอนทิตีรายการเวลาและค่าใช้จ่ายเท่านั้น** - รายการเวลาใช้กริดที่สามารถแก้ไขได้ ตารางแบบอ่านอย่างเดียว และตัวควบคุมปฏิทินจากแพลตฟอร์ม ตัวควบคุมเหล่านี้ทั้งหมดเป็นแบบดั้งเดิม และดังนั้นจะสนับสนุนการกำหนดเอง ใน Project Service Automation รุ่น 3 คุณสามารถเพิ่มฟิลด์แบบกำหนดเองเพิ่มเติม ตั้งค่าฟิลด์การค้นหา และสำรองข้อมูลด้วยมุมมองกำหนดเอง นอกจากนี้คุณยังสามารถตั้งค่าตรรกะทางธุรกิจแบบกำหนดเอง ตามค่าที่เลือกในฟิลด์ที่กำหนดเอง  

**รวบรวมข้อมูลในฟิลด์แบบกำหนดเองในรายการเวลาและค่าใช้จ่ายและเผยแพร่ผ่านเอนทิตีที่สนับสนุนโฟลว์การส่งและการอนุมัติ** - การประมวลผลของรายการเวลาโดยทั่วไปจะแสดงในไดอะแกรมต่อไปนี้

![กระบวนการของโฟลว์รายการเวลา](media/process-time-entries-10.png)

หากความต้องการทางธุรกิจระบุว่าเอนทิตีเวลาและค่าใช้จ่ายต้องบันทึกมิติการกำหนดราคาแบบกำหนดเองและเผยแพร่ค่าต่าง ๆ ที่ถูกตั้งค่า ตามเวลาและทรัพยากรรายการในมิติการกำหนดราคาแบบกำหนดเอง ผ่านเอนทิตีทั้งหมดในกราฟิกก่อนหน้านี้ ดูที่ [ตั้งค่าฟิลด์ที่กำหนดเองเป็นมิติการกำหนดราคา](set-up-pricing-dimensions.md)

เพื่อสนับสนุนความต้องการทางธุรกิจที่เอนทิตีเวลาและค่าใช้จ่ายต้องบันทึกมิติที่ไม่มีการกำหนดราคาที่กำหนดเองและเผยแพร่ค่า คุณสามารถใช้การตั้งค่ามิติการกำหนดราคา และแสดงมิติที่กำหนดเองเป็นมิติการกำหนดราคาโดยไม่มีอัตราต้นทุนหรือบิล อีกสถานการณ์หนึ่งคือ การเพิ่มเขตข้อมูลแบบกำหนดเองให้กับแต่ละเอนทิตี โดยใช้ชื่อเขตข้อมูลเดียวกันในเอนทิตีทั้งหมด สามารถสร้างปลั๊กอินแบบกำหนดเอง เพื่อเชื่อมโยงเรกคอร์ดในเอนทิตีที่มีส่วนร่วมในขั้นตอนการส่ง/อนุมัติ โดยใช้เอนทิตีต้นทางของธุรกรรมและการเชื่อมต่อธุรกรรม  

### <a name="delegate-time-and-expense-entry"></a>มอบหมายรายการเวลาและค่าใช้จ่าย
แพลตฟอร์ม Common Data Service ไม่สนับสนุนให้ผู้ใช้จำลองเป็นผู้ใช้อีกคนหนึ่ง ซึ่งหมายความว่าในรุ่นที่ 3 ของ Project Service Automation จะไม่มีการสนับสนุนสำหรับรายการเวลาและค่าใช้จ่ายที่ได้รับมอบสิทธิ์ อย่างไรก็ตาม คู่ค้าและลูกค้าได้ใช้ประโยชน์จากวิธีการแก้ปัญหา เพื่อให้การสนับสนุนสำหรับประสบการณ์การป้อนข้อมูลเวลาที่มอบหมายในรุ่น 3 นี่เป็นเพียงวิธีหลบปัญหาและไม่ใช่แก้ปัญหาที่สมบูรณ์ ดังนั้นจึงเป็นสิ่งสำคัญที่จะเข้าใจข้อจำกัดและใช้วิธีการนี้เท่านั้นถ้าข้อจำกัดที่เป็นที่ยอมรับได้ 

> [!IMPORTANT]
> ข้อมูลนี้ควรจะถือว่าเป็นคำแนะนำที่แนะนำสำหรับการดำเนินการที่กำหนดเองโดยคู่ค้า/ลูกค้า ทีมงานผลิตภัณฑ์จะไม่นำเสนอการสนับสนุนอย่างเป็นทางการสำหรับฟังก์ชันการทำงานนี้ ผ่านช่องทางการสนับสนุนของเรา

### <a name="customization-details"></a>รายละเอียดการการกำหนดเอง 
การกำหนดเองช่วยให้คุณสามารถเพิ่ม **ทรัพยากรที่สามารถจองได้** ไปยังการสร้างและแก้ไขประสบการณ์ ซึ่งจะอนุญาตให้ผู้ใช้ทำหน้าที่เป็นผู้รับมอบสิทธิ์โดยการเปลี่ยนฟิลด์ **ทรัพยากรการจอง** ให้กับผู้ใช้อื่นสำหรับรายการเวลาและค่าใช้จ่ายที่จำเป็นต้อง ถูกบันทึก ขั้นตอนต่อไปนี้จะครอบคลุมการมอบหมายรายการเวลา ข้อมูลเดียวกันนี้ใช้กับการมอบหมายรายการค่าใช้จ่าย 
 
1.  ตรวจสอบให้แน่ใจว่าผู้ใช้ที่มอบหมายมีการเข้าถึงความปลอดภัยทั้งหมดในโครงการและงานโครงการ 
1.  เนื่องจาก **ทรัพยากรที่จองได้** ซึ่งเป็นฟิลด์ใน **เอนทิตีรายการเวลา** จะไม่ถูกเปิดเผยบน **เพจการสร้างด่วน** คุณจำเป็นต้องเพิ่มเข้าไปเอง

    -หรือ-

    สร้างมุมมองที่กำหนดเองที่มีคอลัมน์ **ทรัพยากรที่จองได้** เพื่อดูเฉพาะรายการเวลาที่สร้างขึ้นสำหรับทรัพยากร เผยแพร่การกำหนดเองบนตัวออกแบบโมดูลของโปรแกรมประยุกต์สำหรับมุมมองนี้เพื่อแสดงภายใต้ **ตัวเลือกมุมมอง** ในหน้า **รายการเวลา** มีสองปลั๊กอินที่จัดการการตั้งค่าตัวจัดการสำหรับรายการเวลาที่ไม่ใช่โครงการ:

    - PreValidateTimeEntryCreate
    - PreValidateTimeEntryUpdate
 
1. สร้างปลั๊กอินใหม่เพื่อเขียนทับฟิลด์ **ตัวจัดการ** ไปยังตัวจัดการของผู้ใช้ที่กำหนดในฟิลด์ **ทรัพยากรที่สามารถจองได้** ใช้ **ขั้นตอนการดำเนินการ** เดียวกันกับปลั๊กอิน Out-of-band (OOB) (ตรวจสอบล่วงหน้า) และใช้ **ลำดับการดำเนินการ** ที่สูงกว่าปลั๊กอิน OOB (มากกว่า 1) จะทำให้แน่ใจว่าปลั๊กอินแบบกำหนดเอง จะถูกดำเนินการหลังจากปลั๊กอิน OOB  
 
### <a name="end-user-experience"></a>ประสบการณ์ผู้ใช้ปลายทาง
1.  เมื่อคุณสร้างรายการเวลาบนหน้าสร้างด่วน ให้ป้อนรายละเอียดงานโครงการและโครงการจากนั้นเลือกผู้ใช้บนฟิลด์ **ทรัพยากรที่สามารถจองได้** ซึ่งจะต้องมีการบันทึกรายการเวลาไว้ 
2.  ฟิลด์นี้จะเป็นค่าเริ่มต้นในการเข้าสู่ระบบผู้ใช้ อย่างไรก็ตามหากผู้ใช้แก้ไขฟิลด์นี้แล้วรายการเวลาจะถูกสร้างขึ้นสำหรับ **ทรัพยากรที่จองได้** ที่ถูกเลือก
3.  เมื่อคุณส่งรายการเวลาที่คุณสร้างสำหรับเรกคอร์ดเหล่านี้ รายการจะถูกจัดคิวขึ้นสำหรับผู้อนุมัติในโครงการตามที่คาดไว้ 
4.  เมื่อคุณเรียกคืนรายการเวลาที่สร้างขึ้นสำหรับผู้ใช้อื่น รายการเวลาจะถูกส่งกลับไปยังสถานะ **แบบร่าง** พร้อมกับฟิลด์ **ทรัพยากรที่สามารถจองได้** ที่ตั้งค่าเป็นผู้ใช้อื่นแล้ว 
5.  อีกทางหนึ่งคือ คุณสามารถสลับไปยังมุมมองที่กำหนดเองเพื่อกรองรายการเวลาที่สร้างสำหรับผู้ใช้อื่น 
 
### <a name="limitations"></a>ข้อจำกัด
ฟังก์ชัน **คัดลอก** และ **นำเข้า** จะทำงานเฉพาะในบริบทของผู้ใช้ที่เข้าสู่ระบบแล้วเท่านั้น ซึ่งหมายความว่า คุณไม่สามารถคัดลอกหรือนำเข้ารายการเวลาที่สร้างขึ้นสำหรับผู้ใช้ที่เข้าสู่ระบบในฐานะทรัพยากรที่สามารถจองได้

รายการเวลาที่ไม่ได้เป็นของโครงการ จะถูกส่งไปเพื่อการอนุมัติไปยังผู้จัดการทรัพยากรที่สามารถจองได้ เฉพาะถ้าขั้นตอนที่ 4 ในส่วน **รายละเอียดการกำหนดเอง** ด้านบนเสร็จสมบูรณ์แล้ว ไม่เช่นนั้น รายการเวลาที่ไม่ใช่โครงการสำหรับผู้ใช้อื่น จะถูกกำหนดเส้นทางอย่างไม่ถูกต้องไปยังผู้จัดการของผู้ใช้ที่เข้าสู่ระบบ 

### <a name="other-changes"></a>การเปลี่ยนแปลงอื่น ๆ 
ฟังก์ชัน **การจองและงาน** ได้ถูกเอาออกแล้ว 

## <a name="multidimensional-pricing"></a>การกำหนดราคาหลายมิติ
เพื่อเพิ่มความยืดหยุ่นและตอบสนองข้อกำหนดทางธุรกิจที่แตกต่างกัน Project Service Automation รุ่น 3 จังสนับสนุนแอพลิเคชันของชุดมิติการกำหนดราคาเป็นอัตราต้นทุนและอัตราการเรียกเก็บเงิน ค่ามิติสามารถตั้งค่าเป็นค่าเริ่มต้น แล้วเผยแพร่ข้ามกระบวนการคำนวณต้นทุนและการกำหนดราคา จากการทำโปรไฟล์ทรัพยากรจนถึงการป้อนข้อมูลเวลาเพื่อประมาณค่าที่จะเกิดขึ้นจริง การกำหนดค่าและการปรับเปลี่ยนหรือการขยายเฉพาะลูกค้าเพิ่มมาตรฐานความสามารถในการปรับแต่งโครงสร้างพื้นฐาน

Project Service Automation มาพร้อมกับชุดค่าเริ่มต้นของมิติการกำหนดราคา และบทบาทและหน่วยทรัพยากร และสามารถตั้งค่าราคาและต้นทุนสำหรับแต่ละบทบาทและแต่ละชุดข้อมูลหน่วยขององค์กร

สำหรับลูกค้าของ Project Service Automation ที่ต้องการใช้ฟิลด์สำเร็จรูปเหล่านี้เป็นมิติการกำหนดราคาในรุ่น 3 จะไม่มีการเปลี่ยนแปลงใดๆ ที่สังเกตได้ คุณสามารถใช้ Project Service Automation ต่อได้ตามปกติ แต่อย่างไรก็ตาม หากคุณต้องการราคาหรือต้นทุนสำหรับทรัพยากรของคุณโดยใช้แอตทริบิวต์เพิ่มเติมอื่นๆ รุ่น 3 ช่วยให้สามารถเพิ่มมิติการกำหนดราคาของคุณเองไปยัง Project Service Automation ส่วนขยายของมิติราคาที่กำหนดเองเป็นประสบการณ์การกำหนดค่าที่ซับซ้อน 

## <a name="quotes-and-contracts"></a>ใบเสนอราคาและสัญญา
ในรุ่นที่ 3 ของ Project Service Automation ลักษณะของการตั้งค่าและการจัดการสำหรับใบเสนอราคาและสัญญามีการเปลี่ยนแปลง ส่วนต่อไปนี้ให้ข้อมูลรายละเอียดเพิ่มเติม

### <a name="set-up-chargeability-options"></a>ตั้งค่าตัวเลือกความสามารถในการคิดค่าธรรมเนียม
ในรุ่น 1 และ 2 การตั้งค่าความสามารถในการคิดค่าธรรมเนียมสำหรับบทบาทและประเภทสำหรับใบเสนอราคาและสัญญาเฉพาะที่ทำโดยใช้มุมมอง **ความสามารถในการคิดค่าธรรมเนียม** ที่อยู่ในการนำทางด้านบนของบรรทัดใบเสนอราคาหรือบรรทัดสัญญา นอกจากนี้ยังเป็นที่ที่คุณสามารถตั้งค่าราคาสำหรับบทบาทและประเภทค่าใช้จ่ายเหล่านั้น

ในรุ่น 3 การตั้งค่าตัวเลือกความสามารถในการคิดค่าธรรมเนียม ตามบทบาทและประเภทค่าใช้จ่ายจะดำเนินการที่ระดับใบเสนอราคาหรือสัญญา การตั้งค่าการกำหนดราคาจะแยกจากการตั้งค่าความสามารถในการคิดค่าธรรมเนียม คุณจะพบ **บทบาทที่คิดค่าธรรมเนียมได้** และ **ประเภทที่คิดค่าธรรมเนียมได้** เป็นแท็บบนหน้า **บรรทัดใบเสนอราคา** และ **บรรทัดสัญญา** โดยไม่ต้องใช้การนำทางด้านบน

![บทบาทที่คิดค่าธรรมเนียมได้](media/chargeable-12.png)
 
การตั้งค่าของบทบาทที่คิดค่าธรรมเนียมได้ และประเภทที่คิดค่าธรรมเนียมได้ ยังใช้ประโยชน์จากการควบคุมกริดที่สามารถแก้ไขได้ที่มีมาตั้งแต่ต้น สำหรับแต่ละบทบาทและประเภท ตัวเลือกที่ได้รับการสนับสนุนสำหรับชนิดการเรียกเก็บเงิน ระหว่างขั้นตอนการเสนอราคาและทำสัญญายังคงไม่เปลี่ยนแปลงจากรุ่นก่อนหน้านี้ โดยเป็น **คิดค่าธรรมเนียมได้** และ **คิดค่าธรรมเนียมไม่ได้** **ส่วนเสริม** ไม่ได้รับการสนับสนุนในระหว่างขั้นตอนการการเสนอราคาหรือการทำสัญญา รองรับ **ส่วนเสริม** ในระหว่างการอนุมัติเวลาหรือค่าใช้จ่ายเท่านั้น  
 
### <a name="create-and-edit-custom-pricing-for-a-project-service-automation-quote-and-project-contract"></a>สร้างและแก้ไขการกำหนดราคาที่กำหนดเองสำหรับใบเสนอราคา Project Service Automation และสัญญาโครงการ
ในรุ่น 1 และ 2 การใช้รายการราคาที่กำหนดเองสำหรับใบเสนอราคาและสัญญาใช้ **แก้ไขราคา** บนมุมมอง **ความสามารถในการคิดค่าธรรมเนียม** มุมมอง **ความสามารถในการคิดค่าธรรมเนียม** อยู่ในการนำทางด้านบนของบรรทัดใบเสนอราคาหรือบรรทัดสัญญา นอกจากนี้ยังเป็นที่ที่คุณสามารถตั้งค่าตัวเลือกการคิดค่าธรรมเนียมสำหรับบทบาทและประเภทค่าใช้จ่ายเหล่านั้น

ในรุ่น 3 การสร้างและการใช้รายการราคาโครงการที่กำหนดเองบนใบเสนอราคา Project Service Automation และสัญญาโครงการ Project Service Automation ถูกแยกออกจากการตั้งค่าการคิดค่าธรรมเนียม ใบเสนอราคา Project Service Automation และสัญญาโครงการ Project Service Automation มีแท็บใหม่ที่เรียกว่า **รายการราคาของโครงการ** แท็บนี้แสดงมุมมองร่วมของราคาโครงการทั้งหมดที่แนบมากับใบเสนอราคา Project Service Automation หรือสัญญาโครงการ ถ้าต้องการสร้างรายการราคาที่กำหนดเองจากรายการราคาที่มีอยู่ ซึ่งเชื่อมโยงกับใบเสนอราคาหรือสัญญาโครงการแล้วให้คลิก **สร้างการกำหนดราคาแบบกำหนดเอง** การดำเนินการนี้จะทำสำเนาของรายการราคาที่เกี่ยวข้องทั้งหมดและแนบไปกับใบเสนอราคาหรือสัญญา ขณะนี้คุณสามารถเปิดรายการราคาและแก้ไขบทบาทหรือราคาตามประเภทค่าใช้จ่าย เพื่อให้การเปลี่ยนแปลงราคาเหล่านั้นจะใช้กับใบเสนอราคาหรือสัญญานี้เท่านั้น

> [!NOTE]
> ความหน่วงสั้น ๆ อาจเกิดขึ้นระหว่างเมื่อคุณคลิก **สร้างการกำหนดราคาแบบกำหนดเอง** กับเมื่อสร้างรายการราคาแบบกำหนดเอง เราขอแนะนำให้รีเฟรชตารางแทนการคลิกหลายครั้ง รายการราคาที่กำหนดเองได้ถูกสร้างขึ้นถ้าชื่อรายการราคาที่เกี่ยวข้องมีชื่อใบเสนอราคาหรือชื่อสัญญาโครงการที่ต่อท้าย
