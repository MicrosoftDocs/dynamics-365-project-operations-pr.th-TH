# <a name="organizational-units"></a>หน่วยองค์กร 

[!include [banner](../includes/psa-now-project-operations.md)]

[!INCLUDE[cc-applies-to-psa-app-3.x](../includes/cc-applies-to-psa-app-3x.md)]

Dynamics 365 Project Service Automation หน่วยองค์กร คือ กลุ่มหรือฝ่ายที่แตกต่างกันในบริษัทที่ให้บริการระดับมืออาชีพ ที่ใช้ทรัพยากรที่เรียกเก็บเงินได้ซึ่งมีอัตราต้นทุน

สำหรับบริษัทที่ให้บริการระดับมืออาชีพที่ใช้ทรัพยากรทางเทคนิคในพื้นที่ปฏิบัติการต่างๆ หรือสายธุรกิจ ต้นทุนในการเติมเต็มบทบาทในพื้นที่ปฏิบัติการหรือสายธุรกิจ อาจแตกต่างจากค่าใช้จ่ายในการเติมเต็มบทบาทในพื้นที่ปฏิบัติการหรือสายธุรกิจอื่น หน่วยขององค์กรแนวคิดช่วยในสถานการณ์นี้โดยการจัดเตรียมวิธีการจัดกลุ่มชุดของบทบาทที่เรียกเก็บเงินได้ ในฝ่ายของบริษัทที่มีโครงสร้างต้นทุนที่แตกต่างกันสำหรับบทบาทเหล่านี้

## <a name="key-attributes-and-associations-of-organizational-units"></a>แอตทริบิวต์ของคีย์และความสัมพันธ์ของหน่วยองค์กร

ใน PSA หน่วยองค์กรใน PSA มีสกุลเงินเฉพาะและรายการราคาต้นทุนเฉพาะ

สกุลเงินของหน่วยองค์กร คือ สกุลเงินหลักที่ใช้ในการติดตามต้นทุน

คุณสามารถแนบรายการราคาต้นทุนหนึ่งรายการหรือมากกว่า ไปกับแต่ละหน่วยองค์กร PSA จะให้ข้อจำกัดต่อไปนี้ในรายการราคาที่สามารถแนบกับหน่วยองค์กร:

- รายการราคาต้องอยู่ในสกุลเงินของหน่วยองค์กร
- รายการราคาต้องเป็นรายการราคาต้นทุน

นอกจากนี้ ยังมีแอตทริบิวต์สำหรับหน่วยขององค์กรบนเอนทิตีทรัพยากร ทรัพยาการแต่ละรายสามารถมอบหมายให้กับหนึ่งหน่วยองค์กร

## <a name="roles-of-organizational-units"></a>บทบาทของหน่วยองค์กร

หน่วยองค์กรมีสองบทบาทใน PSA:

- **หน่วยตามสัญญา** – หน่วยองค์กรที่แสดงถึงกลุ่มหรือฝ่ายของบริษัท ที่มีหน้าที่รับผิดชอบหลักในการขายที่ชนะ และการจัดการการจัดส่งของงานและการบริการให้กับลูกค้า หน่วยตามสัญญาจะถูกระบุโดยฟิลด์ **หน่วยตามสัญญา** ในส่วนหัวของเพจ **โอกาสทางการขาย** **ใบเสนอราคา** **สัญญาโครงการ** และ **โครงการ**
- **หน่วยการจัดเตรียมทรัพยากร** – หน่วยองค์กรที่ทรัพยากรขึ้นอยู่หรือถูกกำหนดให้ หน่วยองค์กรนี้สามารถมอบบทบาทบางอย่างให้กับทรัพยากร เกี่ยวกับรายการของงาน (SOWs) และโครงการที่เป็นของหน่วยตามสัญญา

## <a name="organizational-unit-faqs"></a>คำถามที่พบบ่อยเกี่ยวกับหน่วยองค์กร

ต่อไปนี้คือคำตอบบางส่วนสำหรับคำถามที่ถามบ่อยที่สุดเกี่ยวกับหน่วยองค์กร:

### <a name="how-is-the-organizational-unit-entity-in-psa-related-to-the-organization-entity-that-already-exists-in-dynamics-365"></a>เอนทิตีหน่วยองค์กรใน PSA เกี่ยวข้องกับเอนทิตีองค์กรที่มีอยู่แล้วใน Dynamics 365 อย่างไร

เอนทิตีองค์กรใน Microsoft Dynamics 365 แสดงชื่อของอินสแตนซ์ Dynamics 365 ส่วนกลาง ชื่อนี้มักจะเป็นชื่อขององค์กรส่วนกลาง

เอนทิตีหน่วยองค์กรแสดงถึงกลุ่มหรือฝ่ายในองค์กรส่วนกลาง กลุ่มหรือฝ่ายนี้มีชุดของบทบาทและรายการราคาต้นทุนสำหรับบทบาทเหล่านั้น และบทบาทและรายการราคาเหล่านั้นแตกต่างจากบทบาทและรายการราคาของกลุ่มหรือฝ่ายอื่นๆในองค์กร

เมื่อ PSA ถูกติดตั้ง จะมีการสร้างหน่วยองค์กรเริ่มต้นตามองค์กร ทรัพยากรที่มีอยู่ทั้งหมดจะได้รับมอบหมายให้กับหน่วยองค์กรเริ่มต้น หากผู้ใช้ไดเรกทอรีที่ใช้งานอยู่หรือทรัพยากรใหม่ถูกนำเข้ามาใน Dynamics 365 กระบวนการนำเข้าของผู้ใช้จะมอบหมายพวกเขาให้กับหน่วยองค์กรเริ่มต้นใน PSA
 
### <a name="how-does-the-organizational-unit-entity-differ-from-the-business-unit-entity"></a>เอนทิตีหน่วยองค์กรแตกต่างจากเอนทิตีหน่วยธุรกิจอย่างไร

ใน Dynamics 365 เอนทิตีหน่วยธุรกิจเป็นโครงสร้างความปลอดภัย การเชื่อมโยงของผู้ใช้ที่มีหน่วยธุรกิจจะกำหนดเอนทิตีและเรกคอร์ดของเอนทิตี ที่ผู้ใช้มีสิทธิ์เข้าถึง นอกจากนี้ยังกำหนดสิทธิ์ (สร้าง อ่าน เขียน ลบ ผนวก ผนวกไปยัง มอบหมาย หรือใช้ร่วมกัน) ที่ผู้ใช้มีสำหรับเรกคอร์ดเอนทิตีเหล่านั้น

เอนทิตีหน่วยองค์กรแสดงถึงกลุ่มหรือฝ่ายของบริษัทที่มีอัตราต้นทุนที่แตกต่างกัน สำหรับพนักงานที่ถูกมอบหมายให้ การเชื่อมโยงของทรัพยากรกับหน่วยองค์กรจะกำหนดต้นทุนของทรัพยากรให้กับการมีส่วนร่วมของโครงการ

เมื่อคุณใช้ Dynamics 365 ปรับปรุงการอนุญาตด้านความปลอดภัยสำหรับลำดับชั้นของหน่วยธุรกิจ และการมอบหมายของผู้ใช้ให้กับหน่วยธุรกิจ มอบหมายผู้ใช้ทั้งหมดที่มักจะต้องเข้าถึงชุดเรกคอร์ดเดียวกันกับหน่วยธุรกิจเดียวกัน หน่วยองค์กรไม่มีผลต่อการอนุญาตด้านความปลอดภัยหรือการเข้าถึง

#### <a name="example-of-organizational-units-and-business-units"></a>ตัวอย่างของหน่วยองค์กรและหน่วยธุรกิจ

Contoso Ltd. มีการฝึกปฏิบัติทางเทคโนโลยี Microsoft ที่กำลังเติบโต กสิณ และ ชมพูนุช ทั้งคู่เป็น C\# นักพัฒนา แต่ ชมพูนุช อยู่ในสหรัฐอเมริกา ในขณะที่ กสิณ ที่อยู่ในอินเดีย การมีส่วนร่วมของโครงการส่วนใหญ่ต้องใช้ทรัพยากรจาก Contoso อินเดีย และ Contoso สหรัฐอเมริกา กสิณ และ ชมพูนุช ต้องมีการเข้าถึงการรักษาความปลอดภัยในระดับเดียวกันกับโครงการ ในพื้นที่การฝึกปฏิบัตินี้ อย่างไรก็ตาม ต้นทุนของนักพัฒนาจาก Contoso อินเดีย แตกต่างจากต้นทุนของนักพัฒนาจาก Contoso สหรัฐอเมริกา

นี่คือวิธีที่ดีที่สุดในการออกแบบสำหรับสถานการณ์นี้ โดยใช้ Dynamics 365 และ PSA

1. สร้างการฝึกปฏิบัติทางเทคโนโลยีของ Microsoft ในฐานะหน่วยธุรกิจ และเชื่อมโยงกับ กสิณ และ ชมพูนุช ด้วยวิธีนี้ คุณจะช่วยรับประกันว่าพนักงานทั้งสองมีระดับการเข้าถึงการรักษาความปลอดภัยในระดับเดียวกันกับโครงการใดๆในพื้นที่การฝึกปฏิบัตินั้น พวกเขาทั้งสองจะสามารถตรวจสอบความคืบหน้าและรายงานเวลา ค่าใช้จ่าย และการอัปเดตงาน 
2. สร้างหน่วยองค์กรสองหน่วย เพื่อช่วยรับประกันว่าต้นทุนในโครงการจะสะท้อนอย่างถูกต้อง 
3. เชื่อมโยง ชมพูนุช กับ Contoso สหรัฐอเมริกา และเชื่อมโยง กสิณ กับ Contoso อินเดีย
4. มอบหมายรายการราคาต้นทุนที่เหมาะสมให้กับหน่วยองค์กรของทั้งคู่ TIn ด้วยวิธีนี้ คุณช่วยรับประกันว่าต้นทุนที่บันทึกไว้ในโครงการสำหรับ กสิณ และ ชมพูนุช สะท้อนความแตกต่างของต้นทุนอย่างถูกต้องระหว่าง Contoso สหรัฐอเมริการ และ Contoso อินเดีย

### <a name="are-organizational-units-related-to-sales-territories-in-dynamics-365"></a>หน่วยองค์กรเกี่ยวข้องกับเขตการขายใน Dynamics 365 หรือไม่

ไม่มีการเชื่อมโยงหรือความสัมพันธ์ระหว่างเขตการขายและหน่วยองค์กร โดยทั่วไปเขตการขายจะเป็นพื้นที่ทางภูมิศาสตร์ที่มีการขายเกิดขึ้น รายการราคาขายสามารถเชื่อมโยงกับเขตการขายแต่ละเขตได้

หน่วยองค์กรเป็นกลุ่มหรือฝ่ายภายในของบริษัทที่ติดตามต้นทุน สำหรับชุดของบทบาทที่จะขายให้กับฝ่ายอื่นๆ หรือให้กับลูกค้าที่เป็นบุคคลภายนอก

#### <a name="example-of-organizational-units-and-sales-territories"></a>ตัวอย่างของหน่วยองค์กรและเขตการขาย

Contoso, Ltd. มีศูนย์พัฒนาสองศูนย์: Contoso สหรัฐอเมริกา และ Contoso อินเดีย ต้นทุนของทรัพยากรแตกต่างกันอย่างมากระหว่างศูนย์พัฒนาทั้งสองแห่งนี้

Contoso ขายบริการด้านไอทีในตลาดต่างประเทศจำนวนมาก เช่น ละตินอเมริกา อเมริกาเหนือ เอเชียแปซิฟิก ยุโรปตะวันตก และตะวันออกกลาง อัตราเรียกเก็บเงินสำหรับบทบาทโครงการเดียวกัน อาจแตกต่างกันไปในแต่ละตลาด

Contoso สหรัฐอเมริการ และ Contoso อินเดีย ควรถูกตั้งค่าเป็นหน่วยองค์กร และแต่ละหน่วยองค์กรควรมีรายการราคาต้นทุนของตนเอง เอเชียแปซิฟิก ละตินอเมริกา อเมริกาเหนือ ยุโรปตะวันตก และตะวันออกกลาง ควรจะตั้งค่าเป็นเขตการขาย และแต่ละเขตการขายควรจะมีรายการราคาขายของตัวเอง

### <a name="why-is-there-a-restriction-on-the-association-of-price-lists-with-organizational-units"></a>เหตุใดจึงมีข้อจำกัดเกี่ยวกับความสัมพันธ์ของรายการราคากับหน่วยองค์กร 

การกำหนดราคาขายมักจะไม่ซ้ำกับพื้นที่ทางภูมิศาสตร์หรือตลาดที่มีการขายบริการ ฝ่ายภายในของบริษัท โดยปกติจะไม่มีการกำหนดราคาขายของตนเองสำหรับบริการชนิดเดียวกัน อย่างไรก็ตาม ฝ่ายภายในมีต้นทุนที่แตกต่างกันของสินค้าที่ขาย (COGS) ขึ้นอยู่กับทักษะของผู้คนที่พวกเขาจ้าง และสภาพแรงงานของภูมิภาคที่พวกเขาดำเนินงานอยู่ เนื่องจากหน่วยองค์กรถูกจำลองเป็นส่วนภายในของบริษัท จึงสามารถมีเพียงรายการราคาต้นทุนเท่านั้น

### <a name="why-cant-we-associate-sales-price-lists-to-organizational-units"></a>เหตุใดเราจึงไม่สามารถเชื่อมโยงรายการราคาขายกับหน่วยองค์กรได้

ใน PSA รายการราคาขายสามารถเชื่อมโยงกับลูกค้าและเขตการขายได้ เอนทิตีธุรกรรม เช่น โอกาสทางการขาย ใบเสนอราคา สัญญาโครงการ และโครงการ ใช้รายการราคาขายที่แนบกับบัญชีลูกค้าหรือเขตการขาย เพื่อกำหนดอัตราการเรียกเก็บเงินและรายได้ที่อาจเกิดขึ้น ในการมีส่วนร่วมของโครงการ

รายการราคาต้นทุนเชื่อมโยงกับหน่วยองค์กร เอนทิตีของธุรกรรม เช่น โอกาสทางการขาย ใบเสนอราคา สัญญาโครงการ และโครงการ ใช้รายการราคาต้นทุนที่แนบกับหน่วยตามสัญญา เพื่อกำหนดต้นทุนให้กับการมีส่วนร่วมของโครงการ

### <a name="are-organizational-units-hierarchical-in-psa"></a>หน่วยองค์กรเป็นลำดับชั้นใน PSA หรือไม่

ไม่ ในรุ่นปัจจุบันของ PSA หน่วยองค์กรไม่ได้เป็นลำดับชั้น ซึ่งหมายความว่าคุณไม่สามารถ:

- ตั้งค่าคอนฟิกรูปแบบสำหรับราคาทุนเริ่มต้นที่ทำให้ลำดับชั้นขึ้น 
- รายงานรายได้หรือต้นทุนที่เพิ่มขึ้นในระดับต่างๆ ของลำดับชั้นของหน่วยองค์กร

### <a name="were-a-small-professional-services-company-that-operates-as-only-one-division-how-can-we-best-use-the-organizational-unit-concept-in-the-current-version-of-psa"></a>เราเป็นบริษัทที่ให้บริการระดับมืออาชีพขนาดเล็กที่ดำเนินการเป็นเพียงฝ่ายเดียว เราจะสามารถใช้แนวคิดหน่วยขององค์กรให้ดีที่สุด ใน PSA รุ่นปัจจุบันได้อย่างไร

### <a name="when-a-project-is-created-from-a-quote-or-project-contract-line-the-default-contracting-unit-comes-from-the-quote-or-project-contract-if-a-project-is-created-before-sales-entities-such-as-quote-or-project-contract-what-is-the-default-contracting-unit"></a>เมื่อโครงการถูกสร้างจากใบเสนอราคาหรือรายละเอียดการให้บริการตามสัญญาของโครงการ หน่วยตามสัญญาเริ่มต้นจะมาจากใบเสนอราคาหรือสัญญาโครงการ หากโครงการถูกสร้างขึ้นก่อนเอนทิตีการขาย เช่น ใบเสนอราคา หรือสัญญาโครงการ หน่วยตามสัญญาเริ่มต้นคืออะไร

เมื่อโครงการถูกสร้างด้วยตนเอง หน่วยตามสัญญาเริ่มต้นของโครงการจะขึ้นอยู่กับผู้ใช้ที่สร้างโครงการขึ้นมา ผู้ใช้รายนั้นยังเป็นผู้จัดการโครงการเริ่มต้นด้วย หากโครงการถูกแม็ปกับเอนทิตีการขาย เช่น ใบเสนอราคาหรือสัญญาโครงการ หน่วยตามสัญญาในโครงการจะขึ้นอยู่กับเอนทิตีการขายแทน ในกรณีนี้ การประเมินโครงการอาจถูกคำนวณใหม่ เนื่องจากรายการราคาต้นทุนถูกใช้เพื่อคำนวณการเปลี่ยนแปลงการประเมินต้นทุน ถ้ามีการเปลี่ยนแปลงหน่วยตามสัญญา รายการราคาขายจะใช้ในการคำนวณการประเมินการขาย ที่จะมีการเปลี่ยนแปลงเพื่อให้มีการซิงค์กับรายการราคาของโครงการในใบเสนอราคา

ฟิลด์ **หน่วยตามสัญญา** และ **สกุลเงิน** ในโครงการถูกล็อกเพื่อแก้ไข เนื่องจากต้องอยู่ในการซิงค์กับค่าในเอนทิตีการขาย (ใบเสนอราคาหรือสัญญาโครงการ) ที่มีการแม็ปกับโครงการ
