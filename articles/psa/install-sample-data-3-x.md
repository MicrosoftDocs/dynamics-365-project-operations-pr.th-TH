# <a name="sample-data-installation-for-the-project-service-application"></a>การติดตั้งข้อมูลตัวอย่างสำหรับแอพลิเคชัน Project Service

เพื่อช่วยคุณในการสร้างสภาพแวดล้อมสาธิตของคุณเอง Microsoft มีแพคเกจข้อมูลตัวอย่างที่สามารถดาวน์โหลดได้ ที่แสดงความสามารถของแอปของคุณ มีแพคเกจข้อมูลตัวอย่างสองชนิด ได้แก่:
- ข้อมูลการอ้างอิง/การตั้งค่า
- ข้อมูลสาธิต (การตั้งค่า/การอ้างอิง และข้อมูลธุรกรรม เช่น ใบสั่งผลิตและโครงการ)

แพคเกจข้อมูล **การอ้างอิง** ตัวอย่าง สามารถดาวน์โหลดได้ในแพคเกจที่แตกต่างกันสามแพคเกจ ดังนั้นคุณสามารถติดตั้งข้อมูลได้เฉพาะสำหรับ Project Service หรือเฉพาะสำหรับ Field Service หรือคุณสามารถติดตั้งข้อมูลตัวอย่างได้สำหรับแอพลิเคชันทั้งสองในเวลาเดียวกัน

แพคเกจข้อมูลการอ้างอิง/การตั้งค่าของตัวอย่าง ได้แก่:

- [**V902PSMasterData** - Project Service รุ่น 3.x เท่านั้น](https://go.microsoft.com/fwlink/?linkid=2026540&clcid=0x409)

- [**V902FSMasterData** - Field Service รุ่น 8.x เท่านั้น](https://go.microsoft.com/fwlink/?linkid=2026536&clcid=0x409)

- [**V902FPSMasterData** - Field Service 8.x และ Project Service 3.x](https://go.microsoft.com/fwlink/?linkid=2026041&clcid=0x409)

แพคเกจข้อมูล **สาธิต** ล่าสุด คือ:

 - [**FPSDemoData** - Field Service 8.x และ Project Service 3.x](https://aka.ms/fpsdemodatapackage)

   คำแนะนำในการติดตั้งแตกต่างเล็กน้อยในการที่ผู้ใช้สร้าง และตั้งค่าคอนฟิกส่วน แต่ส่วนเหลือจะเหมือนกับ [**โพสต์ในบล็อก**](https://aka.ms/fpsdemodatablog) ก่อนหน้า แพคเกจนี้มีคุณลักษณะชุดข้อมูลสาธิตที่ลดลง และใช้เวลาประมาณ 3 ชั่วโมงในการติดตั้ง

แพคเกจข้อมูลตัวอย่างเหล่านี้พร้อมใช้งานในภาษาอังกฤษเท่านั้น

> [!IMPORTANT]
> **ไม่มีวิธีในการถอนการติดตั้งข้อมูลตัวอย่าง** คุณควรติดตั้งแพคเกจเหล่านี้ในการสาธิต การประเมิน การฝึก หรือระบบทดสอบเท่านั้น นอกจากนี้ โปรดทราบว่าการติดตั้งแพคเกจแต่ละรายการ และจากนั้นการติดตั้งแพคเกจอื่นแต่ละรายการ ไม่ได้รับการสนับสนุน (อีกนัยหนึ่ง คุณไม่สามารถติดตั้ง **FSMasterData** ที่ตามด้วย **PSMasterData** หรือในทางกลับกันได้) ถ้าคุณเห็นตัวคุณเองต้องการข้อมูลตัวอย่างสำหรับทั้งแอพลิเคชันที่เวลาใดๆ ในอนาคต คุณควรติดตั้งแพคเกจ **v902FPSMasterData**

เมื่อคุณติดตั้งแพคเกจข้อมูลตัวอย่างใดๆ กระบวนการติดตั้งทำการดำเนินการต่อไปนี้:

- สร้าง หรือตั้งค่าพารามิเตอร์เริ่มต้นสำหรับการใช้ Project Service Field Service หรือแอพลิเคชันทั้งสอง (ถ้ามี)

- นำเข้าข้อมูลตัวอย่างสำหรับแอพลิเคชัน เช่น ทรัพยากรที่สามารถจองได้ บทบาทเฉพาะแอพลิเคชัน การขาย และรายการราคาต้นทุน หน่วยองค์กร เรกคอร์ดกระบวนการขาย และเอนทิตีอื่น เพื่อสาธิตความสามารถหลัก  

ด้วยแพคเกจ **ข้อมูลสาธิต** คุณได้รับข้อมูลธุรกรรมด้านบนและเพิ่มเติม เช่น ใบสั่งผลิต และโครงการ

สงสัยว่า ความสามารถใดที่คุณสามารถสาธิตได้ด้วยข้อมูลตัวอย่าง? ดูสถานการณ์สมมติ Fabrikam Robotics ใน [หมายเหตุด้านเทคนิค](#technical-notes)

ถ้าคุณมีคำถามเกี่ยวกับการติดตั้งแพคเกจข้อมูลตัวอย่างเหล่านี้ [ส่งอีเมลให้เราที่ fpsdemodata@microsoft.com](mailto:fpsdemodata@microsoft.com)

## <a name="requirements"></a>ความต้องการ

โพรโทคอลการติดตั้งสันนิษฐานรายการต่อไปนี้เกี่ยวกับอินสแตนซ์เป้าหมายของคุณ (องค์กร):

- ภาษาพื้นฐานเป็นภาษาอังกฤษ และสกุลเงินพื้นฐานเป็นดอลลาร์สหรัฐฯ (USD,$)

- องค์กรไม่มีข้อมูล Project Service หรือ Field Service อยู่ หรือมีเฉพาะข้อมูลเริ่มต้น barebones ที่มาพร้อมกับองค์กรใหม่ใดๆ

- รุ่นที่ถูกต้องของแอพลิเคชันทางธุรกิจถูกติดตั้งเรียบร้อยแล้ว:
       
    - **สำหรับ FPSDemoData or v902FPSMasterData:** องค์กรมี Field Service รุ่น 8.x และ Project Service รุ่น 3.x ที่ติดตั้งอยู่

    - **สำหรับ v902PSMasterData:** องค์กรมี Project Service รุ่น 3.x ที่ติดตั้งอยู่

    - **สำหรับ v902FSMasterData** องค์กรมี Field Service รุ่น 8.x ที่ติดตั้งอยู่

> [!NOTE]
> ถ้าคุณจำเป็นต้องติดตั้งข้อมูลตัวอย่างด้านบนของการทดลองใช้ Project Service และ Field Service ที่มีอยู่ หรือสภาพแวดล้อมสาธิตที่มีข้อมูล (ไม่แนะนำ) อยู่แล้ว คุณจะจำเป็นต้องหยุดการตรวจสอบล่วงหน้าด้านความปลอดภัยชั่วคราวที่ดำเนินการโดยผู้ติดตั้ง สำหรับข้อมูลเพิ่มเติม ดูหมายเหตุด้านเทคนิคด้านล่าง

## <a name="prepare-for-installation"></a>เตรียมสำหรับการติดตั้ง

คุณต้องการเรียกใช้ตัวติดตั้งบนคอมพิวเตอร์ที่มีรุ่นล่าสุดของ Windows (ต้องการ Windows 10)

คุณควรวางแผนเพื่อให้คอมพิวเตอร์ยังคงเชื่อมต่อกับเครือข่าย และเพื่อให้การติดตั้งรันได้นานถึง **1 ชั่วโมง** สำหรับ **ข้อมูลการอ้างอิง/การตั้งค่า** (โดยปกติ การติดตั้งใช้เวลาประมาณ 30 นาทีสำหรับ **FPSMasterData** ซึ่งรวมถึงข้อมูลตัวอย่างสำหรับแอพลิเคชันทั้งสอง) สำหรับ **FPSDemoData** การติดตั้งจะใช้เวลาประมาณ **3 ชั่วโมง**

คอมพิวเตอร์ควรมีการปิดใช้งานฟังก์ชันโปรแกรมรักษาหน้าจอ มิฉะนั้น ข้อมูลประจำตัวของเซสชันสำหรับการติดตั้งอาจหายไป เมื่อโปรแกรมรักษาหน้าจอเกี่ยวข้อง (เว้นแต่ว่าคุณรักษาเซสชันของคุณให้ใช้งานอยู่ตลอด)

> [!div class="mx-imgBorder"]
> ![ภาพหน้าจอของการตั้งค่าโปรแกรมรักษาหน้าจอ ที่มีโปรแกรมรักษาหน้าจอที่ปิดใช้งานอยู่](media/sample-data-1.png)

## <a name="download-and-unpack"></a>ดาวน์โหลด และแยก

มีการกระจายตัวติดตั้งข้อมูลตัวอย่างของ Field Service และ Project Service เป็นการปฏิบัติการที่แยกเองได้ ชื่อแฟ้มอาจแตกต่างกันไปโดยขึ้นอยู่กับแพคเกจข้อมูลตัวอย่าง แต่มิฉะนั้น ขั้นตอนจะเหมือนกัน ไม่ว่าคุณจะติดตั้งแพคเกจใด

หลังจากการดาวน์โหลดแพคเกจ เรียกใช้แฟ้ม .exe และจากนั้นยอมรับข้อกำหนดและเงื่อนไขเพื่อแยกแฟ้ม zip ที่บีบอัดแล้ว จากนั้น คุณต้องขยายเนื้อหาของไฟล์นั้นไปยังโฟลเดอร์บนคอมพิวเตอร์

โดยขึ้นอยู่กับระบบปฏิบัติการและการตั้งค่าการรักษาความปลอดภัย คุณอาจต้องดำเนินการขั้นตอนต่อไปนี้หลังจากการแยกแฟ้ม zip:

1. ค้นหา และคลิกขวาที่แฟ้ม **FPSDemoData.dll** ในโฟลเดอร์ **v902FPSMasterData** / **PackageDeployer_FPSDemoData**

2. เลือก **ยกเลิกบล็อก**

3. เลือก **ใช้**

4. เลือก **ตกลง**


## <a name="create-or-configure-users"></a>สร้าง หรือตั้งค่าคอนฟิกผู้ใช้

แพคเกจ **FPSDemoData** จำเป็นต้องมีผู้ใช้หกราย ขณะที่แพคเกจ **FPSMasterData** จำเป็นต้องมีผู้ใช้หนึ่งราย อ้างอิงถึงรายการที่ถูกต้องสำหรับแพคเกจของข้อมูลตัวอย่างของคุณ

## <a name="create-or-configure-users---setupreference-data-packages"></a>สร้าง หรือตั้งค่าคอนฟิกผู้ใช้ - แพคเกจข้อมูลการตั้งค่า/การอ้างอิง

แพคเกจ **FPSMasterData** มีวัตถุประสงค์เพื่อการติดตั้งด้วยผู้ใช้หนึ่งรายที่ชื่อ Spencer Low ซึ่งมีการตั้งค่าที่อธิบายไว้ที่นี่ เพื่อติดตั้งแพคเกจให้ถูกต้อง คุณจำเป็นต้องสร้าง (หรือเปลี่ยนชื่อชั่วคราว) ผู้ใช้ในสภาพแวดล้อมของคุณ ให้ตรงกับการตั้งค่าคอนฟิกข้อมูลตัวอย่างขาเข้า

เมื่อต้องการสร้าง หรือตั้งค่าคอนฟิกผู้ใช้ ไปที่ **การตั้งค่า** > **ความปลอดภัย** > **ผู้ใช้** และทำสิ่งต่อไปนี้:

1. ตั้งค่า UserFullname = "Spencer Low" ที่มีชื่อผู้ใช้ "spencerl" (**ตัวพิมพ์เล็ก**) เป็นบทบาทผู้จัดการโครงการและผู้จัดการฝ่ายปฏิบัติงาน

2. เลือกผู้ใช้ **Spencer Low** และจากนั้น เลือก **จัดการบทบาท** ค้นหา และเลือกบทบาท **ผู้ดูแลระบบ** และจากนั้นเลือก **ตกลง** เพื่อให้สิทธิผู้ดูแลแบบเต็มรูปแบบกับ Spencer Low ขั้นตอนนี้จำเป็นในการตรวจสอบว่า เรกคอร์ดตัวอย่างถูกสร้างขึ้นด้วยความเป็นเจ้าของผู้ใช้ที่ถูกต้อง และดังนั้น จึงเติมข้อมูลมุมมองได้อย่างถูกต้อง

3. จากแพคเกจที่ดาวน์โหลด คุณจำเป็นต้องปรับปรุงแฟ้มการแมปข้อมูลด้วยที่อยู่อีเมลของบริบทของผู้ใช้เริ่มต้น เมื่อต้องการทำเช่นนี้ เปิด **PkgFolder** และจากนั้นค้นหา และเปิดไฟล์ **ImportUserMapFile.xml** ใน Notepad (หรือ Visual Studio หรือตัวแก้ไข XML อื่น) ตั้งค่าฟิลด์ **DefaultUserToMapTo =** เป็นที่อยู่อีเมลของผู้ใช้ Spencer Low

4. ถ้าคุณไม่ได้ใช้ Spencer Low ที่มีชื่อผู้ใช้ **spencerl** คุณจำเป็นต้องปรับปรุงแฟ้มเพิ่มเติม เปิดแฟ้ม **DemoDataPreImportConfig.xml** และจากนั้น ค้นหาแท็ก **userstocreateandconfigure** ปรับปรุงแท็ก **\<login\>** ที่มีชื่อผู้ใช้ของผู้ใช้ SpencerLow ของคุณ สำหรับราบละเอียดเพิ่มเติม ดู [หมายเหตุด้านเทคนิค](#technical-notes)

## <a name="create-or-configure-users---demo-data-package"></a>สร้าง หรือตั้งค่าคอนฟิกผู้ใช้ - แพคเกจข้อมูลสาธิต

แพคเกจข้อมูลสาธิตจำเป็นต้องมีผู้ใช้หกราย สำหรับแพคเกจในการติดตั้งอย่างถูกต้อง ทำสิ่งต่อไปนี้:

 1. สร้าง หรือเปลี่ยนชื่อผู้ใช้ที่มีอยู่ชั่วคราวให้ตรงกับการตั้งค่าคอนฟิกข้อมูลตัวอย่างขาเข้า โดยการไปยัง **การตั้งค่า** > **ความปลอดภัย** > **ผู้ใช้**
 
    บทบาทเหล่านี้จำเป็นต้องใช้เฉพาะสำหรับการสาธิตส่วนบุคคล  
    - ชื่อเต็มของผู้ใช้ = "David So" เป็นช่างเทคนิค Field Service   
    - ชื่อเต็มของผู้ใช้ = "Jamie Reding" เป็น Customer Service & ผู้จัดส่ง Field Service   
    - ชื่อเต็มของผู้ใช้ = "Molly Clark" เป็นผู้จัดการฝ่ายลูกค้าองค์กร   
    - ชื่อเต็มของผู้ใช้ = "Spencer Low" เป็นผู้จัดการด้านการปฏิบัติการและโครงการ  
    - ชื่อเต็มของผู้ใช้ = "Veronica Quek" เป็นสมาชิกในทีม   
    - ชื่อเต็มของผู้ใช้ = "William Contoso"
  
2. สำหรับวัตถุประสงค์ในการนำเข้าข้อมูลสาธิต มอบหมายผู้ใช้หกรายเหนือบทบาทผู้ดูแลระบบ เพื่อนำเข้าเรกคอร์ดตัวอย่างอย่างถูกต้อง 

3. เปิด **PkgFolder** แล้วจากนั้น ค้นหา และเปิด **ImportUserMapFile.xml** ปรับปรุงฟิลด์ **ใหม่ =** เป็นที่อยู่อีเมลของผู้ใช้ที่สอดคล้องกันในระบบของคุณ

## <a name="technical-notes"></a>หมายเหตุด้านเทคนิค

ดูด้านล่างสำหรับรายละเอียดทางเทคนิคเพิ่มเติมเกี่ยวกับการติดตั้งของข้อมูลนี้

### <a name="installing-sample-data-on-top-of-existing-data-not-recommended"></a>การติดตั้งข้อมูลตัวอย่างลงบนข้อมูลที่มีอยู่ (ไม่แนะนำ)

ถ้าคุณจำเป็นต้องติดตั้งข้อมูลตัวอย่างบนการทดลองใช้ Field Service หรือ Project Service ที่มีอยู่ หรือสภาพแวดล้อมสาธิต ที่มีข้อมูลอยู่แล้ว คุณจะจำเป็นต้องหยุดการตรวจสอบล่วงหน้าด้านความปลอดภัยชั่วคราวที่ดำเนินการโดยผู้ติดตั้ง

เมื่อต้องการทำเช่นนี้ ไปยังโฟลเดอร์ **PkgFolder** เพื่อค้นหา และเปิดแฟ้ม **DemoDataPreImportConfig.xml** ที่มี Notepad (หรือตัวแก้ไข XML อื่น)

ค้นหาค่าต่อไปนี้ และจากนั้น เปลี่ยนแปลงการตั้งค่าจากจริงเป็นเท็จ:

```alias
<TerminateOnPreCheckFailure>true</TerminateOnPreCheckFailure>
```

การเปลี่ยนแปลงนี้ทำให้ตัวติดตั้งข้ามการตรวจสอบด้านความปลอดภัยที่สำคัญบางอย่าง ซึ่งรวมถึง:

- การยืนยันว่า ไม่มีเรกคอร์ด **หน่วยองค์กร** ที่ใช้งานอยู่มากกว่าหนึ่งหน่วย และจากนั้น เปลี่ยนชื่อเป็น **Fabrikam US**

- การยืนยันว่า ไม่มีเรกคอร์ด **เท็มเพลตงาน** ที่ใช้งานอยู่มากกว่าหนึ่งรายการ

- การยืนยันว่า ไม่มีเรกคอร์ด **พารามิเตอร์โครงการ** ที่ใช้งานอยู่มากกว่าหนึ่งรายการ และจากนั้น เปลี่ยนชื่อรายการนั้นเป็น **พารามิเตอร์**

### <a name="configuration-components"></a>คอมโพเนนต์การตั้งค่าคอนฟิก

มีจำนวนของคอมโพเนนต์การตั้งค่าคอนฟิกอื่นๆ มากมายในแฟ้มการตั้งค่าคอนฟิกการนำเข้าล่วงหน้านี้ สำหรับผู้ใช้ทางเทคนิค รายการเหล่านี้บางรายการจะรวมถึง:

- **\<RequiredSolutions\>** ระบุการติดตั้งข้อกำหนดของโซลูชันเบื้องต้นและหมายเลขเวอร์ชัน

- **\<InstallSampleData\>** ควบคุมว่าข้อมูลตัวอย่างแบบสำเร็จรูปและแอปการบริการถูกติดตั้งไว้หรือไม่

    - เท็จ - ข้ามการติดตั้งของข้อมูลภายในนี้ (ซึ่งเอาออกได้)

    - จริง - ติดตั้งการทำงานพร้อมกันของข้อมูลภายในที่มีการติดตั้งของข้อมูลตัวอย่าง FS และ PSA

- **\<PreImportDataCollection\>** ระบุแผนผังข้อมูลแฟ้มคงที่ และเรกคอร์ดที่เชื่อมโยง ที่จะนำเข้าล่วงหน้าการติดตั้งข้อมูลตัวอย่างหลัก

- **\<EntitiesToEnableScheduling\>** ระบุเอนทิตีที่ควรเปิดใช้งานสำหรับการจองใน Microsoft Dynamics Scheduling (หรือเรียกอีกอย่างหนึ่งว่า Universal Resource Scheduling)

- **\<UsersToCreateAndConfigure\>** ระบุทรัพยากรที่สามารถจองได้ที่จะถูกสร้างขึ้น (ถ้าไม่มีอยู่ก่อนแล้ว) ก่อนที่จะดำเนินการนำเข้าข้อมูลตัวอย่าง โปรดสังเกตว่า ทรัพยากรที่สามารถจองได้ของข้อมูลตัวอย่างระบบแหล่งที่มา ตรงกับเรกคอร์ดทรัพยากรที่สามารถจองได้ของระบบเป้าหมายในชื่อเต็มและการเข้าสู่ระบบของทรัพยากรแต่ละรายการ ดังนั้น จึงไม่สามารถเปลี่ยนชื่อในแฟ้มการตั้งค่าคอนฟิกล่วงหน้านี้ได้ นอกเสียจากคุณนำเข้าข้อมูลตัวอย่างลงในระบบปลายทางโดยใช้ชื่อเหล่านี้ก่อน จากนั้นเปลี่ยนชื่อทรัพยากรที่สามารถจองได้เป็นชื่อที่ต้องการของคุณ ซึ่งตั้งค่าพร้อมกับเรกคอร์ดผู้ใช้ที่เปิดใช้งานแล้ว และจากนั้น ส่งออกข้อมูลอีกครั้งสำหรับการนำเข้ามาในระบบปลายทางสุดท้ายของคุณ (ซึ่งปรับปรุงรายการเก่าและใหม่ของ **ImportUserMapFile.xml** ให้สอดคล้องกัน)

- **\<PluginsToDisable\>** ระบุปลั๊กอินของรายการที่แยกต่างหากที่ต้องถูกปิดใช้งาน ในระหว่างการนำเข้าข้อมูลตัวอย่าง และถูกเปิดใช้งานอีกครั้งในภายหลัง

### <a name="fabrikam-robotics-fictitious-scenario"></a>สถานการณ์สมมติ Fabrikam Robotics

แพ็คเกจข้อมูลการอ้างอิงตัวอย่างของ Field Service และ Project Service ติดตั้ง **โซลูชัน Fabrikam Manufacturing Master Data (v3.0.0.0)** พร้อมกับเรกคอร์ด 4,000 รายการ โดยประมาณ และเอนทิตีที่แตกต่างกันประมาณ 40 รายการ แพคเกจข้อมูลตัวอย่างที่แยกต่างหากสำหรับ Field Service หรือ Project Service ประกอบด้วย เซตย่อยของข้อมูลตัวอย่าง **v902FPSMasterData** สำหรับแอพลิเคชันนั้น แพคเกจ **ข้อมูลสาธิต** ติดตั้ง **โซลูชัน Fabrikam Manufacturing Demo Data (v3.0.0.7)** ที่มีเรกคอร์ดประมาณ 22,000 รายการ ในเอนทิตี 148 รายการ

บริษัทสมมติ Fabrikam Robotics เป็นผู้ผลิตของหุ่นยนต์ในงานแอสเซมบลีอุปกรณ์อิเล็กทรอนิกส์ และเป็นที่รู้จักในด้านคุณภาพของผลิตภัณฑ์ นวัตกรรม และส่วนบริการลูกค้าที่แข็งแกร่ง ซึ่งรวมถึงการวางแผนการติดตั้ง การนำไปใช้งาน และการบริการบำรุงรักษาที่ต่อเนื่อง Fabrikam เป็นสำนักงานใหญ่ในสหรัฐอเมริกา (Fabrikam US) และมีการดำเนินการบริการตามโครงการในฝรั่งเศส อินเดีย สหราชอาณาจักร และสวิตเซอร์แลนด์

การดำเนินงาน Field service เป็นศูนย์กลางในสหรัฐอเมริกา ส่วนใหญ่ในพื้นที่ซีแอตเทิลส่วนมาก บริษัทจะมุ่งใช้ประโยชน์จากการเชื่อมต่อ Internet of Things (IoT) เมื่อต้องการตรวจสอบประสิทธิภาพสินทรัพย์ของลูกค้า และจัดส่งบริการนอกสถานที่เชิงรุกเพิ่มขึ้นเรื่อยๆ

ภาพรวมระดับสูงของข้อมูลตัวอย่างมีดังต่อไปนี้:

- องค์ประกอบข้อมูลตัวอย่างทั่วไป (ซึ่งรวมสำหรับแอพลิเคชันทั้งสอง)

    - สินทรัพย์ของลูกค้า 22 รายการ

    - ชนิดของปัญหา 9 ชนิดที่มีช่วงของลักษณะทรัพยากรที่เกี่ยวข้อง (9) การบริการ (13) และภารกิจการให้บริการ (13)
   
แพคเกจ **ข้อมูลสาธิต** ติดตั้งใบสั่งงานประมาณ 179 ใบ โครงการ 12 โครงการ และข้อมูลเกี่ยวกับธุรกรรมที่เกี่ยวข้อง 

### <a name="change-the-work-hours-for-sample-resources"></a>เปลี่ยนแปลงชั่วโมงทำงานสำหรับทรัพยากรตัวอย่าง

โดยค่าเริ่มต้น ทรัพยากรที่สามารถจองได้ทั้งหมดมีปฏิทินชั่วโมงทำงาน 24 ชั่วโมง

ถ้าคุณต้องการเปลี่ยนแปลงชั่วโมงทำงานสำหรับทรัพยากรที่สามารถจองได้ของตัวอย่าง ไปยัง **Universal Resource Scheduling** > **การจัดกำหนดการ** > **ทรัพยากร**

เลือกผู้ใช้ (ตัวอย่างเช่น Spencer Low) และเปลี่ยนแปลงชั่วโมงทำงานของ Spencer เป็นชั่วโมงที่คุณต้องการนำไปใช้กับผู้ใช้หลายราย ไปยัง **Universal Resource Scheduling** > **การตั้งค่า** > **แม่แบบชั่วโมงทำงาน** และแก้ไขเรกคอร์ด **แม่แบบงานเริ่มต้น** ในฟิลด์ **เท็มเพลตทรัพยากร** เลือกผู้ใช้ที่มีชั่วโมงการทำงานที่คุณต้องการนำไปใช้กับทรัพยากรอื่น ไปที่ **Universal Resource Scheduling** > **การจัดกำหนดการ** > **ทรัพยากร** > **ทรัพยากรที่สามารถจองได้** เลือกทรัพยากรที่คุณต้องการเปลี่ยนแปลง และจากนั้น เลือก **ตั้งค่าปฏิทิน** ในรายการแบบหล่นลง **เท็มเพลตงาน** เลือกเท็มเพลต **ชั่วโมงการทำงานเริ่มต้น** หรือเท็มเพลตอื่นที่มีทรัพยากรการจัดเท็มเพลตที่ถูกต้อง เมื่อคุณไปยังบอร์ดกำหนดการ คุณควรจะเห็นว่า ขณะนี้ทรัพยากรได้ปรับปรุงชั่วโมงทำงาน
