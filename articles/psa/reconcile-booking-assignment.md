<span data-ttu-id="b8445-104">การจองโครงการของสมาชิกทีมโครงการและการมอบหมายงานโครงการเป็นคู่แบบหลวมๆ</span><span class="sxs-lookup"><span data-stu-id="b8445-104">A project team member's project bookings and project task assignments are loosely coupled.</span></span> <span data-ttu-id="b8445-105">ดังนั้น ทรัพยากรสามารถมีการมอบหมายงานที่ไม่สอดคล้องกับการจอง และการจองที่ไม่สอดคล้องกับการมอบหมายงาน</span><span class="sxs-lookup"><span data-stu-id="b8445-105">Therefore, a resource can have task assignments that don't correspond to bookings and bookings that don't correspond to task assignments.</span></span> <span data-ttu-id="b8445-106">ตามหลักการแล้ว การจองและการมอบหมายโครงการจะถูกจัดตำแหน่ง เพื่อให้ทรัพยากรมีกำลังการผลิตที่ยืนยันที่จะปฏิบัติงานที่ได้รับมอบหมาย</span><span class="sxs-lookup"><span data-stu-id="b8445-106">Ideally, project bookings and assignments are aligned, so that resources have committed capacity to perform their task assignments.</span></span> <span data-ttu-id="b8445-107">อย่างไรก็ตาม ความเป็นจริงคือการจองที่สามารถเกิดขึ้นได้ตามความพร้อมใช้งาน และการกำหนดเวลาที่สามารถเปลี่ยนแปลงได้ เนื่องจากโครงการยังคงอยู่ตลอดวงจรชีวิต</span><span class="sxs-lookup"><span data-stu-id="b8445-107">However, the reality is that bookings can occur based on availability, and task timings can change as the project continues through its lifecycle.</span></span> <span data-ttu-id="b8445-108">ดังนั้น การจับเป็นคู่กันอย่างหลวมๆ ช่วยให้มีความยืดหยุ่น</span><span class="sxs-lookup"><span data-stu-id="b8445-108">Therefore, the loose coupling allows for flexibility.</span></span>

<span data-ttu-id="b8445-109">เนื่องจากการจับเป็นคู่กันอย่างหลวมๆ ของการจองโครงการ และการมอบหมายงาน แท็บ **การกระทบยอด** จะรวมอยู่ในเอนทิตีโครงการ</span><span class="sxs-lookup"><span data-stu-id="b8445-109">Because of the loose coupling of project bookings and task assignments, a **Reconciliation** tab is included on the Project entity.</span></span> <span data-ttu-id="b8445-110">แท็บนี้ช่วยให้ผู้จัดการโครงการกระทบยอดการจองของสมาชิกในทีมและงานมอบหมายสำหรับทีมโครงการของพวกเขา</span><span class="sxs-lookup"><span data-stu-id="b8445-110">This tab helps project managers reconcile team members' bookings and their assignments for their project team.</span></span>

<span data-ttu-id="b8445-111">สำหรับสมาชิกทีมที่ระบุชื่อแต่ละคน แท็บ **การกระทบยอด** แสดงการจองและการมอบหมายไปจนถึงการมอบหมายงานแต่ละคน</span><span class="sxs-lookup"><span data-stu-id="b8445-111">For each named team member, the **Reconciliation** tab shows bookings and assignments down to the individual task assignment.</span></span> <span data-ttu-id="b8445-112">จะแสดงชั่วโมงในเซลล์ที่สามารถแสดงช่วงเวลาตั้งแต่เดือนไปจนถึงวัน</span><span class="sxs-lookup"><span data-stu-id="b8445-112">It shows hours in cells that can represent periods from months down to days.</span></span>

<span data-ttu-id="b8445-113">ในฟิลด์ **สเกลเวลา** คุณสามารถเลือก **เดือน** **สัปดาห์** หรือ **วัน**</span><span class="sxs-lookup"><span data-stu-id="b8445-113">In the **Timescale** field, you can select **Month**, **Week**, or **Day**.</span></span> <span data-ttu-id="b8445-114">โดยค่าเริ่มต้น **สัปดาห์** จะถูกเลือกไว้</span><span class="sxs-lookup"><span data-stu-id="b8445-114">By default, **Week** is selected.</span></span> <span data-ttu-id="b8445-115">อย่างไรก็ตาม คุณสามารถเปลี่ยนค่าเริ่มต้นได้โดยการเลือกปุ่ม **การตั้งค่า**</span><span class="sxs-lookup"><span data-stu-id="b8445-115">However, you can change the default value by selecting the **Settings** button.</span></span> <span data-ttu-id="b8445-116">เมื่อเปิดแท็บ **การกระทบยอด** จะแสดงวันที่ปัจจุบัน แต่คุณสามารถใช้ตัวควบคุมปฏิทิน เพื่อเลื่อนเวลาไปข้างหน้าหรือย้อนกลับได้</span><span class="sxs-lookup"><span data-stu-id="b8445-116">When the **Reconciliation** tab is opened, it shows the current date, but you can use the calendar control to move forward or backward in time.</span></span> <span data-ttu-id="b8445-117">เมื่อโครงการมีวันที่เริ่มต้นที่อยู่ในอนาคต แท็บจะแสดงวันที่นั้นเมื่อมีการเปิด</span><span class="sxs-lookup"><span data-stu-id="b8445-117">When a project has a start date that is in the future, the tab shows that date when it's opened.</span></span> <span data-ttu-id="b8445-118">ตัวควบคุมปฏิทินยังมีตัวเลือกที่ช่วยให้คุณสามารถเลื่อนไปยังวันที่เริ่มต้นและสิ้นสุดของโครงการ</span><span class="sxs-lookup"><span data-stu-id="b8445-118">The calendar control also has options that let you move to the project start and end dates.</span></span>

<span data-ttu-id="b8445-119">คุณสามารถใช้ตัวควบคุมตัวขยายบนแต่ละทรัพยากร เพื่อแสดงรายละเอียดของการจองของทรัพยากรนั้น</span><span class="sxs-lookup"><span data-stu-id="b8445-119">You can use the expander controls on each resource to show the details of that resource's bookings.</span></span> <span data-ttu-id="b8445-120">นอกจากนี้คุณยังสามารถขยายการมอบหมายทรัพยากรแต่ละรายการไปยังระดับของแต่ละงานได้ด้วย</span><span class="sxs-lookup"><span data-stu-id="b8445-120">You can also expand each resource's assignments to the level of the individual task.</span></span>

<span data-ttu-id="b8445-121">ด้านล่างของแท็บ **การกระทบยอด** แสดงผลรวมสุทธิสำหรับโครงการ และแท็บยังมีคอลัมน์รวม</span><span class="sxs-lookup"><span data-stu-id="b8445-121">The bottom of the **Reconciliation** tab shows an overall net total for the project, and the tab also includes a total column.</span></span> <span data-ttu-id="b8445-122">สำหรับทุกทรัพยากร แท็บจะใช้ความแตกต่างระหว่างการจองของสมาชิกในทีมในโครงการ และค่าสะสมของการมอบหมายงานของสมาชิกทีม</span><span class="sxs-lookup"><span data-stu-id="b8445-122">For each resource, the tab takes the difference between a team member's bookings on the project and rollup of that team member's task assignments.</span></span> <span data-ttu-id="b8445-123">ตามหลักการแล้ว ความแตกต่างควรเป็น 0 (ศูนย์)</span><span class="sxs-lookup"><span data-stu-id="b8445-123">Ideally, the difference should be 0 (zero).</span></span> <span data-ttu-id="b8445-124">กล่าวอีกนัยหนึ่ง ไม่ควรจะมีความแตกต่างระหว่างการจองของทรัพยากรและการมอบหมายงานของทรัพยากร</span><span class="sxs-lookup"><span data-stu-id="b8445-124">In other words, there should be no difference between the resource's bookings and its task assignments.</span></span> <span data-ttu-id="b8445-125">ความแตกต่างใดๆ จะถูกบ่งชี้ด้วย สี และแรเงา เพื่อใช้เรียกสองเงื่อนไขต่อไปนี้</span><span class="sxs-lookup"><span data-stu-id="b8445-125">Any differences are indicated by color and shading to call out two conditions:</span></span>

- <span data-ttu-id="b8445-126">**การจองไม่เพียงพอ** – การจองไม่เพียงพอเกิดขึ้นเมื่อทรัพยากรมีการมอบหมายมากกว่าการจอง</span><span class="sxs-lookup"><span data-stu-id="b8445-126">**Booking shortage** – Booking shortages occur when a resource has more assignments than bookings.</span></span> <span data-ttu-id="b8445-127">เนื่องจากกำลังการผลิตนี้ไม่ได้รับการสงวนสิทธิ์ ผู้จัดการโครงการสามารถแก้ไขเงื่อนไขนี้ โดยการขยายการจองของทรัพยากร เพื่อครอบคลุมความขาดแคลน</span><span class="sxs-lookup"><span data-stu-id="b8445-127">Because this capacity hasn't been reserved, a project manager can correct this condition by extending the resource's bookings to cover the shortage.</span></span>
- <span data-ttu-id="b8445-128">**การจองเกินจำนวน** การจองเกินจำนวนจะเกิดขึ้นเมื่อมีการจองทรัพยากรไปยังโครงการ แต่ยังไม่ได้กำหนดให้กับงาน</span><span class="sxs-lookup"><span data-stu-id="b8445-128">**Excess bookings** – Excess bookings occur when a resource has been booked to the project but hasn't been assigned to tasks.</span></span> <span data-ttu-id="b8445-129">เงื่อนไขนี้อาจเป็นที่ยอมรับได้ ถ้า ตัวอย่างเช่น ทรัพยากรมีการจองก่อนที่การมอบหมายงานจะเกิดขึ้น</span><span class="sxs-lookup"><span data-stu-id="b8445-129">This condition might be acceptable if, for example, the resource has been booked before task assignment occurs.</span></span> <span data-ttu-id="b8445-130">อย่างไรก็ตามในกรณีอื่นๆ ทรัพยากรอาจไม่ได้ถูกวางแผนไว้เพื่อจะถูกมอบหมาย</span><span class="sxs-lookup"><span data-stu-id="b8445-130">However, in other cases, the resource might not be planned to be assigned.</span></span> <span data-ttu-id="b8445-131">ในกรณีเหล่านี้ ผู้จัดการโครงการควรพิจารณายกเลิกการจองของทรัพยากร เพื่อให้สามารถใช้กำลังการผลิตสำหรับโครงการอื่น</span><span class="sxs-lookup"><span data-stu-id="b8445-131">In these cases, the project manager should consider canceling the resource's bookings, so that the capacity can be used for another project.</span></span>

> [!NOTE]
> <span data-ttu-id="b8445-132">คำอธิบายสำหรับเงื่อนไขเหล่านี้อาจถูกซ่อนไว้ เพื่อให้มีพื้นที่ว่างสำหรับกริด</span><span class="sxs-lookup"><span data-stu-id="b8445-132">The legend for these conditions might be hidden to leave more room for the grid.</span></span> <span data-ttu-id="b8445-133">ในกรณีนี้ คุณสามารถทำให้คำอธิบายมองเห็นได้โดยการเลือกปุ่ม **การตั้งค่า**</span><span class="sxs-lookup"><span data-stu-id="b8445-133">In this case, you can make the legend visible by selecting the **Settings** button.</span></span>

<span data-ttu-id="b8445-134">ในบางกรณี เมื่อฟิลด์ **สเกลเวลา** ถูกตั้งค่าเป็นระดับที่สูงกว่า **วัน** ความแตกต่างอาจถูกคำนวณเป็น 0 (ศูนย์)</span><span class="sxs-lookup"><span data-stu-id="b8445-134">In some cases, when the **Timescale** field is set to a level that is higher than **Day**, differences might be calculated as 0 (zero).</span></span> <span data-ttu-id="b8445-135">ตัวอย่างเช่น ในระดับ **เดือน** ความแตกต่างสุทธิสำหรับทรัพยากรอาจเป็น 0 (ศูนย์) เพื่อบ่งชี้ว่าการจองเท่ากับการมอบหมาย</span><span class="sxs-lookup"><span data-stu-id="b8445-135">For example, at the **Month** level, the net difference for a resource might be 0 (zero) to indicate that bookings equal assignments.</span></span> <span data-ttu-id="b8445-136">อย่างไรก็ตาม ถ้าคุณดูในระดับ **สัปดาห์** คุณอาจเห็นว่ามีการมอบหมายของชั่วโมงเป็นศูนย์ (0) และการจองของ 40 ชั่วโมงในสัปดาห์แรกของเดือน และการมอบหมายงานของ 40 ชั่วโมง และการจองเป็นศูนย์ (0) ชั่วโมงในสัปดาห์ที่สองของเดือน</span><span class="sxs-lookup"><span data-stu-id="b8445-136">However, if you look at the **Week** level, you might see that there are assignments of 0 (zero) hours and bookings of 40 hours in the first week of the month, and assignments of 40 hours and bookings of 0 (zero) hours in the second week of the month.</span></span> <span data-ttu-id="b8445-137">ถึงแม้ว่า การจองและงานมอบหมายทั้งหมดของเดือนจะเท่ากัน แต่จะแตกต่างกันไปในแต่ละสัปดาห์</span><span class="sxs-lookup"><span data-stu-id="b8445-137">Although the total bookings and assignments for the month are equal, they differ by week.</span></span>

<span data-ttu-id="b8445-138">เมื่อคุณดูเวลาในระดับเวลาที่สูงกว่า แท็บ **การกระทบยอด** แสดงตัวบ่งชี้เซลล์เพื่อแจ้งให้คุณทราบว่ามีความแตกต่างในระดับเวลาที่ต่ำกว่า</span><span class="sxs-lookup"><span data-stu-id="b8445-138">When you view higher time levels, the **Reconciliation** tab shows a cell indicator to notify you that there are differences at lower time levels.</span></span> <span data-ttu-id="b8445-139">ตัวอย่างเช่น ในภาพประกอบต่อไปนี้ ตัวบ่งชี้เซลล์จะปรากฏในเซลล์สำหรับเดือนตุลาคม 2018 สำหรับทรัพยากรที่ชื่อ กฤติกา เกษมศานต์</span><span class="sxs-lookup"><span data-stu-id="b8445-139">For example, in the following illustration, a cell indicator appears in the cell for the month of October 2018 for the resource that is named Katelyn Merritt.</span></span> <span data-ttu-id="b8445-140">ดังนั้น คุณจะเห็นได้ว่า แม้ว่าการจองและการมอบหมายทรัพยากรจะเท่ากัน เมื่อมีการรวมในระดับ **เดือน** แต่จะไม่ตรงกันในระดับที่ต่ำกว่า</span><span class="sxs-lookup"><span data-stu-id="b8445-140">Therefore, you can see that, even though the resource's bookings and assignments are equal when they are aggregated at the **Month** level, they don't match at lower levels.</span></span>
