## <a name="set-up-bookable-resource-as-a-pricing-dimension"></a><span data-ttu-id="10201-168">ตั้งค่าทรัพยากรที่สามารถจองได้เป็นมิติการกำหนดราคา</span><span class="sxs-lookup"><span data-stu-id="10201-168">Set up bookable resource as a pricing dimension</span></span>

1. <span data-ttu-id="10201-169">ในเว็บอินเทอร์เฟซ ไปที่ **Project Service** > **การตั้งค่า** > **พารามิเตอร์**</span><span class="sxs-lookup"><span data-stu-id="10201-169">In the web interface, go to **Project Service** > **Settings** > **Parameters**.</span></span> <span data-ttu-id="10201-170">บนหน้า **พารามิเตอร์** บนแท็บ **มิติการกำหนดราคาตามจำนวนเงิน** ให้สังเกตว่ากริดบนแท็บแสดงเรกคอร์ดในเอนทิตีมิติการกำหนดราคา</span><span class="sxs-lookup"><span data-stu-id="10201-170">On the **Parameter** page, on the **Amount-Based Pricing Dimensions** tab, notice that the grid on the tab shows the records in the pricing dimensions entity.</span></span> 
2. <span data-ttu-id="10201-171">เพิ่ม **ทรัพยากรที่สามารถจองได้** ไปยังรายการของมิติการกำหนดราคาเป็น **msdyn_bookableresource**</span><span class="sxs-lookup"><span data-stu-id="10201-171">Add **Bookable Resource** to this list of pricing dimensions as **msdyn_bookableresource**.</span></span> 
3. <span data-ttu-id="10201-172">ระบุบริบทที่ทรัพยากรที่จองได้ทำงานเป็นมิติการกำหนดราคาและตั้งค่ามูลค่า **ใช้กับต้นทุน** และ **ใช้กับการขาย**</span><span class="sxs-lookup"><span data-stu-id="10201-172">Indicate the context in which the bookable resource works as a pricing dimension and set the **Applicable to cost** and **Applicable to sales** values.</span></span>
4. <span data-ttu-id="10201-173">ในฟิลด์ **ชนิดมิติ** ให้เลือก **ตามจำนวนเงิน**</span><span class="sxs-lookup"><span data-stu-id="10201-173">In the **Dimension Type** field, select **Amount-based**.</span></span> 
5. <span data-ttu-id="10201-174">เลือกต้นทุนและระดับความสำคัญของการขายสำหรับทรัพยากรที่สามารถจองได้</span><span class="sxs-lookup"><span data-stu-id="10201-174">Select the cost and sales priority for the bookable resource.</span></span> <span data-ttu-id="10201-175">โดยทั่วไปเมื่อรวมเป็นมิติการกำหนดราคา ทรัพยากรที่จองได้จะมีลำดับความสำคัญสูงสุด ดังนั้นการตั้งค่าเป็น **1** หรือ **0** ขึ้นอยู่กับวิธีที่คุณนับระดับความสำคัญ จะทำให้แน่ใจถึงลักษณะการทำงานนั้น</span><span class="sxs-lookup"><span data-stu-id="10201-175">Typically, when included as a pricing dimension, a bookable resource has the highest priority so setting this to **1** (or **0** depending on how you count the priority) would ensure that behavior.</span></span>
