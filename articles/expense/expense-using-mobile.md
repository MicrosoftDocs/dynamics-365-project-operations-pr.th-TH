# <a name="mobile-expense-app"></a>แอปค่าใช้จ่ายสำหรับอุปกรณ์เคลื่อนที่

_**นำไปใช้กับ:** Project Operations สำหรับสถานการณ์ตามทรัพยากร/ไม่ได้เก็บในคลัง การปรับใช้งานแบบ Lite - จัดการกับการออกใบแจ้งหนี้ชั่วคราว_

บทความนี้ให้ข้อมูลเกี่ยวกับพื้นที่ทำงานแบบเคลื่อนที่ของ **การจัดการค่าใช้จ่าย** พื้นที่ทำงานนี้ช่วยให้ผู้ใช้สามารถบันทึกและอัปโหลดใบเสร็จรับเงิน เพื่อให้พวกเขาสามารถแนบใบเสร็จรับเงินกับรายงานค่าใช้จ่ายในภายหลัง ผู้ใช้ยังสามารถสร้างรายการค่าใช้จ่ายได้อย่างรวดเร็วโดยใช้ใบเสร็จรับเงินที่แนบมา และสร้างและจัดการรายงานค่าใช้จ่ายของพวกเขา นอกจากนี้ ผู้อนุมัติสามารถใช้พื้นที่ทำงานสำหรับอุปกรณ์เคลื่อนที่ของ **การจัดการค่าใช้จ่าย** เพื่อดูรายงานค่าใช้จ่ายที่กำหนดให้และอนุมัติหรือปฏิเสธรายงานค่าใช้จ่ายเหล่านั้น

พื้นที่ทำงานสำหรับอุปกรณ์เคลื่อนที่นี้มีวัตถุประสงค์เพื่อใช้กับแอปสำหรับอุปกรณ์เคลื่อนที่ Dynamics 365 Unified Ops

หลายองค์กรกำหนดให้แนบสำเนาใบเสร็จรับเงินในรายงานค่าใช้จ่ายที่เกี่ยวข้องกับการเดินทางหรือเกี่ยวกับธุรกิจที่พนักงานยื่นเพื่อขอรับเงินคืน พื้นที่ทำงานสำหรับอุปกรณ์เคลื่อนที่ของ **การจัดการค่าใช้จ่าย** ช่วยให้ผู้ใช้สามารถสร้างรายการค่าใช้จ่ายใหม่บนอุปกรณ์เคลื่อนที่ที่ต้องการได้อย่างรวดเร็วโดยใช้รูปถ่ายใบเสร็จรับเงินที่แนบมา หรือผู้ใช้สามารถถ่ายภาพใบเสร็จรับเงินแล้วแนบกับรายงานค่าใช้จ่ายในภายหลัง พนักงานยังสามารถสร้างและจัดการรายงานค่าใช้จ่ายของพวกเขา จากนั้นส่งเพื่อขออนุมัติและการชำระเงินคืนโดยใช้อุปกรณ์เคลื่อนที่ของพวกเขา

โดยเฉพาะอย่างยิ่ง พื้นที่ทำงานสำหรับอุปกรณ์เคลื่อนที่ของ **การจัดการค่าใช้จ่าย** ช่วยให้ผู้ใช้สามารถทำงานเหล่านี้ได้:

- ถ่ายรูปใบเสร็จรับเงิน อัปโหลดรูปถ่ายใบเสร็จและแนบไปกับรายงานค่าใช้จ่ายในภายหลัง
- อัปโหลดไฟล์เป็นใบเสร็จรับเงินที่บันทึกไว้ จากนั้นคุณสามารถแนบไฟล์นั้นกับรายงานค่าใช้จ่ายได้ในภายหลัง
- สร้างรายการค่าใช้จ่ายใหม่โดยใช้ใบเสร็จรับเงินที่แนบมา จากนั้นคุณสามารถเพิ่มการแสดงรายการลงในรายงานค่าใช้จ่ายในภายหลัง และส่งเพื่อขออนุมัติและการขอเงินคืน

นอกจากนี้ คุณยังสามารถใช้คุณลักษณะเหล่านี้:

- สร้างรายงานค่าใช้จ่ายใหม่
- แนบธุรกรรมบัตรเครดิตและค่าใช้จ่ายอื่นๆ ที่สร้างไว้ก่อนหน้ากับรายงานค่าใช้จ่าย
- สร้างค่าใช้จ่ายใหม่สำหรับรายงานค่าใช้จ่าย
- แนบใบเสร็จรับเงินกับค่าใช้จ่ายสำหรับรายงานค่าใช้จ่าย ไม่ว่าจะโดยการถ่ายภาพใบเสร็จรับเงินหรืออัปโหลดไฟล์เป็นใบเสร็จรับเงินที่บันทึกไว้
- เพิ่มรายชื่อแขกในค่าใช้จ่าย ทั้งนี้ขึ้นอยู่กับนโยบายค่าใช้จ่ายของบริษัท
- ลงรายการค่าใช้จ่าย ขึ้นอยู่กับนโยบายค่าใช้จ่ายของบริษัท
- ส่งรายงานค่าใช้จ่ายเพื่อขอการอนุมัติและการขอเงินคืน
- อนุมัติหรือปฏิเสธรายงานค่าใช้จ่ายที่คุณเป็นผู้อนุมัติที่ได้รับมอบหมาย

## <a name="prerequisites-if-you-use-dynamics-365-finance"></a>ข้อกำหนดเบื้องต้นหากคุณใช้ Dynamics 365 Finance

ถ้า Finance ได้รับการปรับใช้สำหรับองค์กรของคุณ ผู้ดูแลระบบต้องเผยแพร่พื้นที่ทำงานสำหรับอุปกรณ์เคลื่อนที่ **การจัดการค่าใช้จ่าย** 

## <a name="download-and-install-the-dynamics-365-unified-ops-mobile-app"></a>ดาวน์โหลดและติดตั้งแอปสำหรับอุปกรณ์เคลื่อนที่ Dynamics 365 Unified Ops
ดาวน์โหลดและติดตั้งแอปสำหรับอุปกรณ์เคลื่อนที่ Dynamics 365 Unified Ops:

- [สำหรับโทรศัพท์ Android](https://go.microsoft.com/fwlink/?linkid=850662)
- [สำหรับ iPhone](https://go.microsoft.com/fwlink/?linkid=850663)

## <a name="sign-in-to-the-mobile-app"></a>ลงชื่อเข้าใช้แอปสำหรับอุปกรณ์เคลื่อนที่
1. เริ่มต้นแอปบนอุปกรณ์เคลื่อนที่ของคุณ
2. ป้อน URL ของ Dynamics 365
4. ในครั้งแรกที่คุณลงชื่อเข้าใช้ คุณจะได้รับแจ้งให้ใส่ชื่อผู้ใช้และรหัสผ่านของคุณ ป้อนข้อมูลประจำตัวของคุณ
5. หลังจากที่คุณลงชื่อเข้าใช้ พื้นที่ทำงานที่พร้อมใช้งานสำหรับบริษัทของคุณจะปรากฏขึ้น หากผู้ดูแลระบบของคุณเผยแพร่พื้นที่ทำงานใหม่ในภายหลัง คุณจะต้องรีเฟรชรายการของพื้นที่ทำงานสำหรับอุปกรณ์เคลื่อนที่

## <a name="capture-a-receipt-by-using-the-expense-management-mobile-workspace"></a>บันทึกใบเสร็จรับเงินโดยใช้สำหรับพื้นที่ทำงานสำหรับอุปกรณ์เคลื่อนที่การจัดการค่าใช้จ่าย

1. บนอุปกรณ์เคลื่อนที่ของคุณ เปิดพื้นที่ทำงาน **การจัดการค่าใช้จ่าย**
2. เลือก **บันทึกใบเสร็จรับเงิน**
3. เลือก **ถ่ายภาพ** หรือ **เลือกรูปภาพ**
4. ทำตามหนึ่งในขั้นตอนเหล่านี้:

    - ถ้าคุณเลือก **ถ่ายภาพ** ให้ทำตามขั้นตอนเหล่านี้:

        1. คุณถูกนำไปที่กล้องบนอุปกรณ์เคลื่อนที่ของคุณ เพื่อให้คุณสามารถถ่ายภาพใบเสร็จรับเงินได้ 
        2. เมื่อคุณถ่ายภาพเสร็จแล้ว ให้เลือก **ตกลง** เพื่อยอมรับภาพถ่าย
        3. ไม่บังคับ: ป้อนชื่อสำหรับภาพถ่ายและป้อนบันทึกต่างๆ

    - ถ้าคุณเลือก **เลือกรูปภาพ** ให้ทำตามขั้นตอนเหล่านี้:

        1. เลือกรูปภาพในรายการ
        2. ไม่บังคับ: ป้อนชื่อสำหรับรูปภาพและป้อนบันทึกต่างๆ

5. เลือก **สำเร็จ**

## <a name="quickly-enter-expenses-by-using-the-expense-management-mobile-workspace"></a>ป้อนค่าใช้จ่ายได้อย่างรวดเร็วโดยใช้สำหรับพื้นที่ทำงานสำหรับอุปกรณ์เคลื่อนที่การจัดการค่าใช้จ่าย

1. บนอุปกรณ์เคลื่อนที่ของคุณ เปิดพื้นที่ทำงาน **การจัดการค่าใช้จ่าย**
2. เลือก **รายการค่าใช้จ่ายด่วน**
3. เลือกประเภทค่าใช้จ่าย คุณจะเห็นรายการประเภทค่าใช้จ่ายที่โหลดลงในแอปของคุณสำหรับการใช้งานแบบออฟไลน์ โดยค่าเริ่มต้น มีการโหลด 50 รายการ แต่นักพัฒนาสามารถเปลี่ยนจำนวนนี้ได้ สำหรับข้อมูลเพิ่มเติม นักพัฒนาควรเห็น [แพลตฟอร์ม Mobile](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-getting-started) หากประเภทของคุณไม่อยู่ในรายการ ให้เลือก **ค้นหา** เพื่อทำการค้นหาออนไลน์ ค้นหาตามประเภทค่าใช้จ่าย หรือเปลี่ยนไปค้นหาตามชนิดค่าใช้จ่าย
4. ป้อนวันที่ของธุรกรรมค่าใช้จ่าย
5. ไม่บังคับ: ป้อนผู้จำหน่ายสำหรับค่าใช้จ่าย
6. ป้อนจำนวนเงินของค่าใช้จ่าย
7. เลือกสกุลเงินของค่าใช้จ่าย คุณจะเห็นรายการของรหัสสกุลเงินที่โหลดลงในแอปของคุณสำหรับการใช้งานแบบออฟไลน์ โดยค่าเริ่มต้น มีการโหลดสกุลเงิน 400 รายการ แต่นักพัฒนาสามารถเปลี่ยนจำนวนนี้ได้ สำหรับข้อมูลเพิ่มเติม นักพัฒนาควรเห็น [แพลตฟอร์ม Mobile](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-getting-started) หากสกุลเงินของคุณไม่อยู่ในรายการ ให้เลือก **ค้นหา** เพื่อทำการค้นหาออนไลน์ ค้นหาตามสกุลเงิน หรือเปลี่ยนไปค้นหาตามชื่อ
8. เลือก **ถ่ายภาพ** หรือ **เลือกรูปภาพ**
9. ทำตามหนึ่งในขั้นตอนเหล่านี้:

    - ถ้าคุณเลือก **ถ่ายภาพ** คุณถูกนำไปที่กล้องบนอุปกรณ์เคลื่อนที่ของคุณ เพื่อให้คุณสามารถถ่ายภาพใบเสร็จรับเงินได้ เมื่อคุณถ่ายภาพเสร็จแล้ว ให้เลือก **ตกลง** เพื่อยอมรับภาพถ่าย
    - ถ้าคุณเลือก **เลือกรูปภาพ** ให้เลือกรูปภาพในรายการ

10. เลือก **สำเร็จ**

## <a name="approve-an-expense-report-by-using-the-expense-management-mobile-workspace"></a>อนุมัติรายงานค่าใช้จ่ายโดยใช้พื้นที่ทำงานการจัดการค่าใช้จ่ายบนอุปกรณ์เคลื่อนที่

1. บนอุปกรณ์เคลื่อนที่ของคุณ เปิดพื้นที่ทำงาน **การจัดการค่าใช้จ่าย**
2. **การอนุมัติค่าใช้จ่าย** แสดงจำนวนของรายงานค่าใช้จ่ายที่กำหนดให้คุณอนุมัติ จำนวนจะมีการปรับปรุงทุกๆ 30 นาทีโดยประมาณ เลือก **การอนุมัติค่าใช้จ่าย**

    โดยจะมีการแสดงรายการของรายงานค่าใช้จ่ายที่กำหนดให้คุณอนุมัติ

3. เลือกรายงานค่าใช้จ่ายเพื่อดูรายละเอียดค่าใช้จ่าย
4. เลือกค่าใช้จ่ายเพื่อดูรายละเอียดค่าใช้จ่าย ข้อมูลที่แสดงสำหรับค่าใช้จ่ายประกอบด้วยใบเสร็จรับเงิน แขก และรายละเอียดการแสดงรายการ
5. กลับไปที่เพจ **รายงานค่าใช้จ่าย** เลือกเพื่ออนุมัติหรือปฏิเสธรายงานค่าใช้จ่าย
6. ป้อนข้อคิดเห็นสำหรับการดำเนินการอนุมัติ
7. เลือก **สำเร็จ**

## <a name="create-a-new-expense-report-and-submit-it-for-approval-by-using-the-expense-management-mobile-workspace"></a>สร้างรายงานค่าใช้จ่ายใหม่และส่งเพื่อขอการอนุมัติโดยใช้พื้นที่ทำงานการจัดการค่าใช้จ่ายบนอุปกรณ์เคลื่อนที่

1. บนอุปกรณ์เคลื่อนที่ของคุณ เปิดพื้นที่ทำงาน **การจัดการค่าใช้จ่าย**
2. เลือก **รายการค่าใช้จ่าย**
3. เลือก **รายงานใหม่** ห รือเลือกรายงานค่าใช้จ่ายที่มีอยู่ในรายการ
4. สำหรับรายงานค่าใช้จ่ายใหม่ ให้ป้อนวัตถุประสงค์และข้อมูลเพิ่มเติมที่มีอยู่ ข้อมูลนี้แตกต่างกัน ขึ้นอยู่กับวิธีที่กำหนดค่าการจัดการค่าใช้จ่ายสำหรับบริษัทของคุณ
5. เลือก **สำเร็จ**
6. หากต้องการเพิ่มค่าใช้จ่ายที่มีอยู่ เช่น ธุรกรรมบัตรเครดิตในรายงานค่าใช้จ่าย ให้เลือก **แนบ**
7. เลือกค่าใช้จ่ายในรายการอย่างน้อยหนึ่งรายการ
8. เลือก **สำเร็จ**
9. หากต้องการเพิ่มค่าใช้จ่ายใหม่ในรายงานค่าใช้จ่าย ให้เลือก **ค่าใช้จ่ายใหม่**
10. เลือกประเภทสำหรับค่าใช้จ่าย คุณจะเห็นรายการประเภทค่าใช้จ่ายที่โหลดลงในแอปของคุณสำหรับการใช้งานแบบออฟไลน์ โดยค่าเริ่มต้น มีการโหลด 50 รายการ แต่นักพัฒนาสามารถเปลี่ยนจำนวนนี้ได้ สำหรับข้อมูลเพิ่มเติม นักพัฒนาควรเห็น [แพลตฟอร์ม Mobile](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-getting-started) หากประเภทของคุณไม่อยู่ในรายการ ให้เลือก **ค้นหา** เพื่อทำการค้นหาออนไลน์ ค้นหาตามประเภทค่าใช้จ่าย หรือเปลี่ยนไปค้นหาตามชนิดค่าใช้จ่าย
11. ไม่บังคับ: ป้อนผู้จำหน่ายสำหรับค่าใช้จ่าย
12. ป้อนวันที่ของธุรกรรมค่าใช้จ่าย
13. ป้อนจำนวนเงินของค่าใช้จ่าย
14. เลือกสกุลเงินของค่าใช้จ่าย คุณจะเห็นรายการของรหัสสกุลเงินที่โหลดลงในแอปของคุณสำหรับการใช้งานแบบออฟไลน์ โดยค่าเริ่มต้น มีการโหลดสกุลเงิน 400 รายการ แต่นักพัฒนาสามารถเปลี่ยนจำนวนนี้ได้ สำหรับข้อมูลเพิ่มเติม นักพัฒนาควรเห็น [แพลตฟอร์ม Mobile](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-getting-started) หากสกุลเงินของคุณไม่อยู่ในรายการ ให้เลือก **ค้นหา** เพื่อทำการค้นหาออนไลน์ ค้นหาตามสกุลเงิน หรือเปลี่ยนไปค้นหาตามชื่อ
15. เลือก **สำเร็จ**
16. หากต้องการเพิ่มรายละเอียดค่าใช้จ่าย ให้เลือก **เพิ่มรายละเอียดเพิ่มเติม** ฟิลด์ที่พร้อมใช้งานขึ้นอยู่กับการกำหนดค่าการจัดการค่าใช้จ่ายสำหรับบริษัทของคุณ
17. หากนโยบายของบริษัทต้องการใบเสร็จรับเงินสำหรับค่าใช้จ่าย ให้เลือก **ใบเสร็จรับเงิน** แล้วทำตามขั้นตอนเหล่านี้:

    1. เลือก **บันทึกใบเสร็จรับเงิน** หรือ **แนบใบเสร็จรับเงิน**
    2. ทำตามหนึ่งในขั้นตอนเหล่านี้:

        - ถ้าคุณเลือก **บันทึกใบเสร็จรับเงิน** ให้ทำตามขั้นตอนเหล่านี้:

            1. เลือก **ถ่ายภาพ** หรือ **เลือกรูปภาพ**
            2. ทำตามหนึ่งในขั้นตอนเหล่านี้:

                - ถ้าคุณเลือก **ถ่ายภาพ** ให้ทำตามขั้นตอนเหล่านี้:

                    1. คุณถูกนำไปที่กล้องบนอุปกรณ์เคลื่อนที่ของคุณ เพื่อให้คุณสามารถถ่ายภาพใบเสร็จรับเงินได้ เมื่อคุณถ่ายภาพเสร็จแล้ว ให้เลือก **ตกลง** เพื่อยอมรับภาพถ่าย
                    2. ไม่บังคับ: ป้อนชื่อสำหรับภาพถ่ายและป้อนบันทึกต่างๆ

                - ถ้าคุณเลือก **เลือกรูปภาพ** ให้ทำตามขั้นตอนเหล่านี้:

                    1. เลือกรูปภาพในรายการ
                    2. ไม่บังคับ: ป้อนชื่อสำหรับรูปภาพและป้อนบันทึกต่างๆ

            3. เลือก **สำเร็จ**

        - ถ้าคุณเลือก **แนบใบเสร็จรับเงิน** ให้ทำตามขั้นตอนเหล่านี้:

            1. เลือกรูปภาพในรายการอย่างน้อยหนึ่งรายการ
            2. เลือก **สำเร็จ**

    3. เลือกปุ่ม **ย้อนกลับ** เพื่อกลับไปที่รายละเอียดค่าใช้จ่าย

18. หากนโยบายของบริษัทกำหนดแขกสำหรับค่าใช้จ่าย ให้เลือก **แขก** แล้วทำตามขั้นตอนเหล่านี้:

    1. เลือก **แขก**, **แขกก่อนหน้า** หรือ **เพื่อนร่วมงาน**
    2. ทำตามหนึ่งในขั้นตอนเหล่านี้:

        - ถ้าคุณเลือก **แขก** ให้ทำตามขั้นตอนเหล่านี้:

            1. ใส่ชื่อของแขก
            2. ไม่บังคับ: ป้อนองค์กรและ/หรือประเทศของแขก
            3. ไม่บังคับ: ใส่ตำแหน่งของแขก
            4. เลือก **สำเร็จ**

        - ถ้าคุณเลือก **แขกก่อนหน้า** ให้ทำตามขั้นตอนเหล่านี้:

            1. เลือกแขกในรายการอย่างน้อยหนึ่งรายการ คุณจะเห็นรายชื่อของแขกก่อนหน้าที่คุณได้เพิ่มไว้ในรายงานค่าใช้จ่ายก่อนหน้าที่โหลดไว้ในแอปของคุณเพื่อใช้งานแบบออฟไลน์ โดยค่าเริ่มต้น มีการโหลด 50 รายการ แต่นักพัฒนาสามารถเปลี่ยนจำนวนนี้ได้ สำหรับข้อมูลเพิ่มเติม นักพัฒนาควรเห็น [แพลตฟอร์ม Mobile](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-getting-started) หากแขกก่อนหน้าของคุณไม่อยู่ในรายการ ให้เลือก **ค้นหา** เพื่อทำการค้นหาออนไลน์ ค้นหาตามชื่อ หรือเปลี่ยนไปค้นหาตามองค์กร ประเทศ หรือตำแหน่ง
            2. เลือก **สำเร็จ**

        - ถ้าคุณเลือก **เพื่อนร่วมงาน** ให้ทำตามขั้นตอนเหล่านี้:

            1. เลือกเพื่อนร่วมงานในรายการอย่างน้อยหนึ่งราย คุณจะเห็นรายชื่อของเพื่อนร่วมงานที่โหลดลงในแอปของคุณสำหรับการใช้งานแบบออฟไลน์ โดยค่าเริ่มต้น มีการโหลด 50 รายการ แต่นักพัฒนาสามารถเปลี่ยนจำนวนนี้ได้ สำหรับข้อมูลเพิ่มเติม นักพัฒนาควรเห็น [แพลตฟอร์ม Mobile](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-getting-started) หากเพื่อนร่วมงานของคุณไม่อยู่ในรายการ ให้เลือก **ค้นหา** เพื่อทำการค้นหาออนไลน์ ค้นหาตามชื่อ หรือเปลี่ยนไปค้นหาตามบริษัทหรือตำแหน่ง
            2. เลือก **สำเร็จ**

    3. เลือกปุ่ม **ย้อนกลับ** เพื่อกลับไปที่รายละเอียดค่าใช้จ่าย

19. หากนโยบายของบริษัทต้องการให้แสดงรายการค่าใช้จ่าย ให้เลือก **แสดงรายการ** แล้วทำตามขั้นตอนเหล่านี้:

    1. เลือกวันแรกที่จะแสดงรายการ
    2. เลือก **เพิ่มการแสดงรายการ**
    3. เลือกประเภทย่อยสำหรับการแสดงรายการค่าใช้จ่าย คุณจะเห็นรายการประเภทย่อยของค่าใช้จ่ายที่โหลดลงในแอปของคุณสำหรับการใช้งานแบบออฟไลน์ โดยค่าเริ่มต้น มีการโหลด 50 รายการ แต่นักพัฒนาสามารถเปลี่ยนจำนวนนี้ได้ สำหรับข้อมูลเพิ่มเติม นักพัฒนาควรเห็น [แพลตฟอร์ม Mobile](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-getting-started) หากประเภทย่อยของคุณไม่อยู่ในรายการ ให้เลือก **ค้นหา** เพื่อทำการค้นหาออนไลน์ ค้นหาตามชื่อประเภทย่อยของค่าใช้จ่าย
    4. ป้อนจำนวนเงินของธุรกรรมสำหรับการแสดงรายการ
    5. แก้ไขวันที่ของธุรกรรม หากจำเป็น
    6. เลือก **สำเร็จ**
    7. ทำซ้ำขั้นตอนก่อนหน้าจนกว่าคุณจะเพิ่มการแสดงรายการทั้งหมดสำหรับวันที่ที่เลือกเสร็จ
    8. สำหรับวันเพิ่มเติม คุณสามารถเลือก **คัดลอกไปยังวันถัดไป** เพื่อคัดลอกการแสดงรายการไปยังวันถัดไป อีกวิธีหนึ่งคือ คุณสามารถเลือกวันที่ที่จะแสดงรายการ จากนั้นเพิ่มการแสดงรายการตามที่คุณทำในวันแรก
    9. หลังจากคุณแสดงรายการค่าใช้จ่ายเสร็จแล้ว ให้เลือกปุ่ม **ย้อนกลับ** เพื่อกลับไปที่รายละเอียดค่าใช้จ่าย

20. เลือกปุ่ม **ย้อนกลับ** เพื่อกลับไปที่เพจ **รายงานค่าใช้จ่าย**
21. ทำซ้ำขั้นตอนก่อนหน้าจนกระทั่งคุณเพิ่มค่าใช้จ่ายทั้งหมดเสร็จสิ้น
22. เลือก **ส่ง**
23. ป้อนข้อคิดเห็นสำหรับผู้อนุมัติ
24. เลือก **สำเร็จ**

## <a name="frequently-asked-questions"></a>คำถามที่ถามบ่อย

### <a name="why-doesnt-the-expense-mobile-app-enter-the-payment-method-by-default"></a>เหตุใดแอปค่าใช้จ่ายสำหรับอุปกรณ์เคลื่อนที่จึงไม่ป้อนวิธีการชำระเงินโดยค่าเริ่มต้น

องค์กรสามารถปรับแต่งการตั้งค่า **วิธีการชำระเงินเริ่มต้น** สำหรับค่าใช้จ่ายแต่ละประเภทตามที่สร้างขึ้นได้ นอกจากนี้ เมื่อคุณตั้งค่าวิธีการชำระเงิน คุณสามารถตั้งค่าฟิลด์ **วิธีการชำระเงินเริ่มต้น** เป็น **นำเข้าเท่านั้น** ได้

เมื่อเปิดใช้งาน **นำเข้าเท่านั้น** สำหรับวิธีการชำระเงิน จะมีการป้อนวิธีการชำระเงินโดยค่าเริ่มต้น ประเภทค่าใช้จ่ายจะเว้นว่างเมื่อมีการตั้งค่าวิธีการชำระเงินนี้ ลักษณะการทำงานนี้สอดคล้องกันทั้งในการใช้งานบนเว็บและการใช้งานบนอุปกรณ์เคลื่อนที่
    
เมื่อไม่ได้เปิดใช้งาน **นำเข้าเท่านั้น** สำหรับวิธีการชำระเงิน ค่าที่ตั้งไว้จะถูกป้อนตามค่าเริ่มต้นสำหรับประเภทค่าใช้จ่ายที่มีการตั้งค่าวิธีการชำระเงินนี้ อย่างไรก็ตาม มีปัญหาที่ทราบอยู่แล้วเมื่อไม่ได้ป้อนค่าเริ่มต้นในแอปค่าใช้จ่ายสำหรับอุปกรณ์เคลื่อนที่ เมื่อต้องการแก้ไขปัญหานี้ ให้เลือกวิธีการชำระเงินด้วยตนเองก่อนที่คุณจะบันทึกรายงานค่าใช้จ่าย 

### <a name="why-cant-i-add-or-edit-financial-dimensions-in-the-expense-mobile-app"></a>เหตุใดฉันจึงไม่สามารถเพิ่มหรือแก้ไขมิติทางการเงินในแอปค่าใช้จ่ายสำหรับอุปกรณ์เคลื่อนที่

ไม่รองรับการป้อนมิติและการกระจาย เมื่อต้องการหลีกเลี่ยงข้อจำกัดนี้ คุณสามารถตั้งค่าฟิลด์เหล่านี้ตามค่าเริ่มต้นในแอปสำหรับอุปกรณ์เคลื่อนที่โดยการตั้งค่ามิติทางการเงินเริ่มต้นต่อโครงการหรือพนักงาน

### <a name="why-do-i-sometimes-see-a-synchronization-error-in-the-expense-mobile-app"></a>เหตุใดบางครั้งฉันจึงเห็นข้อผิดพลาดในการซิงโครไนซ์ในแอปค่าใช้จ่ายสำหรับอุปกรณ์เคลื่อนที่

ถ้ารายการค่าใช้จ่ายไม่ตรงตามข้อกำหนดของนโยบาย และผู้ใช้ส่งรายงานค่าใช้จ่ายโดยไม่ระบุคำเตือนเกี่ยวกับนโยบาย ข้อมูลบนอุปกรณ์เคลื่อนที่จะไม่ถูกซิงค์กับเซิร์ฟเวอร์ และเกิดความล้มเหลวในการซิงโครไนซ์ รายงานค่าใช้จ่ายทั้งหมดที่ส่งหลังจากเกิดความล้มเหลวในการซิงโครไนซ์จะยังคงอยู่ในสถานะล้มเหลวและทำให้การซิงโครไนซ์ล้มเหลวมากขึ้น วิธีเดียวที่จะแก้ไขสถานการณ์นี้คือ การลบการแจ้งเตือนการซิงโครไนซ์ด้วยตนเอง ปัญหานี้ได้รับการแก้ไขแล้วโดยการหยุดการส่งรายงานค่าใช้จ่ายเมื่อไม่มีคำเตือนเกี่ยวกับนโยบายเพื่อหลีกเลี่ยงข้อผิดพลาดในการซิงโครไนซ์

### <a name="why-isnt-project-and-category-validation-correctly-reflected-in-the-expense-mobile-app"></a>เหตุใดการตรวจสอบความถูกต้องของโครงการและประเภทจึงแสดงไม่ถูกต้องในแอปค่าใช้จ่ายสำหรับอุปกรณ์เคลื่อนที่

ยังไม่รองรับการตรวจสอบความถูกต้องนี้ อย่างไรก็ตาม อาจมีการเพิ่มการรองรับในอนาคต 

### <a name="what-document-types-are-supported-in-the-expense-mobile-app"></a>แอปค่าใช้จ่ายสำหรับอุปกรณ์เคลื่อนที่รองรับเอกสารชนิดใดบ้าง

แอปค่าใช้จ่ายสำหรับอุปกรณ์เคลื่อนที่รองรับเฉพาะรูปภาพ ขณะนี้ยังไม่รองรับ PDF หรือเอกสารอื่นๆ
