# <a name="bookings-vs-assignments"></a>การจองและการมอบหมาย

_**นำไปใช้กับ:** Project Operations สำหรับสถานการณ์ตามทรัพยากร/วัสดุที่ไม่ได้เก็บในคลัง การปรับใช้งานแบบ Lite - จัดการกับการออกใบแจ้งหนี้ชั่วคราว_

การจองคือการจัดสรรที่ตายตัวและไม่ตายตัวของทรัพยากรให้กับโครงการ การจองแบบตายตัวใช้ความสามารถรองรับของทรัพยากร 

การมอบหมายคือการมอบหมายของทรัพยากรให้กับงานโครงการในหมายกำหนดการให้บริการโครงการ ทรัพยากรสามารถเป็นได้ทั้งทรัพยากรจริงหรือทั่วไป 

ตามหลักการแล้วสำหรับทรัพยากรจริง การจองและการมอบหมายควรยอมรับเพราะพวกเขาไม่แตกต่างกัน อย่างไรก็ตาม Microsoft Dynamics Project Operations ไม่บังคับใช้ข้อตกลงนี้ มุมมอง **การกระทบยอด** แสดงตำแหน่งผู้จัดการโครงการที่การจองและการมอบหมายทรัพยากรไม่เห็นด้วย
