# <a name="resource-reconciliation-overview"></a><span data-ttu-id="89e55-103">ภาพรวมการกระทบยอดทรัพยากร</span><span class="sxs-lookup"><span data-stu-id="89e55-103">Resource reconciliation overview</span></span>

<span data-ttu-id="89e55-104">_**นำไปใช้กับ:** Project Operations สำหรับสถานการณ์ตามทรัพยากร/วัสดุที่ไม่ได้เก็บในคลัง การปรับใช้งานแบบ Lite - จัดการกับการออกใบแจ้งหนี้ชั่วคราว_</span><span class="sxs-lookup"><span data-stu-id="89e55-104">_**Applies To:** Project Operations for resource/non-stocked based scenarios, Lite deployment - deal to proforma invoicing_</span></span>

<span data-ttu-id="89e55-105">สำหรับสมาชิกในทีมการจองและการมอบหมายถูกจับเป็นคู่กันอย่างหลวมๆ</span><span class="sxs-lookup"><span data-stu-id="89e55-105">For team members, bookings and assignments are loosely coupled.</span></span> <span data-ttu-id="89e55-106">กล่าวอีกนัยหนึ่ง ทรัพยากรสามารถมีการมอบหมายและไม่มีการจอง หรือทรัพยากรสามารถมีการจองและไม่มีการมอบหมาย</span><span class="sxs-lookup"><span data-stu-id="89e55-106">In other words, resources can have assignments and no bookings, or they can have bookings and no assignments.</span></span> <span data-ttu-id="89e55-107">ตามหลักการแล้ว การจองและการมอบหมายควรจะถูกจัดตำแหน่งเพื่อให้ทรัพยากรมีความมุ่งมั่นในความสามารถรองรับเพื่อที่จะปฏิบัติงานที่ได้รับมอบหมาย</span><span class="sxs-lookup"><span data-stu-id="89e55-107">Ideally, bookings and assignments should be aligned, so that resources have committed capacity to perform the task assignments.</span></span> <span data-ttu-id="89e55-108">อย่างไรก็ตามการจองอาจจะขึ้นอยู่กับความพร้อมใช้งานและการกำหนดเวลาที่อาจจะเปลี่ยนแปลงในขณะที่โครงการยังคงดำเนินต่อไป</span><span class="sxs-lookup"><span data-stu-id="89e55-108">However, the bookings might be based on availability, and task timings might change as the project continues.</span></span> <span data-ttu-id="89e55-109">ดังนั้นการจับเป็นคู่กันอย่างหลวมๆของการจองและงานมอบหมายทำให้มีความยืดหยุ่น</span><span class="sxs-lookup"><span data-stu-id="89e55-109">Therefore, the loose coupling of bookings and assignments provides flexibility.</span></span>

<span data-ttu-id="89e55-110">แท็บ **การกระทบยอด** บนหน้า **โครงการ** อนุญาตให้ผู้จัดการโครงการกระทบยอดการจองของสมาชิกในกลุ่มคนและการมอบหมายสำหรับกลุ่มคนของโครงการ</span><span class="sxs-lookup"><span data-stu-id="89e55-110">The **Reconciliation** tab on the **Projects** page lets project managers reconcile team members' bookings and their assignments for project teams.</span></span>

<span data-ttu-id="89e55-111">แท็บ **การกระทบยอด** แสดงการจองและการมอบหมายไปจนถึงระดับของการมอบหมายงานสำหรับสมาชิกในทีมแต่ละคน</span><span class="sxs-lookup"><span data-stu-id="89e55-111">The **Reconciliation** tab shows bookings and assignments down to the level of the individual task assignment for each team member.</span></span> <span data-ttu-id="89e55-112">มีการแสดงชั่วโมงในเซลล์ที่แสดงช่วงเวลาตั้งแต่เดือนไปจนถึงวัน</span><span class="sxs-lookup"><span data-stu-id="89e55-112">Hours are shown in cells that represent periods from months to days.</span></span> <span data-ttu-id="89e55-113">แท็บยังแสดงภาพรวมผลรวมสุทธิสำหรับโครงการพร้อมกับคอลัมน์ **ผลรวม**</span><span class="sxs-lookup"><span data-stu-id="89e55-113">The tab also shows an overall net total for the project, together with a **Total** column.</span></span>

<span data-ttu-id="89e55-114">สำหรับทุกทรัพยากร แท็บ **การกระทบยอด** จะคำนวณความแตกต่างระหว่างการจองของสมาชิกในกลุ่มคน และค่าสะสมของการมอบหมายงานของสมาชิกในกลุ่มคน</span><span class="sxs-lookup"><span data-stu-id="89e55-114">For each resource, the **Reconciliation** tab calculates the difference between the team member's bookings and a rollup of that team member's task assignments.</span></span> <span data-ttu-id="89e55-115">ตามหลักการแล้ว ความแตกต่างนี้ควรเป็น 0 (ศูนย์)</span><span class="sxs-lookup"><span data-stu-id="89e55-115">Ideally, this difference should be 0 (zero).</span></span> <span data-ttu-id="89e55-116">กล่าวอีกนัยหนึ่งคือไม่ควรจะมีความแตกต่างระหว่างการจองและงานมอบหมาย</span><span class="sxs-lookup"><span data-stu-id="89e55-116">In other words, there should be no difference between bookings and assignments.</span></span> <span data-ttu-id="89e55-117">ความแตกต่างของสีและแรเงาเพื่อดึงดูดให้ความสนใจกับสองเงื่อนไข:</span><span class="sxs-lookup"><span data-stu-id="89e55-117">Differences are colored and shaded to draw attention to two conditions:</span></span>

- <span data-ttu-id="89e55-118">**ความขาดแคลนการจอง** – ความขาดแคลนการจองเกิดขึ้นเมื่อทรัพยากรมีการมอบหมายมากกว่าการจอง</span><span class="sxs-lookup"><span data-stu-id="89e55-118">**Booking shortage** – A booking shortage occurs when a resource has more assignments than bookings.</span></span> <span data-ttu-id="89e55-119">เนื่องจากกำลังการผลิตนี้ไม่ได้ถูกจองไว้ ผู้จัดการโครงการอาจต้องการแก้ไขเงื่อนไขนี้โดยการขยายการจองของทรัพยากรเพื่อครอบคลุมการขาด</span><span class="sxs-lookup"><span data-stu-id="89e55-119">Because the capacity hasn't been reserved, the project manager might want to correct this condition by extending the resource's bookings to cover the deficit.</span></span>
- <span data-ttu-id="89e55-120">**การจองเกินจำนวน** การจองเกินจำนวนจะเกิดขึ้นเมื่อมีการจองทรัพยากรไปยังโครงการ แต่ยังไม่ได้กำหนดให้กับงาน</span><span class="sxs-lookup"><span data-stu-id="89e55-120">**Excess bookings** – Excess bookings occur when a resource has been booked to the project but hasn't been assigned to tasks.</span></span> <span data-ttu-id="89e55-121">เงื่อนไขนี้อาจยอมรับได้ในกรณีที่ทรัพยากรถูกจองไปยังโครงการ ก่อนที่จะมีการมอบหมายงาน</span><span class="sxs-lookup"><span data-stu-id="89e55-121">This condition might be acceptable in cases where the resource was booked to the project before task assignment occurred.</span></span> <span data-ttu-id="89e55-122">อย่างไรก็ตาม ในกรณีอื่นๆ ที่ไม่มีแผนที่จะมอบหมายทรัพยากรให้กับงาน ผู้จัดการโครงการควรพิจารณาการยกเลิกการจองของทรัพยากร</span><span class="sxs-lookup"><span data-stu-id="89e55-122">However, in other cases, where there is no plan to assign the resource to tasks, the project manager should consider canceling the resource's bookings.</span></span> <span data-ttu-id="89e55-123">ด้วยวิธีนี้ กำลังการผลิตสามารถใช้สำหรับโครงการอื่นได้</span><span class="sxs-lookup"><span data-stu-id="89e55-123">In that way, the capacity can be used for another project.</span></span>

<span data-ttu-id="89e55-124">ในบางกรณี เมื่อคุณดูเวลาในระดับที่สูงกว่าระดับวัน (ตัวอย่างเช่น ระดับเดือน) คุณอาจเห็นความแตกต่างสุทธิของศูนย์สำหรับทรัพยากร (กล่าวอีกนัยหนึ่ง การจองเท่ากับการมอบหมาย)</span><span class="sxs-lookup"><span data-stu-id="89e55-124">In some cases, when you view time at a higher level than the day level (for example, the month level), you might see a net difference of zero for a resource (in other words, bookings equal assignments).</span></span> <span data-ttu-id="89e55-125">อย่างไรก็ตามถ้าคุณดูเวลาในระดับสัปดาห์ คุณอาจเห็นว่ามีการมอบหมายของชั่วโมงเป็นศูนย์และการจองของ 40 ชั่วโมงในสัปดาห์แรกแต่การมอบหมายงานของ 40 ชั่วโมงและการจองเป็นศูนย์ชั่วโมงในสัปดาห์ที่สอง</span><span class="sxs-lookup"><span data-stu-id="89e55-125">However, if you view time at the week level, you might see that there are assignments of zero hours and bookings of 40 hours in the first week, but assignments of 40 hours and bookings of zero hours in the second week.</span></span> <span data-ttu-id="89e55-126">โดยรวมแล้วการจองและการมอบหมายงานจะกระทบยอดแต่จะแตกต่างจากหนึ่งสัปดาห์ต่อไป</span><span class="sxs-lookup"><span data-stu-id="89e55-126">Overall, the bookings and assignments are reconciled, but they differ from one week to the next.</span></span>

<span data-ttu-id="89e55-127">เมื่อคุณดูเวลาในระดับที่สูงกว่า เซลล์บนแท็บ **การกระทบยอด** มีตัวบ่งชี้เพื่อแจ้งให้คุณทราบว่ามีความแตกต่างในระดับที่ต่ำกว่า</span><span class="sxs-lookup"><span data-stu-id="89e55-127">When you view time at higher levels, cells on the **Reconciliation** tab have an indicator to inform you that there are differences at lower levels.</span></span> <span data-ttu-id="89e55-128">โดยการแตะสองครั้งหรือการคลิกสองครั้งในเซลล์ คุณสามารถขยายเพื่อดูความแตกต่างได้</span><span class="sxs-lookup"><span data-stu-id="89e55-128">By double-tapping or double-clicking in a cell, you can zoom in to view the difference.</span></span> <span data-ttu-id="89e55-129">จากนั้น คุณสามารถเลือกและกดค้างไว้ (หรือคลิกขวา) เพื่อซูมออก โดยการเลือกทรัพยากร แล้วจากนั้น ใช้ตัวควบคุม **ความแตกต่างถัดไป** บนแถบเครื่องมือกริด คุณสามารถไปที่ความแตกต่างถัดไประหว่างการจองและการมอบหมายงานสำหรับทรัพยากรนั้น</span><span class="sxs-lookup"><span data-stu-id="89e55-129">You can then select and hold (or right-click) to zoom out. By selecting a resource and then using the **Next difference** control on the grid toolbar, you can go to the next difference between bookings and assignments for that resource.</span></span> <span data-ttu-id="89e55-130">จากนั้นคุณสามารถใช้ตัวควบคุม **ความแตกต่างก่อนหน้านี้** เพื่อกลับไป</span><span class="sxs-lookup"><span data-stu-id="89e55-130">You can then use the **Previous difference** control to go back.</span></span> <span data-ttu-id="89e55-131">นอกจากนี้คุณยังสามารถปิดตัวบ่งชี้ความแตกต่างและแถบนำทางภายใต้ **การตั้งค่า**</span><span class="sxs-lookup"><span data-stu-id="89e55-131">You can also turn off the difference indicator and navigation behavior under **Settings**.</span></span>

<span data-ttu-id="89e55-132">ถ้าคุณมีการมอบหมายงานสำหรับทรัพยากร แต่ไม่มีการจอง เลือกการขาดแคลนการจองบนแท็บ **การกระทบยอด** ของหน้า **โครงการ** และจากนั้น ให้เลือก **ขยายการจอง**</span><span class="sxs-lookup"><span data-stu-id="89e55-132">If you have task assignments for a resource but no bookings, select the booking shortage on the **Reconciliation** tab of the **Projects** page, and then select **Extend Booking**.</span></span> <span data-ttu-id="89e55-133">กล่องโต้ตอบ **ขยายการจอง** ที่ปรากฏขึ้น จะแสดงการจองที่จำเป็นในการจัดการการขาดแคลนของทรัพยากร</span><span class="sxs-lookup"><span data-stu-id="89e55-133">The **Extend Booking** dialog box that appears shows the booking that is required to address the resource's shortage.</span></span> <span data-ttu-id="89e55-134">กล่องโต้ตอบนี้ยังแสดงการจองที่มีอยู่ของทรัพยากรในโครงการทั้งหมดหรือเอนทิตีที่จัดกำหนดการอื่นๆ</span><span class="sxs-lookup"><span data-stu-id="89e55-134">The dialog box also shows the resource's existing bookings across all projects or other schedulable entities.</span></span> <span data-ttu-id="89e55-135">ถ้าคุณเลือก **ตกลง** เพื่อสร้างการจองสำหรับทรัพยากรโดยไม่คำนึงถึงความพร้อมใช้งานของทรัพยากรนั้น คุณอาจทำให้เกิดการจองเกิน</span><span class="sxs-lookup"><span data-stu-id="89e55-135">If you select **OK** to create the booking for the resource, regardless of that resource's availability, you might cause overbooking.</span></span>

<span data-ttu-id="89e55-136">การจองที่สร้างผ่านการดำเนินการ **ขยายการจอง** เกี่ยวข้องกับความต้องการของโครงการหลัก</span><span class="sxs-lookup"><span data-stu-id="89e55-136">Bookings that are created through the **Extend Booking** action are associated with the primary project requirement.</span></span> <span data-ttu-id="89e55-137">เมื่อเริ่มต้นส่วนขยาย จะไม่สามารถกำหนดข้อกำหนดเฉพาะที่ต้องขยายได้ เนื่องจากทรัพยากรอาจเชื่อมโยงกับข้อกำหนดมากกว่าหนึ่งข้อสำหรับโครงการ</span><span class="sxs-lookup"><span data-stu-id="89e55-137">When an extension is initiated, the specific requirement that must be extended can't be determined, because the resource might be associated with more than one requirement for the project.</span></span>

<span data-ttu-id="89e55-138">จากนั้น ผู้จัดการโครงการหรือผู้จัดการทรัพยากรสามารถใช้บอร์ดกำหนดการเพื่อจัดการกับสถานการณ์ใดๆ ที่ทรัพยากรถูกจองเกินกว่ากำลังการผลิต</span><span class="sxs-lookup"><span data-stu-id="89e55-138">The project manager or resource manager can then use the Schedule board to manage any situations where a resource is overbooked beyond its capacity.</span></span>
