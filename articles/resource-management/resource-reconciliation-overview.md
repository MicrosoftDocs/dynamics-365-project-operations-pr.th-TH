# <a name="resource-reconciliation-overview"></a><span data-ttu-id="f1876-103">ภาพรวมการกระทบยอดทรัพยากร</span><span class="sxs-lookup"><span data-stu-id="f1876-103">Resource reconciliation overview</span></span>

<span data-ttu-id="f1876-104">_**นำไปใช้กับ:** Project Operations สำหรับสถานการณ์ที่อิงตามทรัพยากร/ที่ไม่ได้เก็บในสต็อก การปรับใช้ Lite - จัดการกับการออกใบแจ้งหนี้ชั่วคราว_</span><span class="sxs-lookup"><span data-stu-id="f1876-104">_**Applies To:** Project Operations for resource/non-stocked based scenarios, Lite deployment - deal to proforma invoicing_</span></span>

<span data-ttu-id="f1876-105">สำหรับสมาชิกในทีมการจองและการมอบหมายถูกจับเป็นคู่กันอย่างหลวมๆ</span><span class="sxs-lookup"><span data-stu-id="f1876-105">For team members, bookings and assignments are loosely coupled.</span></span> <span data-ttu-id="f1876-106">กล่าวอีกนัยหนึ่ง ทรัพยากรสามารถมีการมอบหมายได้แต่ไม่มีการจอง หรือทรัพยากรสามารถมีการจองได้แต่ไม่มีการมอบหมาย</span><span class="sxs-lookup"><span data-stu-id="f1876-106">In other words, resources can have assignments but no bookings, or they can have bookings but no assignments.</span></span> <span data-ttu-id="f1876-107">ตามหลักการแล้ว การจองและการมอบหมายควรจะถูกจัดตำแหน่งเพื่อให้ทรัพยากรมีความมุ่งมั่นในความสามารถรองรับเพื่อที่จะปฏิบัติงานที่ได้รับมอบหมาย</span><span class="sxs-lookup"><span data-stu-id="f1876-107">Ideally, bookings and assignments should be aligned, so that resources have committed capacity to perform the task assignments.</span></span> <span data-ttu-id="f1876-108">อย่างไรก็ตามการจองอาจจะขึ้นอยู่กับความพร้อมใช้งานและการกำหนดเวลาที่อาจจะเปลี่ยนแปลงในขณะที่โครงการยังคงดำเนินต่อไป</span><span class="sxs-lookup"><span data-stu-id="f1876-108">However, the bookings might be based on availability, and task timings might change as the project continues.</span></span> <span data-ttu-id="f1876-109">ดังนั้นการจับเป็นคู่กันอย่างหลวมๆของการจองและงานมอบหมายทำให้มีความยืดหยุ่น</span><span class="sxs-lookup"><span data-stu-id="f1876-109">Therefore, the loose coupling of bookings and assignments provides flexibility.</span></span>

<span data-ttu-id="f1876-110">แท็บ **การกระทบยอด** บนฟอร์ม **โครงการ** ให้ผู้จัดการโครงการกระทบยอดการจองของสมาชิกในทีมและงานมอบหมายสำหรับทีมโครงการ</span><span class="sxs-lookup"><span data-stu-id="f1876-110">The **Reconciliation** tab on the **Project** form lets project managers reconcile team members' bookings and their assignments for project teams.</span></span>

<span data-ttu-id="f1876-111">แท็บ **การกระทบยอด** ยังแสดงการจองและการมอบหมายไปจนถึงระดับของการมอบหมายงานสำหรับสมาชิกในทีมแต่ละคนด้วย</span><span class="sxs-lookup"><span data-stu-id="f1876-111">The **Reconciliation** tab also shows bookings and assignments down to the level of the individual task assignment for each team member.</span></span> <span data-ttu-id="f1876-112">โดยจะแสดงชั่วโมงในเซลล์ที่แสดงช่วงเวลาตั้งแต่เดือนไปจนถึงวัน</span><span class="sxs-lookup"><span data-stu-id="f1876-112">Hours are displayed in cells that represent time periods from months down to days.</span></span>

<span data-ttu-id="f1876-113">แท็บยังแสดงภาพรวมผลรวมสุทธิสำหรับโครงการพร้อมกับคอลัมน์ **ผลรวม**</span><span class="sxs-lookup"><span data-stu-id="f1876-113">The tab also shows an overall net total for the project, together with a **Total** column.</span></span>

<span data-ttu-id="f1876-114">สำหรับทุกทรัพยากร แท็บจะคำนวณความแตกต่างระหว่างการจองของสมาชิกในทีมและค่าสะสมของการกำหนดงานที่มอบหมายของสมาชิกในทีม</span><span class="sxs-lookup"><span data-stu-id="f1876-114">For each resource, the tab calculates the difference between the team member's bookings and a rollup of the team member's task assignments.</span></span> <span data-ttu-id="f1876-115">ตามหลักการแล้ว ความแตกต่างนี้ควรเป็น 0 (ศูนย์)</span><span class="sxs-lookup"><span data-stu-id="f1876-115">Ideally, this difference should be 0 (zero).</span></span> <span data-ttu-id="f1876-116">กล่าวอีกนัยหนึ่งคือไม่ควรจะมีความแตกต่างระหว่างการจองและงานมอบหมาย</span><span class="sxs-lookup"><span data-stu-id="f1876-116">In other words, there should be no difference between bookings and assignments.</span></span> <span data-ttu-id="f1876-117">ความแตกต่างของสีและแรเงาเพื่อดึงดูดให้ความสนใจกับสองเงื่อนไข:</span><span class="sxs-lookup"><span data-stu-id="f1876-117">Differences are colored and shaded to draw attention to two conditions:</span></span>

- <span data-ttu-id="f1876-118">**ความขาดแคลนการจอง** – ความขาดแคลนการจองเกิดขึ้นเมื่อทรัพยากรมีการมอบหมายมากกว่าการจอง</span><span class="sxs-lookup"><span data-stu-id="f1876-118">**Booking shortage** – A booking shortage occurs when a resource has more assignments than bookings.</span></span> <span data-ttu-id="f1876-119">เนื่องจากกำลังการผลิตนี้ไม่ได้รับการสงวนสิทธิ์ ผู้จัดการโครงการอาจต้องการแก้ไขเงื่อนไขนี้โดยการขยายการจองของทรัพยากรเพื่อครอบคลุมการขาดดุล</span><span class="sxs-lookup"><span data-stu-id="f1876-119">Because this capacity hasn't been reserved, a project manager might want to correct this condition by extending the resource's bookings to cover the deficit.</span></span>
- <span data-ttu-id="f1876-120">**การจองเกินจำนวน** การจองเกินจำนวนจะเกิดขึ้นเมื่อมีการจองทรัพยากรไปยังโครงการ แต่ยังไม่ได้กำหนดให้กับงาน</span><span class="sxs-lookup"><span data-stu-id="f1876-120">**Excess bookings** – Excess bookings occur when a resource has been booked to the project but hasn't been assigned to tasks.</span></span> <span data-ttu-id="f1876-121">เงื่อนไขนี้อาจเป็นที่ยอมรับในกรณีที่ทรัพยากรถูกจองไปยังโครงการก่อนที่จะมีการมอบหมายงาน</span><span class="sxs-lookup"><span data-stu-id="f1876-121">This condition might be acceptable in the cases where the resource was booked to the project before task assignment occurred.</span></span> <span data-ttu-id="f1876-122">อย่างไรก็ตามในกรณีอื่นๆ ทรัพยากรไม่ได้วางแผนเพื่อจะมอบหมายให้กับงาน</span><span class="sxs-lookup"><span data-stu-id="f1876-122">However, in other cases, the resource isn't planned to be assigned to tasks.</span></span> <span data-ttu-id="f1876-123">ในกรณีเหล่านี้ ผู้จัดการโครงการควรพิจารณายกเลิกการจองของทรัพยากร เพื่อให้สามารถใช้กำลังการผลิตสำหรับโครงการอื่น</span><span class="sxs-lookup"><span data-stu-id="f1876-123">In these cases, the project manager should consider canceling the resource's bookings, so that the capacity can be used for another project.</span></span>

<span data-ttu-id="f1876-124">ในบางกรณีเมื่อคุณดูเวลาในระดับที่สูงกว่าระดับวัน (ตัวอย่างเช่นระดับเดือน) คุณอาจเห็นความแตกต่างสุทธิของศูนย์สำหรับทรัพยากร (กล่าวอีกนัยหนึ่ง การจอง = การมอบหมาย)</span><span class="sxs-lookup"><span data-stu-id="f1876-124">In some cases, when you view time at a higher level than the day level (for example, the month level), you might see a net difference of zero for a resource (in other words, bookings = assignments).</span></span> <span data-ttu-id="f1876-125">อย่างไรก็ตามถ้าคุณดูเวลาในระดับสัปดาห์ คุณอาจเห็นว่ามีการมอบหมายของชั่วโมงเป็นศูนย์และการจองของ 40 ชั่วโมงในสัปดาห์แรกแต่การมอบหมายงานของ 40 ชั่วโมงและการจองเป็นศูนย์ชั่วโมงในสัปดาห์ที่สอง</span><span class="sxs-lookup"><span data-stu-id="f1876-125">However, if you view time at the week level, you might see that there are assignments of zero hours and bookings of 40 hours in the first week, but assignments of 40 hours and bookings of zero hours in the second week.</span></span> <span data-ttu-id="f1876-126">โดยรวมแล้วการจองและการมอบหมายงานจะกระทบยอดแต่จะแตกต่างจากหนึ่งสัปดาห์ต่อไป</span><span class="sxs-lookup"><span data-stu-id="f1876-126">Overall, the bookings and assignments are reconciled, but they differ from one week to the next.</span></span>

<span data-ttu-id="f1876-127">เมื่อคุณดูเวลาในระดับที่สูงกว่า เซลล์ในแท็บ **การกระทบยอด** มีตัวบ่งชี้เพื่อแจ้งให้คุณทราบว่ามีความแตกต่างในระดับที่ต่ำกว่า</span><span class="sxs-lookup"><span data-stu-id="f1876-127">When you view time at higher levels, cells in the **Reconciliation** tab have an indicator to inform you that there are differences at lower levels.</span></span> <span data-ttu-id="f1876-128">เมื่อคลิกสองครั้งในเซลล์ คุณสามารถขยายเพื่อดูความแตกต่างได้</span><span class="sxs-lookup"><span data-stu-id="f1876-128">By double-clicking in a cell, you can zoom in to view the difference.</span></span> <span data-ttu-id="f1876-129">จากนั้นคุณสามารถคลิกขวาเพื่อซูมออก โดยการเลือกทรัพยากรแล้วใช้ตัวควบคุม **ความแตกต่างถัดไป** บนแถบเครื่องมือกริด คุณสามารถไปที่ความแตกต่างถัดไประหว่างการจองและการมอบหมายงานสำหรับทรัพยากรนั้น</span><span class="sxs-lookup"><span data-stu-id="f1876-129">You can then right-click to zoom out. By selecting a resource and then using the **Next difference** control on the grid toolbar, you can go to the next difference between bookings and assignments for that resource.</span></span> <span data-ttu-id="f1876-130">จากนั้นคุณสามารถใช้ตัวควบคุม **ความแตกต่างก่อนหน้านี้** เพื่อกลับไป</span><span class="sxs-lookup"><span data-stu-id="f1876-130">You can then use the **Previous difference** control to go back.</span></span> <span data-ttu-id="f1876-131">นอกจากนี้คุณยังสามารถปิดตัวบ่งชี้ความแตกต่างและแถบนำทางภายใต้ **การตั้งค่า**</span><span class="sxs-lookup"><span data-stu-id="f1876-131">You can also turn off the difference indicator and navigation behavior under **Settings**.</span></span>


<span data-ttu-id="f1876-132">ถ้าคุณมีการมอบหมายงานสำหรับทรัพยากรแต่ไม่มีการจองบนหน้า **โครงการ** บนแท็บ **การกระทบยอด** ให้เลือกการขาดแคลนการจองแล้วเลือก **ขยายการจอง**</span><span class="sxs-lookup"><span data-stu-id="f1876-132">If you have task assignments for a resource but no bookings, on the **Projects** page, on the **Reconciliation** tab, select the booking shortage, and then select **Extend Booking**.</span></span> <span data-ttu-id="f1876-133">กล่องโต้ตอบ **การขยายการจอง** ปรากฏขึ้นและแสดงการจองที่จำเป็นเพื่อจัดการการขาดแคลนของทรัพยากร</span><span class="sxs-lookup"><span data-stu-id="f1876-133">The **Extend Booking** dialog box appears and shows the booking that is needed to address the resource's shortage.</span></span> <span data-ttu-id="f1876-134">นอกจากนี้ยังแสดงการจองที่มีอยู่ของทรัพยากรในโครงการทั้งหมดหรือเอนทิตีที่จัดกำหนดการอื่นๆ</span><span class="sxs-lookup"><span data-stu-id="f1876-134">It also shows the resource's existing bookings across all projects or other schedulable entities.</span></span> <span data-ttu-id="f1876-135">ถ้าคุณเลือก **ตกลง** เพื่อสร้างการจองสำหรับทรัพยากรโดยไม่คำนึงถึงความพร้อมใช้งานของทรัพยากรนั้น คุณอาจทำให้เกิดการจองเกิน</span><span class="sxs-lookup"><span data-stu-id="f1876-135">If you select **OK** to create the booking for the resource, regardless of that resource's availability, you might cause overbooking.</span></span>

<span data-ttu-id="f1876-136">ผู้จัดการโครงการหรือผู้จัดการทรัพยากรสามารถใช้ตารางหมายกำหนดการให้บริการเพื่อจัดการกับสถานการณ์ใดๆที่ทรัพยากรถูกจองเกินกว่ากำลังการผลิต</span><span class="sxs-lookup"><span data-stu-id="f1876-136">The project manager or resource manager can then use the Schedule Board to manage any situations where a resource is overbooked beyond its capacity.</span></span>
