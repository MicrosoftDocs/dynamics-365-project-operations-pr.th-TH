# <a name="manage-a-proforma-project-based-invoice"></a>จัดการใบแจ้งหนี้ตามโครงการชั่วคราว

_**นำไปใช้กับ:** Project Operations สำหรับสถานการณ์ตามทรัพยากร/วัสดุที่ไม่ได้เก็บในคลัง_

ใน Dynamics 365 Project Operations ใบแจ้งหนี้ชั่วคราวถูกสร้างขึ้นเพื่อเป็นส่วนขยายของใบแจ้งหนี้ใน Dynamics 365 Sales อย่างไรก็ตาม มีความแตกต่างหลายประการในกระบวนการออกใบแจ้งหนี้ระหว่าง Sales และ Project Operations เมื่อพูดถึงการออกใบแจ้งหนี้ ตัวอย่างเช่น ไม่สามารถสร้างใบแจ้งหนี้ใหม่จากหน้า **รายการใบแจ้งหนี้** ใน Project Operations แต่สามารถทำได้ใน Sales ความแตกต่างและส่วนขยายเหล่านี้มีไว้เพื่อรองรับกระบวนการออกใบแจ้งหนี้สำหรับโครงการที่แตกต่างจากใบแจ้งหนี้ทั่วไปสำหรับใบสั่งขาย

> [!IMPORTANT]
> เนื่องจากความแตกต่าง อย่าใช้ใบแจ้งหนี้ใน Sales และ Project Operations แทนกัน

## <a name="project-based-invoice-lines"></a>รายละเอียดการให้บริการตามใบแจ้งหนี้ตามโครงการ

ใน Project Operations จะมีรายการใบแจ้งหนี้หนึ่งรายการสำหรับทุกรายละเอียดการให้บริการตามสัญญาของโครงการเสมอ รายการใบแจ้งหนี้มีการสร้างแม้ว่าจะไม่มีอยู่จริงก็ตาม ข้อมูลต่อไปนี้มีอยู่ในรายการใบแจ้งหนี้ชั่วคราว

| ฟิลด์ | สถานที่ | รายละเอียด | 
| --- | --- | --- |
| **รหัสใบแจ้งหนี้** | แท็บ **ทั่วไป** | การอ้างอิงไปยังรหัสใบแจ้งหนี้ ฟิลด์อ่านอย่างเดียวที่ถูกล็อกจากการแก้ไข ลิงก์รหัสใบแจ้งหนี้สามารถใช้เพื่อย้อนกลับไปยังส่วนหัวของใบแจ้งหนี้ | 
| **ชื่อ** | แท็บ **ทั่วไป** | ชื่อของรายการใบแจ้งหนี้ที่กำหนดโดยค่าเริ่มต้นจากชื่อรายการสัญญา ฟิลด์นี่สามารถแก้ไขได้ |
| **Project** | แท็บ **ทั่วไป** | โครงการตามรายละเอียดการให้บริการตามสัญญาของโครงการที่เกี่ยวข้อง ฟิลด์อ่านอย่างเดียวที่ถูกล็อกจากการแก้ไข ลิงก์โครงการสามารถใช้เพื่อนำทางไปยังโครงการ | 
| **วิธีการเรียกเก็บเงิน** | แท็บ **ทั่วไป** | วิธีการเรียกเก็บเงินตามรายละเอียดการให้บริการตามสัญญาของโครงการที่เกี่ยวข้อง ฟิลด์อ่านอย่างเดียวที่ถูกล็อกจากการแก้ไข |
| **จำนวนเงินในรายละเอียดการให้บริการตามสัญญา** | แท็บ **ทั่วไป** | จำนวนเงินตามสัญญาในรายละเอียดการให้บริการตามสัญญาโครงการที่เกี่ยวข้อง ฟิลด์อ่านอย่างเดียวที่ถูกล็อกจากการแก้ไข | 
| **ออกใบแจ้งหนี้จนถึงวันที่** | แท็บ **ทั่วไป** | ผลรวมของยอดเงินในรายละเอียดรายการใบแจ้งหนี้ทั้งหมดของใบแจ้งหนี้นี้ ฟิลด์อ่านอย่างเดียวที่ถูกล็อกจากการแก้ไข | 
| **จำนวน** | แท็บ **ทั่วไป** | ผลรวมของยอดเงินในรายละเอียดรายการใบแจ้งหนี้ที่คิดค่าธรรมเนียมได้ทั้งหมดของใบแจ้งหนี้นี้ ฟิลด์อ่านอย่างเดียวที่ถูกล็อกจากการแก้ไข ฟิลด์นี้ใช้ในการคำนวณจำนวนเงินสุดท้ายที่ส่วนหัวของใบแจ้งหนี้ | 
| **ภาษี** | แท็บ **ทั่วไป** | ผลรวมของยอดเงินภาษีในรายละเอียดรายการใบแจ้งหนี้ทั้งหมดของรายการใบแจ้งหนี้นี้ ฟิลด์อ่านอย่างเดียวที่ถูกล็อกจากการแก้ไข ฟิลด์นี้ใช้ในการคำนวณจำนวนเงินภาษีสุดท้ายที่ส่วนหัวของใบแจ้งหนี้ | 
| **จำนวนเงินรวม** | แท็บ **ทั่วไป** | ผลรวมของยอดเงินทั้งหมด (**ภาษี + จำนวนเงิน**) ในรายละเอียดรายการใบแจ้งหนี้ที่คิดค่าธรรมเนียมได้ทั้งหมดของรายการใบแจ้งหนี้นี้ ฟิลด์อ่านอย่างเดียวที่ถูกล็อกจากการแก้ไข ฟิลด์นี้ใช้ในการคำนวณจำนวนเงินสุดท้ายที่ส่วนหัวของใบแจ้งหนี้ |

## <a name="invoice-line-details"></a>รายละเอียดรายการใบแจ้งหนี้

แต่ละรายการใบแจ้งหนี้ในใบแจ้งหนี้โครงการประกอบด้วยรายละเอียดรายการใบแจ้งหนี้ รายละเอียดรายการเหล่านี้เกี่ยวข้องกับยอดขายจริงที่ยังไม่เรียกเก็บเงินและหลักเป้าหมายที่เกี่ยวข้องกับรายละเอียดการให้บริการตามสัญญาที่อ้างอิงโดยรายการใบแจ้งหนี้ ธุรกรรมทั้งหมดนี้มีการทำเครื่องหมาย **พร้อมที่จะออกใบแจ้งหนี้**

สำหรับรายการ **ใบแจ้งหนี้เวลาและวัสดุ** รายละเอียดรายการใบแจ้งหนี้จะจัดกลุ่มเป็น **คิดค่าธรรมเนียมได้**, **คิดค่าธรรมเนียมไม่ได้** และ **ฟรี** ในเพจ **รายการใบแจ้งหนี้** รายละเอียด **รายการใบแจ้งหนี้ที่คิดค่าธรรมเนียมได้** จะรวมอยู่ในยอดรวมรายการใบแจ้งหนี้ **ฟรี** และ **จำนวนเงินตามจริงที่คิดค่าธรรมเนียมไม่ได้** ไม่รวมยอดรวมในรายการใบแจ้งหนี้

สำหรับรายการ **ใบแจ้งหนี้ราคาคงที่** รายละเอียดรายการใบแจ้งหนี้สร้างขึ้นจากหลักเป้าหมายที่ทำเครื่องหมายเป็น **พร้อมที่จะออกใบแจ้งหนี้** ในรายละเอียดการให้บริการตามสัญญาที่เกี่ยวข้อง หลังจากสร้างรายละเอียดรายการใบแจ้งหนี้จากหลักเป้าหมายแล้ว สถานะการเรียกเก็บเงินในหลักเป้าหมายจะอัปเดตเป็น **สร้างใบแจ้งหนี้ของลูกค้าแล้ว**

### <a name="edit-invoice-line-details"></a>แก้ไขรายละเอียดรายการใบแจ้งหนี้

ฟิลด์ต่อไปนี้มีอยู่ในรายละเอียดรายการใบแจ้งหนี้ที่ได้รับการสนับสนุนโดยยอดขายจริงที่ยังไม่เรียกเก็บเงิน

| ฟิลด์ | รายละเอียด |
| --- | --- |

## <a name="refresh-invoice-transactions"></a>รีเฟรชธุรกรรมของใบแจ้งหนี้

หากคุณมีข้อมูลจริงที่เข้ามาหลังจากสร้างใบแจ้งหนี้แล้ว คุณสามารถรวมข้อมูลจริงเหล่านี้ไว้ในใบแจ้งหนี้ได้

1. ใน **มุมมองรายการงานค้างของการเรียกเก็บเงิน** ทำเครื่องหมายข้อมูลเป็น **พร้อมที่จะออกใบแจ้งหนี้**   
2. เปิดแบบร่างใบแจ้งหนี้ชั่วคราว และบน ribbon **การดำเนินการ** เลือก **รีเฟรชธุรกรรมรายการใบแจ้งหนี้**

  รายละเอียดรายการใบแจ้งหนี้สร้างขึ้นสำหรับข้อมูลจริงที่ลงวันที่ในอดีตและทำเครื่องหมายเป็น **พร้อมที่จะออกใบแจ้งหนี้** แต่ไม่รวมอยู่ในใบแจ้งหนี้
