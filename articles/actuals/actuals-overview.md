# <a name="actuals"></a>ข้อมูลจริง 

_**นำไปใช้กับ:** Project Operations สำหรับสถานการณ์ตามทรัพยากร/สินค้าที่ไม่ได้เก็บในคลัง การปรับใช้งานแบบ Lite - จัดการกับการออกใบแจ้งหนี้ชั่วคราว_

ข้อมูลจริงแสดงถึงความคืบหน้าทางการเงินที่ได้รับการตรวจสอบและอนุมัติและและจัดกำหนดการในโครงการ โดยสร้างขึ้นจากการอนุมัติเวลา ค่าใช้จ่าย รายการการใช้วัสดุ และรายการสมุดรายวันและใบแจ้งหนี้

## <a name="journal-lines-and-time-submission"></a>การส่งบรรทัดสมุดรายวันและเวลา

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับรายการเวลา โปรดดู [ภาพรวมของรายการเวลา](../time/time-entry-overview.md)

### <a name="time-and-materials"></a>เวลาและวัสดุ

เมื่อรายการเวลาที่ส่งมีการเชื่อมโยงกับโครงการที่แม็ปกับรายละเอียดการให้บริการตามสัญญาของเวลาและวัสดุ ระบบจะสร้างบรรทัดสมุดรายวันสองรายการ รายการหนึ่งสำหรับต้นทุนและอีกรายการสำหรับการขายที่ยังไม่เรียกเก็บเงิน

### <a name="fixed-price"></a>ราคาคงที่

เมื่อรายการเวลาที่ส่งมีการเชื่อมโยงกับโครงการที่แม็ปกับรายละเอียดการให้บริการตามสัญญาที่มีราคาคงที่ ระบบจะสร้างบรรทัดสมุดรายวันหนึ่งรายการสำหรับต้นทุน

### <a name="default-pricing"></a>การกำหนดราคาเริ่มต้น

ตรรกะสำหรับการสร้างราคาเริ่มต้นอยู่ในบรรทัดสมุดรายวัน ค่าฟิลด์จากรายการเวลาจะถูกคัดลอกไปยังบรรทัดสมุดรายวัน ค่าเหล่านี้รวมถึงวันที่ของธุรกรรม รายละเอียดการให้บริการตามสัญญาที่โครงการถูกแม็ปด้วย และสกุลเงินที่เป็นผลในรายการราคาที่เหมาะสม

ฟิลด์ที่มีผลต่อการกำหนดราคาเริ่มต้น เช่น **บทบาท** และ **หน่วยการจัดเตรียมทรัพยากร** มีการใช้เพื่อกำหนดราคาที่เหมาะสมในบรรทัดสมุดรายวัน คุณสามารถเพิ่มฟิลด์ที่กำหนดเองในรายการเวลา หากคุณต้องการให้ค่าฟิลด์กระจายเป็นข้อมูลจริง ให้สร้างฟิลด์ในตาราง **ข้อมูลจริง** และ **บรรทัดสมุดรายวัน** ใช้โค้ดแบบกำหนดเองเพื่อเผยแพร่ค่าฟิลด์ที่เลือกจากรายการเวลาไปยังข้อมูลจริงผ่านบรรทัดสมุดรายวันโดยใช้ที่มาของธุรกรรม สำหรับข้อมูลเพิ่มเติมเกี่ยวกับที่มาของธุรกรรมและการเชื่อมต่อ โปรดดู [การเชื่อมโยงข้อมูลจริงกับเรกคอร์ดเดิม](linkingactuals.md#example-how-transaction-origin-works-with-transaction-connection)

## <a name="journal-lines-and-basic-expense-submission"></a>การส่งบรรทัดสมุดรายวันและค่าใช้จ่ายพื้นฐาน

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับรายการค่าใช้จ่าย โปรดดู [ภาพรวมของค่าใช้จ่าย](../expense/expense-overview.md)

### <a name="time-and-materials"></a>เวลาและวัสดุ

เมื่อรายการค่าใช้จ่ายพื้นฐานที่ส่งมีการเชื่อมโยงกับโครงการที่แม็ปกับรายละเอียดการให้บริการตามสัญญาของเวลาและวัสดุ ระบบจะสร้างบรรทัดสมุดรายวันสองรายการ รายการหนึ่งสำหรับต้นทุนและอีกรายการสำหรับการขายที่ยังไม่เรียกเก็บเงิน

### <a name="fixed-price"></a>ราคาคงที่

เมื่อรายการค่าใช้จ่ายพื้นฐานที่ส่งเชื่อมโยงกับโครงการที่ถูกแม็ปกับรายละเอียดการให้บริการตามสัญญาที่มีราคาคงที่ ระบบจะสร้างบรรทัดสมุดรายวันเดียวสำหรับต้นทุน

### <a name="default-pricing"></a>การกำหนดราคาเริ่มต้น

ตรรกะสำหรับการป้อนราคาเริ่มต้นสำหรับค่าใช้จ่ายจะขึ้นอยู่กับประเภทค่าใช้จ่าย วันที่ของธุรกรรม รายละเอียดการให้บริการตามสัญญาที่โครงการถูกแม็ปด้วย และสกุลเงิน จะถูกใช้เพื่อระบุรายการราคาที่เหมาะสม ฟิลด์ที่มีผลต่อราคาเริ่มต้น เช่น **ประเภทธุรกรรม** และ **หน่วย** มีการใช้เพื่อกำหนดราคาที่เหมาะสมในบรรทัดสมุดรายวัน อย่างไรก็ตาม วิธีนี้ใช้ได้เฉพาะเมื่อวิธีการกำหนดราคาในรายการราคาเป็น **ราคาต่อหน่วย** หากวิธีการกำหนดราคาเป็น **ตามต้นทุน** หรือ **คิดส่วนเพิ่มราคากับต้นทุน** ราคาที่ป้อนเมื่อสร้างรายการค่าใช้จ่ายจะใช้สำหรับต้นทุน และราคาในรายการสมุดรายวันการขายจะคำนวณตามวิธีการกำหนดราคา 

คุณสามารถเพิ่มฟิลด์แบบกำหนดเองในรายการค่าใช้จ่ายได้ หากคุณต้องการให้ค่าฟิลด์กระจายเป็นข้อมูลจริง ให้สร้างฟิลด์ในตาราง **ข้อมูลจริง** และ **บรรทัดสมุดรายวัน** ใช้โค้ดแบบกำหนดเองเพื่อเผยแพร่ค่าฟิลด์ที่เลือกจากรายการเวลาไปยังข้อมูลจริงผ่านบรรทัดสมุดรายวันโดยใช้ที่มาของธุรกรรม สำหรับข้อมูลเพิ่มเติมเกี่ยวกับที่มาของธุรกรรมและการเชื่อมต่อ โปรดดู [การเชื่อมโยงข้อมูลจริงกับเรกคอร์ดเดิม](linkingactuals.md#example-how-transaction-origin-works-with-transaction-connection)

## <a name="journal-lines-and-material-usage-log-submission"></a>บรรทัดสมุดรายวันและการส่งบันทึกการใช้วัสดุ

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับรายการค่าใช้จ่าย โปรดดู [บันทึกการใช้วัสดุ](../material/material-usage-log.md)

### <a name="time-and-materials"></a>เวลาและวัสดุ

เมื่อรายการบันทึกการใช้วัสดุที่ส่งเชื่อมโยงกับโครงการที่แม็ปกับรายละเอียดการให้บริการตามสัญญาสำหรับเวลาและวัสดุ ระบบจะสร้างบรรทัดสมุดรายวันสองรายการ รายการหนึ่งสำหรับต้นทุน และอีกรายการสำหรับการขายที่ยังไม่เรียกเก็บเงิน

### <a name="fixed-price"></a>ราคาคงที่

เมื่อรายการบันทึกการใช้วัสุดที่ส่งเชื่อมโยงกับโครงการที่ถูกแม็ปกับรายละเอียดการให้บริการตามสัญญาที่มีราคาคงที่ ระบบจะสร้างบรรทัดสมุดรายวันเดียวสำหรับต้นทุน

### <a name="default-pricing"></a>การกำหนดราคาเริ่มต้น

ตรรกะในการป้อนราคาเริ่มต้นสำหรับวัสดุจะอิงตามชุดผสมของผลิตภัณฑ์และหน่วย วันที่ของธุรกรรม รายละเอียดการให้บริการตามสัญญาที่โครงการถูกแม็ปด้วย และสกุลเงิน จะถูกใช้เพื่อระบุรายการราคาที่เหมาะสม ฟิลด์ที่มีผลต่อราคาเริ่มต้น เช่น **รหัสผลิตภัณฑ์** และ **หน่วย** มีการใช้เพื่อกำหนดราคาที่เหมาะสมในบรรทัดสมุดรายวัน อย่างไรก็ตาม วิธีนี้ใช้ได้กับผลิตภัณฑ์ในแคตตาล็อกเท่านั้น สำหรับผลิตภัณฑ์ที่เขียนเพิ่ม ราคาที่ป้อนเมื่อสร้างรายการบันทึกการใช้วัสดุจะใช้สำหรับต้นทุนและราคาขายในบรรทัดสมุดรายวัน 

คุณสามารถเพิ่มฟิลด์แบบกำหนดเองในรายการ **บันทึกการใช้วัสดุ** ได้ หากคุณต้องการให้ค่าฟิลด์กระจายเป็นข้อมูลจริง ให้สร้างฟิลด์ในตาราง **ข้อมูลจริง** และ **บรรทัดสมุดรายวัน** ใช้โค้ดแบบกำหนดเองเพื่อเผยแพร่ค่าฟิลด์ที่เลือกจากรายการเวลาไปยังข้อมูลจริงผ่านบรรทัดสมุดรายวันโดยใช้ที่มาของธุรกรรม สำหรับข้อมูลเพิ่มเติมเกี่ยวกับที่มาของธุรกรรมและการเชื่อมต่อ โปรดดู [การเชื่อมโยงข้อมูลจริงกับเรกคอร์ดเดิม](linkingactuals.md#example-how-transaction-origin-works-with-transaction-connection)

## <a name="use-entry-journals-to-record-costs"></a>ใช้รายการสมุดรายวันเพื่อบันทึกต้นทุน

คุณสามารถใช้รายการสมุดรายวันเพื่อบันทึกต้นทุนหรือรายได้ ในคลาสวัสดุ ค่าธรรมเนียม เวลา ค่าใช้จ่าย หรือธุรกรรมภาษีได้ สมุดรายวันสามารถใช้เพื่อวัตถุประสงค์ดังต่อไปนี้:

- ย้ายข้อมูลจริงของธุรกรรมจากระบบอื่นไปยัง Microsoft Dynamics 365 Project Operations
- บันทึกต้นทุนที่เกิดขึ้นในระบบอื่น ต้นทุนเหล่านี้อาจรวมถึงต้นทุนการจัดซื้อจัดจ้างหรือการรับเหมาช่วง

> [!IMPORTANT]
> แอปพลิเคชันไม่ตรวจสอบความถูกต้องของชนิดบรรทัดสมุดรายวันหรือการกำหนดราคาที่เกี่ยวข้องซึ่งป้อนในบรรทัดสมุดรายวัน ดังนั้น เฉพาะผู้ใช้ที่ทราบถึงผลกระทบทางการบัญชีที่เกิดขึ้นจริงต่อโครงการเท่านั้นที่ควรใช้รายการสมุดรายวันเพื่อสร้างข้อมูลจริง เนื่องจากผลกระทบของชนิดสมุดรายวันนี้ คุณควรเลือกอย่างรอบคอบว่าใครมีสิทธิ์ในการสร้างรายการสมุดรายวัน

## <a name="record-actuals-based-on-project-events"></a>บันทึกข้อมูลจริงตามเหตุการณ์ของโครงการ
