# <a name="work-breakdown-structures-overview"></a><span data-ttu-id="ee1b8-104">ภาพรวมโครงสร้างการแบ่งงาน</span><span class="sxs-lookup"><span data-stu-id="ee1b8-104">Work breakdown structures overview</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="ee1b8-105">โครงสร้างการแบ่งงาน (WBS) คือ Description ของงานที่จะทำสำหรับโครงการ</span><span class="sxs-lookup"><span data-stu-id="ee1b8-105">A work breakdown structure (WBS) is a description of the work that will be done for a project.</span></span> <span data-ttu-id="ee1b8-106">ซึ่งเป็นลำดับชั้นของงานที่แสดงถึงความเข้าใจของกลุ่มคนในโครงการเกี่ยวกับองค์ประกอบของงาน และของขนาด ต้นทุน และระยะเวลา ของแต่ละองค์ประกอบหรืองาน</span><span class="sxs-lookup"><span data-stu-id="ee1b8-106">It’s a hierarchy of tasks that represents the project team’s understanding of the composition of work, and of the size, cost, and duration of each component or task.</span></span> <span data-ttu-id="ee1b8-107">WBS มีจุดประสงค์หลักสามประการ:</span><span class="sxs-lookup"><span data-stu-id="ee1b8-107">A WBS has three major purposes:</span></span>

-   <span data-ttu-id="ee1b8-108">อธิบายรายละเอียดแยกย่อยหรือองค์ประกอบของงานในงาน</span><span class="sxs-lookup"><span data-stu-id="ee1b8-108">Describe the breakdown or composition of work in tasks.</span></span>

<span data-ttu-id="ee1b8-111">ระดับของรายละเอียดใน WBS ขึ้นอยู่กับระดับความแม่นยำที่จำเป็นในการประมาณการและระดับของการติดตามที่จำเป็นสำหรับการประมาณการเหล่านั้น</span><span class="sxs-lookup"><span data-stu-id="ee1b8-111">The degree of detail in a WBS depends on the level of accuracy that is required in estimates and the level of tracking that is required against those estimates.</span></span> <span data-ttu-id="ee1b8-112">โครงการที่มีความอดทนต่อการเลื่อนหลุดในกำหนดการหรือค่าใช้จ่ายต่ำมาก มักจะต้องมี WBS ที่ละเอียดกว่านี้ และการติดตามความคืบหน้าของงานและต้นทุนที่มากกว่านี้ เมื่อเทียบกับ WBS</span><span class="sxs-lookup"><span data-stu-id="ee1b8-112">Projects that have very low tolerance for slippages in schedule or cost usually require a more detailed WBS, and diligent tracking of work progress and cost against the WBS.</span></span> <span data-ttu-id="ee1b8-113">โครงการชนิดนี้มีอยู่ทั่วไปในอุตสาหกรรมการก่อสร้างและวิศวกรรม</span><span class="sxs-lookup"><span data-stu-id="ee1b8-113">This kind of project is common in the construction and engineering industries.</span></span>

<span data-ttu-id="ee1b8-237">มุมมองการวางแผนแสดงการประมาณตามแผนหรือพื้นฐานของกำหนดการและข้อมูลต้นทุน</span><span class="sxs-lookup"><span data-stu-id="ee1b8-237">The Planning view displays the planned or baseline estimate of the schedule and cost information.</span></span> <span data-ttu-id="ee1b8-238">แม้ว่าจะไม่มีคุณลักษณะสำหรับการติดตามเวอร์ชันและพื้นฐานสำหรับโครงการ WBS ค่าในมุมมองนี้มีจุดมุ่งหมายเพื่อแสดงถึงเวอร์ชันพื้นฐาน</span><span class="sxs-lookup"><span data-stu-id="ee1b8-238">Although there are no features for tracking the version and baseline for a project WBS, the values in this view are intended to represent the baseline version.</span></span> <span data-ttu-id="ee1b8-239">ส่วนการประมาณกำหนดการและการประมาณต้นทุนของหัวข้อนี้ อธิบายมุมมองนี้และวิธีใช้ในการสร้าง WBS</span><span class="sxs-lookup"><span data-stu-id="ee1b8-239">The Schedule estimation and Cost estimation sections of this topic describe this view and how it’s used to create a WBS.</span></span>

### <a name="effort-tracking-view"></a><span data-ttu-id="ee1b8-240">มุมมองการติดตามความพยายามที่ใช้</span><span class="sxs-lookup"><span data-stu-id="ee1b8-240">Effort tracking view</span></span>

<span data-ttu-id="ee1b8-241">มุมมองการติดตามความพยายามจะแสดงการติดตามความคืบหน้าของงานใน WBS</span><span class="sxs-lookup"><span data-stu-id="ee1b8-241">The Effort tracking view displays the tracking of progress for tasks in the WBS.</span></span> <span data-ttu-id="ee1b8-242">ซึ่งจะเปรียบเทียบชั่วโมงความพยายามจริงสะสมสำหรับงานกับชั่วโมงความพยายามที่วางแผนไว้</span><span class="sxs-lookup"><span data-stu-id="ee1b8-242">It compares the accumulated actual effort hours for a task to the planned effort hours.</span></span> <span data-ttu-id="ee1b8-243">สูตรต่อไปนี้ระบุค่าในมุมมองการติดตามความพยายาม:</span><span class="sxs-lookup"><span data-stu-id="ee1b8-243">The following formulas provide the values in the Effort tracking view:</span></span>

-   <span data-ttu-id="ee1b8-244">เปอร์เซ็นต์ความคืบหน้า = ความพยายามจริงจนถึงปัจจุบัน ÷ ความพยายามที่วางแผนไว้สำหรับงาน</span><span class="sxs-lookup"><span data-stu-id="ee1b8-244">Progress percentage = Actual effort to date ÷ Planned effort for the task</span></span>
-   <span data-ttu-id="ee1b8-245">ความพยายามที่เหลืออยู่ (หรือที่เรียกว่า Estimate-to-complete \[ฯลฯ\]) = ความพยายามตามแผน - ความพยายามจริงจนถึงปัจจุบัน</span><span class="sxs-lookup"><span data-stu-id="ee1b8-245">Remaining effort (also known as Estimate-to-complete \[ETC\]) = Planned effort – Actual effort to date</span></span>
-   <span data-ttu-id="ee1b8-246">การประมาณการที่ใช้เมื่อดำเนินงานเสร็จสมบูรณ์ (EAC) = ความพยายามที่เหลือ + ความพยายามจริงจนถึงปัจจุบัน</span><span class="sxs-lookup"><span data-stu-id="ee1b8-246">Estimate at complete (EAC) = Remaining effort + Actual effort to date</span></span>
-   <span data-ttu-id="ee1b8-247">ผลต่างความพยายามที่คาดการณ์ = ความพยายามที่วางแผนไว้ – EAC</span><span class="sxs-lookup"><span data-stu-id="ee1b8-247">Projected effort variance = Planned effort – EAC</span></span>

<span data-ttu-id="ee1b8-248">มุมมองการติดตามความพยายามแสดงการคาดการณ์ผลต่างของความพยายามสำหรับงาน โดยพิจารณาว่า EAC มากกว่าหรือน้อยกว่าความพยายามที่วางแผนไว้:</span><span class="sxs-lookup"><span data-stu-id="ee1b8-248">The Effort tracking view displays a projection of the effort variance for the task, based on whether EAC is more or less than the planned effort:</span></span>

-   <span data-ttu-id="ee1b8-249">ถ้า EAC มากกว่าความพยายามที่วางแผนไว้ งานจะถูกคาดการณ์ให้ใช้เวลามากกว่าที่วางแผนไว้แต่เดิม และล่าช้ากว่ากำหนด</span><span class="sxs-lookup"><span data-stu-id="ee1b8-249">If EAC is more than the planned effort, the task is projected to take more time than originally planned and is behind schedule.</span></span>
-   <span data-ttu-id="ee1b8-250">ถ้า EAC น้อยกว่าความพยายามที่วางแผนไว้ งานจะถูกคาดการณ์ให้ใช้เวลาน้อยกว่าที่วางแผนไว้แต่เดิม และเร็วกว่ากำหนด</span><span class="sxs-lookup"><span data-stu-id="ee1b8-250">If EAC is less than the planned effort, the task is projected to take less time than originally planned and is ahead of schedule.</span></span>

<span data-ttu-id="ee1b8-251">**การประมาณความพยายามอีกครั้งของผู้จัดการโครงการ** ในบางครั้ง ผู้จัดการโครงการหรือบุคคลอื่นที่ติดตามความคืบหน้าของโครงการจะต้องแก้ไขประมาณการเดิมของงาน</span><span class="sxs-lookup"><span data-stu-id="ee1b8-251">**Project manager’s re-projection of effort** Occasionally, the project manager or another persona that is tracking the progress of a project will have to revise the original estimates on a task.</span></span> <span data-ttu-id="ee1b8-252">งานอาจเดินเร็วขึ้นหรือช้ากว่าที่คาดการณ์ไว้ในตอนแรก เนื่องด้วยเหตุผลหลายประการ</span><span class="sxs-lookup"><span data-stu-id="ee1b8-252">The task might be moving faster or slower than originally anticipated for various reasons.</span></span> <span data-ttu-id="ee1b8-253">ตัวอย่างเช่น ขอบเขตถูกลดลง หรือพนักงานมีประสบการณ์น้อยกว่าที่วางแผนไว้ในตอนแรก</span><span class="sxs-lookup"><span data-stu-id="ee1b8-253">For example, the scope has been reduced, or workers have less experience than originally planned.</span></span> <span data-ttu-id="ee1b8-254">การคาดการณ์เป็นการรับรู้ของผู้จัดการโครงการในการประมาณการ โดยยึดตามสถานะปัจจุบันในโครงการ</span><span class="sxs-lookup"><span data-stu-id="ee1b8-254">Projections are a project manager’s perception of estimates, based on the current status on a project.</span></span> <span data-ttu-id="ee1b8-255">โดยทั่วไป คุณไม่ควรเปลี่ยนเลขเส้นฐาน เนื่องจากเส้นฐานของโครงการเป็นเอกสารที่เผยแพร่อย่างดีสำหรับกำหนดการและการประเมินต้นทุนของโครงการที่ผู้เกี่ยวข้องทั้งหมดในโครงการได้ตกลงไว้</span><span class="sxs-lookup"><span data-stu-id="ee1b8-255">In general, you should not change the baseline numbers, because a project baseline represents a well-published document for the project’s schedule and cost estimate that all stakeholders on the project have agreed to.</span></span> 

<span data-ttu-id="ee1b8-256">มีสองวิธีที่ผู้จัดการโครงการสามารถปรับเปลี่ยนความพยายามในงานได้:</span><span class="sxs-lookup"><span data-stu-id="ee1b8-256">There are two ways that project managers can modify the effort on tasks:</span></span>

-   <span data-ttu-id="ee1b8-257">ปรับเปลี่ยนความพยายามที่เหลืออยู่ซึ่งถูกตั้งค่าโดยอัตโนมัติเพื่อปรับปรุงความพยายามที่เหลืออยู่จริงในงาน</span><span class="sxs-lookup"><span data-stu-id="ee1b8-257">Modify the remaining effort that is automatically set to update the actual remaining effort on the task.</span></span>
-   <span data-ttu-id="ee1b8-258">ปรับเปลี่ยนเปอร์เซ็นต์ความคืบหน้าซึ่งถูกตั้งค่าโดยอัตโนมัติเพื่อปรับปรุงความคืบหน้าจริงในงาน</span><span class="sxs-lookup"><span data-stu-id="ee1b8-258">Modify the progress percentage that is automatically set to update the true progress on the task.</span></span>

<span data-ttu-id="ee1b8-259">วิธีเหล่านี้ทั้งสองทำให้เกิดการคำนวณใหม่ของ ETC, EAC และเปอร์เซ็นต์ความคืบหน้าของงาน และผลต่างความพยายามที่คาดการณ์ไว้ในงาน</span><span class="sxs-lookup"><span data-stu-id="ee1b8-259">Both of these approaches cause a recalculation of the task’s ETC, EAC, and progress percentage, and the projected effort variance on the task.</span></span> <span data-ttu-id="ee1b8-260">นอกจากนี้ EAC, ETC, และเปอร์เซ็นต์ความคืบหน้าในงานสรุปจะถูกคำนวณอีกครั้ง และมีการปรับปรุงผลต่างความพยายามที่มีการคาดการณ์ของพวกเขา</span><span class="sxs-lookup"><span data-stu-id="ee1b8-260">The EAC, ETC, and progress percentage on summary tasks are also recalculated, and their projected effort variance is updated.</span></span> 

<span data-ttu-id="ee1b8-261">**ความพยายามที่ปรับเปลี่ยนในงานสรุป** คุณสามารถปรับเปลี่ยนความพยายามในงานสรุปหรืองานคอนเทนเนอร์</span><span class="sxs-lookup"><span data-stu-id="ee1b8-261">**Modified effort on summary tasks** You can modify the effort on summary or container tasks.</span></span> <span data-ttu-id="ee1b8-262">โดยไม่คำนึงถึงว่าคุณปรับเปลี่ยนค่าเหล่านี้หรือไม่ โดยใช้ความพยายามที่เหลือ หรือเปอร์เซ็นต์ความคืบหน้าในงานสรุป การคำนวณจะเกิดขึ้นโดยอัตโนมัติตามลำดับต่อไปนี้:</span><span class="sxs-lookup"><span data-stu-id="ee1b8-262">Regardless of whether you modify these values by using the remaining effort or the progress percentage on the summary tasks, the calculations occur automatically in the following order:</span></span>

<span data-ttu-id="ee1b8-269">คลิก **ขยายไปยังระดับ** ในมุมมองการติดตามความพยายาม เพื่อกำหนดระดับที่จะติดตามและรักษา WBS ของคุณ</span><span class="sxs-lookup"><span data-stu-id="ee1b8-269">Click **Expand to level** in the Effort tracking view to set the level at which to track and maintain your WBS.</span></span> <span data-ttu-id="ee1b8-270">WBS จะถูกขยายโดยอัตโนมัติไปที่ระดับนั้นในมุมมองการติดตามความพยายาม ทุกครั้งที่คุณเปิด</span><span class="sxs-lookup"><span data-stu-id="ee1b8-270">The WBS is automatically expanded to that level in the Effort tracking view whenever you open it.</span></span>

### <a name="cost-tracking-view"></a><span data-ttu-id="ee1b8-271">มุมมองการติดตามต้นทุน</span><span class="sxs-lookup"><span data-stu-id="ee1b8-271">Cost tracking view</span></span>

<span data-ttu-id="ee1b8-272">มุมมองการติดตามต้นทุนจะแสดงการติดตามการใช้ต้นทุนสำหรับงาน</span><span class="sxs-lookup"><span data-stu-id="ee1b8-272">The Cost tracking view displays the tracking of cost consumption for a task.</span></span> <span data-ttu-id="ee1b8-273">ในมุมมองนี้ ต้นทุนจริงที่ใช้กับงานจนถึงปัจจุบันจะถูกเปรียบเทียบกับต้นทุนที่วางแผนไว้สำหรับงานนั้น</span><span class="sxs-lookup"><span data-stu-id="ee1b8-273">In this view, the actual cost that has been spent against a task to date is compared to the planned cost for the task.</span></span> <span data-ttu-id="ee1b8-274">สูตรต่อไปนี้ระบุค่าในมุมมองการติดตามต้นทุน:</span><span class="sxs-lookup"><span data-stu-id="ee1b8-274">The following formulas provide the values in the Cost tracking view:</span></span>

-   <span data-ttu-id="ee1b8-278">ผลต่างต้นทุนที่คาดการณ์ = ต้นทุนที่วางแผนไว้ – EAC</span><span class="sxs-lookup"><span data-stu-id="ee1b8-278">Projected cost variance = Planned cost – EAC</span></span>

<span data-ttu-id="ee1b8-279">มุมมองการติดตามต้นทุนจะแสดงการคาดการณ์ความแปรปรวนของต้นทุนสำหรับงาน โดยพิจารณาว่า EAC มากกว่าหรือน้อยกว่าต้นทุนที่วางแผนไว้:</span><span class="sxs-lookup"><span data-stu-id="ee1b8-279">The Cost tracking view displays a projection of the cost variance for the task, based on whether EAC is more or less than the planned cost:</span></span>

<span data-ttu-id="ee1b8-334">หากมูลค่าตามแผนสำหรับช่วงเวลาหนึ่งน้อยกว่ามูลค่าที่ได้รับ จำนวนงานที่ทำในโครงการจะมากกว่าที่วางแผนไว้</span><span class="sxs-lookup"><span data-stu-id="ee1b8-334">If the planned value for a period is less than the earned value, the amount of work that has been done on a project is more than what was planned.</span></span> <span data-ttu-id="ee1b8-335">ดังนั้น โครงการจึงเร็วกว่ากำหนด</span><span class="sxs-lookup"><span data-stu-id="ee1b8-335">Therefore, the project is ahead of schedule.</span></span> <span data-ttu-id="ee1b8-336">นอกจากนี้ ระยะเวลารอคอยสินค้านี้ยังมีการกำหนดมูลค่าเป็นตัวเงิน</span><span class="sxs-lookup"><span data-stu-id="ee1b8-336">This lead time is also given a monetary value.</span></span>

<span data-ttu-id="ee1b8-341">หากค่าใช้จ่ายจริงที่ใช้จ่ายในช่วงเวลาหนึ่งมากกว่ามูลค่าที่ได้รับ แสดงว่ามีการใช้เงินมากกว่าที่ได้รับ</span><span class="sxs-lookup"><span data-stu-id="ee1b8-341">If the actual cost that is spent for a period is more than the earned value, more money was spent than earned.</span></span> <span data-ttu-id="ee1b8-342">ดังนั้น โครงการจึงเกินงบประมาณ</span><span class="sxs-lookup"><span data-stu-id="ee1b8-342">Therefore, the project is over budget.</span></span>
