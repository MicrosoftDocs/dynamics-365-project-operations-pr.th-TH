# <a name="manage-project-invoice-proposals"></a>จัดการข้อเสนอใบแจ้งหนี้โครงการชั่วคราว

_**นำไปใช้กับ:** Project Operations สำหรับสถานการณ์ตามทรัพยากร/วัสดุที่ไม่ได้เก็บในคลัง_

ข้อเสนอใบแจ้งหนี้โครงการสามารถดำเนินการได้โดยฝ่ายเรียกเก็บเงินของคุณ เมื่อตรงตามเงื่อนไขสองข้อต่อไปนี้:

  - ผู้จัดการโครงการยืนยันใบแจ้งหนี้ชั่วคราวใน Microsoft Dataverse
  - ธุรกรรมการขายของเวลาและวัสดุที่ไม่ได้เรียกเก็บเงินทั้งหมดที่รวมอยู่ในใบแจ้งหนี้ชั่วคราว จะถูกลงรายการบัญชีโดยใช้สมุดรายวัน Dynamics 365 **Project Operations Integration**

ใช้ขั้นตอนต่อไปนี้เพื่อกรอกข้อเสนอใบแจ้งหนี้โครงการใน Dynamics 365 Finance

1. Project Operationsตรวจสอบข้อมูลการเรียกเก็บเงินสำหรับธุรกรรมเวลาและวัสดุ และลงรายการบัญชีสมุดรายวัน **Project Operations Integration**
2. รีวิวข้อมูลการเรียกเก็บเงินสำหรับเหตุการณ์สำคัญในการเรียกเก็บเงินราคาคงที่
3. รีวิวและจัดรูปแบบข้อเสนอใบแจ้งหนี้ของโครงการ
4. ลงรายการบัญชีและพิมพ์ใบแจ้งหนี้ของโครงการ

## <a name="manage-billing-information-in-the-project-operations-integration-journal"></a>จัดการข้อมูลการเรียกเก็บเงินในสมุดรายวัน Project Operations Integration

ข้อมูลจริงของโครงการที่สร้างขึ้นใน Dataverse ได้รับการตรวจสอบและลงรายการบัญชีใน Finance โดยนักบัญชีโครงการ สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการทำงานกับสมุดรายวัน Integration ดู [สมุดรายวัน Integration ใน Project Operations](../project-accounting/project-operations-integration-journal.md)

ต้นทุนจริงและยอดขายจริงที่ยังไม่เรียกเก็บเงิน จะถูกเพิ่มลงในสมุดรายวัน Integration เป็นรายการที่แยกกัน:

  - ราคาต้นทุนต่อหน่วยและจำนวนต้นทุนบนต้นทุนที่แท้จริง เริ่มต้นจากธุรกรรมต้นทุนจริงของโครงการใน Dataverse
  - ราคาขายต่อหน่วยและยอดขายในธุรกรรมการขายที่ยังไม่เรียกเก็บเงิน เริ่มต้นจากธุรกรรมการขายจริงที่ยังไม่เรียกเก็บเงินของโครงการใน Dataverse

### <a name="billing-sales-tax"></a>ภาษีขายสำหรับการเรียกเก็บเงิน

การคำนวณภาษีขายสำหรับการเรียกเก็บเงินจะพิจารณาจากชุดข้อมูลของฟิลด์ **กลุ่มภาษีขายสำหรับการเรียกเก็บเงิน** และ **กลุ่มภาษีขายของรายการเรียกเก็บเงิน** ในเรกคอร์ดการขายที่ยังไม่เรียกเก็บเงินในบรรทัดสมุดรายวัน **Project Operations Integration** ระบบจะตั้งค่าฟิลด์เหล่านี้เป็นค่าเริ่มต้นตามการตั้งค่าบนแท็บ **การเงิน** บนหน้า **พารามิเตอร์การจัดการโครงการและการบัญชี**

- **วิธีการกลุ่มภาษีขาย** กำหนดกลุ่มภาษีขายสำหรับการเรียกเก็บเงินที่กำหนดตรรกะเริ่มต้น:
  
  - **โครงการ**: จะกำหนดกลุ่มภาษีขายสำหรับการเรียกเก็บเงินเริ่มต้นจากโครงการเสมอ คุณสามารถรีวิวหรือเปลี่ยนกลุ่มภาษีขายสำหรับการเรียกเก็บเงินเริ่มต้นในโครงการได้โดยเลือก **แสดงการบัญชีเริ่มต้น** บนหน้า **โครงการทั้งหมด**
  - **สัญญาโครงการ**: จะกำหนดกลุ่มภาษีขายสำหรับการเรียกเก็บเงินเริ่มต้นจากสัญญาโครงการเสมอ คุณสามารถรีวิวหรือเปลี่ยนกลุ่มภาษีขายเริ่มต้นในสัญญาโครงการได้โดยเลือก **แสดงการบัญชีเริ่มต้น** บนหน้า **สัญญาโครงการ**
  - **ลูกค้า**: จะกำหนดกลุ่มภาษีขายสำหรับการเรียกเก็บเงินเริ่มต้นจากลูกค้าเสมอ
  - **ค้นหา**: จะค้นหาในเอนทิตีข้างต้นทั้งหมด และเลือกค่าแรกที่พร้อมใช้งาน การค้นหาเริ่มต้นด้วยเอนทิตี **โครงการ** ตามด้วยเอนทิตี **สัญญาโครงการ** และสุดท้ายคือเอนทิตี **ลูกค้า**

- **วิธีการกลุ่มภาษีขายของสินค้า** กำหนดกลุ่มภาษีขายของรายการเรียกเก็บเงินที่กำหนดตรรกะเริ่มต้น:
  
  - สำหรับชนิดธุรกรรมเวลา ค่าใช้จ่าย และค่าธรรมเนียม กลุ่มภาษีขายของรายการเรียกเก็บเงินจะเริ่มต้นจากเอนทิตี **ประเภทโครงการ**
  - สำหรับชนิดธุรกรรมวัสดุ กลุ่มภาษีขายของรายการเรียกเก็บเงินจะเริ่มต้นโดยยึดตามการตั้งค่า **วิธีการกลุ่มภาษีขายสินค้า** ใน **พารามิเตอร์การจัดการโครงการและการบัญชี**. หมายเลขสินค้ากำหนดค่าเริ่มต้นของกลุ่มภาษีขายสินค้าจากเอนทิตี **ผลิตภัณฑ์ที่วางจำหน่าย** ประเภทกำหนดค่าเริ่มต้นของกลุ่มภาษีขายสินค้าจากเอนทิตี **ประเภทโครงการ**

### <a name="financial-dimensions"></a>มิติทางการเงิน

มิติทางการเงินสำหรับเรกคอร์ดการขายที่ยังไม่เรียกเก็บเงินในค่าเริ่มต้นของสมุดรายวัน **Project Operations Integration** จากเอนทิตี **โครงการ** สามารถตรวจสอบและปรับเปลี่ยนมิติทางการเงินได้โดยการเลือก **แจกจ่ายจำนวนเงิน** หากคุณต้องการเปลี่ยนมิติทางการเงินของเรกคอร์ดการขายที่ยังไม่เรียกเก็บเงิน หลังจากที่ลงรายการบัญชีสมุดรายวัน Integration แต่ก่อนที่ใบแจ้งหนี้ชั่วคราวจะได้รับการยืนยัน ให้ไปที่ **โครงการทั้งหมด** > **จัดการ** > **ลงรายการบัญชีธุรกรรม** เลือกธุรกรรม และจากนั้น เลือก **กระบวนการ** > **ปรับการบัญชี**

### <a name="exchange-rates"></a>อัตราแลกเปลี่ยน

สกุลเงินของธุรกรรมที่ยังไม่เรียกเก็บใน Dataverse ถูกใช้เป็นสกุลเงินของธุรกรรมใน Finance และแปลงเป็นสกุลเงินทางบัญชีของบริษัทโดยใช้อัตราแลกเปลี่ยนที่กำหนดไว้ใน Finance


## <a name="manage-the-financial-attributes-of-billing-milestones"></a>จัดการแอตทริบิวต์ทางการเงินของเหตุการณ์สำคัญในการเรียกเก็บเงิน 

รายละเอียดการให้บริการตามสัญญาโครงการที่ใช้วิธีการเรียกเก็บเงินราคาคงที่ จะออกใบแจ้งหนี้ผ่าน [เหตุการณ์สำคัญของราคาคงที่](../sales/invoice-schedules-contract-line.md#create-a-fixed-price-invoice-schedule-for-a-contract-line) นักบัญชีโครงการสามารถรีวิวเหตุการณ์สำคัญในการเรียกเก็บเงินใน Finance ได้โดยไปที่ **การจัดการโครงการและการบัญชี** > **โครงการทั้งหมด** > **จัดการ** > **ธุรกรรมในบัญชี**

### <a name="billing-sales-tax"></a>ภาษีขายสำหรับการเรียกเก็บเงิน

รีวิว **กลุ่มภาษีขาย** และ **กลุ่มภาษีขายสินค้า** เริ่มต้นจากการตั้งค่าเมื่อมีการสร้างเหตุการณ์สำคัญในการเรียกเก็บเงินใหม่ใน Dataverse ระบบจะตั้งค่าเริ่มต้นเป็นฟิลด์เหล่านี้ตามการตั้งค่าบนแท็บ **ทางการเงิน** บนหน้า **พารามิเตอร์การจัดการโครงการและการบัญชี**

- **วิธีการกลุ่มภาษีขาย** กำหนดตรรกะเริ่มต้นของ **กลุ่มภาษีขายสำหรับการเรียกเก็บเงิน**:

    - **โครงการ** จะกำหนดกลุ่มภาษีขายสำหรับการเรียกเก็บเงินเริ่มต้นจากโครงการเสมอ คุณสามารถรีวิวหรือเปลี่ยนกลุ่มภาษีขายเริ่มต้นในโครงการได้โดยเลือก **แสดงการบัญชีเริ่มต้น** บนหน้า **โครงการทั้งหมด**
    - **สัญญาโครงการ** จะกำหนดกลุ่มภาษีขายสำหรับการเรียกเก็บเงินเริ่มต้นจากสัญญาโครงการเสมอ คุณสามารถรีวิวหรือเปลี่ยนกลุ่มภาษีขายเริ่มต้นในสัญญาโครงการได้โดยเลือก **แสดงการบัญชีเริ่มต้น** บนหน้า **สัญญาโครงการ**
    - **ลูกค้า** จะกำหนดค่าเริ่มต้นเป็นกลุ่มภาษีขายสำหรับการเรียกเก็บเงินจากลูกค้าเสมอ
    - **ค้นหา** จะค้นหาในเอนทิตีทั้งหมดในรายการนี้ และเลือกค่าแรกที่พร้อมใช้งาน การค้นหาเริ่มต้นด้วยเอนทิตี **โครงการ** ตามด้วยเอนทิตี **สัญญาโครงการ** และจากนั้น เอนทิตี **ลูกค้า**

- **กลุ่มภาษีขายสินค้าของหลักเป้าหมายราคาคงที่** ถูกใช้เป็นค่าเริ่มต้นในฟิลด์ **กลุ่มภาษีขายสินค้า** สำหรับเหตุการณ์สำคัญของการเรียกเก็บเงิน นักบัญชีสามารถตรวจสอบและแก้ไขค่านี้ในหน้า **ธุรกรรมในบัญชี** ระบบใช้มูลค่าจากธุรกรรมในบัญชี เมื่อสร้างรายการข้อเสนอใบแจ้งหนี้โครงการ
 

### <a name="financial-dimensions"></a>มิติทางการเงิน

มิติทางการเงินเริ่มต้นสำหรับเหตุการณ์สำคัญในการเรียกเก็บเงินราคาคงที่ถูกกำหนดไว้ในรายละเอียดการให้บริการตามสัญญาโครงการ ไปที่ **สัญญาโครงการ** > **แสดงการบัญชีเริ่มต้น** และบนแท็บ **รายละเอียดการให้บริการตามสัญญา** เลือก **รายละเอียดการให้บริการตามสัญญาของราคา** จากนั้น ตั้งค่ามิติทางการเงินที่คุณต้องการใช้เป็นค่าเริ่มต้น

นักบัญชีโครงการสามารถแก้ไขข้อมูลภาษีขายและมิติทางการเงินในเหตุการณ์สำคัญของใบแจ้งหนี้ จนกว่าจะมีการสร้างข้อเสนอใบแจ้งหนี้โครงการ


## <a name="create-project-invoice-proposals"></a>สร้างข้อเสนอใบแจ้งหนี้โครงการ

สามารถตรวจสอบข้อเสนอใบแจ้งหนี้โครงการได้ในโมดูล **การจัดการโครงการและการบัญชี** โดยไปที่ **ใบแจ้งหนี้โครงการ** > **ข้อเสนอใบแจ้งหนี้โครงการ**

ส่วนหัวข้อเสนอใบแจ้งหนี้โครงการถูกสร้างขึ้นใน Finance เมื่อใบแจ้งหนี้ชั่วคราวได้รับการยืนยันใน Dataverse เพื่อให้การกระทบยอดง่ายขึ้น ระบบจะตั้งค่าหมายเลขข้อเสนอใบแจ้งหนี้โครงการใน Finance เป็นหมายเลขเดียวกับรหัสใบแจ้งหนี้ชั่วคราวใน Dataverse เนื่องจากใบแจ้งหนี้ชั่วคราวไม่จำเป็นต้องได้รับการยืนยันในลำดับเดียวกันกับที่สร้างขึ้น ลำดับหมายเลขข้อเสนอใบแจ้งหนี้โครงการใน Finance ต้องอนุญาตให้มีการเปลี่ยนแปลงเป็นตัวเลขที่ต่ำกว่าและสูงกว่า ตั้งค่าคอนฟิกลำดับหมายเลขโดยใช้ขั้นตอนต่อไปนี้:

1. ใน Finance ไปที่ **การจัดการองค์การ** > **ลำดับหมายเลข** > **ลำดับหมายเลข**
2. ในตัวกรอง **พื้นที่** เลือก **โครงการ**
3. ในตัวกรอง **การอ้างอิง** เลือก **ข้อเสนอใบแจ้งหนี้**
4. ใช้ฟิลด์ **บริษัท** เพื่อกรองนิติบุคคลแต่ละรายด้วยการรวม Project Operations Dataverse ที่เปิดใช้งาน
5. เปิด **รายละเอียดลำดับหมายเลข** และบนชุดแท็บ **ทั่วไป**:

    - **อนุญาตให้ผู้ใช้เปลี่ยนแปลง: เป็นตัวเลขที่ต่ำกว่า** = **ใช่**
    - **อนุญาตให้ผู้ใช้เปลี่ยนแปลง: เป็นตัวเลขที่สูงกว่า** = **ใช่**

ระบบจะเพิ่มรายการข้อเสนอใบแจ้งหนี้โครงการโดยใช้กระบวนการตามงวด **นำเข้าจากตารางการจัดเตรียม** (**การจัดการโครงการและการบัญชี** > **เป็นระยะ** > **การรวม Project Operations** > **นำเข้าจากตารางการจัดเตรียม**) สามารถเรียกใช้กระบวนการนี้ได้ด้วยตนเองหรือโดยใช้กำหนดการเป็นระยะ ระบบจะไม่เพิ่มรายการในเอกสารข้อเสนอใบแจ้งหนี้ จนกว่าทุกรายการจะพร้อมสำหรับการออกใบแจ้งหนี้ ธุรกรรมเวลาและวัสดุพร้อมที่จะออกใบแจ้งหนี้ ก็ต่อเมื่อมีการลงรายการบัญชีโดยใช้สมุดรายวัน **Project Operations Integration**

## <a name="format-and-print-invoice-proposals"></a>จัดรูปแบบและพิมพ์ข้อเสนอใบแจ้งหนี้

นักบัญชีโครงการสามารถปรับแต่งการพิมพ์ใบแจ้งหนี้โครงการโดยใช้หน้า **จัดรูปแบบข้อเสนอใบแจ้งหนี้** และพิมพ์ความสามารถในการจัดการหน้า

### <a name="format-invoice-proposals"></a>จัดรูปแบบข้อเสนอใบแจ้งหนี้

หน้า **จัดรูปแบบข้อเสนอใบแจ้งหนี้** อนุญาตให้ธุรกรรมการจัดกลุ่มที่กำหนดเองแสดงในใบแจ้งหนี้โครงการของลูกค้า

1. บนหน้า **ข้อเสนอใบแจ้งหนี้โครงการ** ให้เลือก **พิมพ์** > **จัดรูปแบบข้อเสนอใบแจ้งหนี้**
2. เลือก **ใหม่** เพื่อสร้างการจัดกลุ่มใหม่สำหรับการพิมพ์ใบแจ้งหนี้โครงการ
3. ในฟิลด์ **รายละเอียด/สรุป** เลือกตัวเลือกสำหรับการจัดกลุ่มนี้:

    - เลือก **รายละเอียด** เพื่อพิมพ์รายละเอียดธุรกรรมของใบแจ้งหนี้ของลูกค้า
    - เลือก **สรุป** เพื่อพิมพ์สรุปธุรกรรมของใบแจ้งหนี้ของลูกค้า

> [!NOTE]
> การเลือกในฟิลด์ **รายละเอียด/สรุป** บนหน้า **จัดรูปแบบข้อเสนอใบแจ้งหนี้** จะแทนที่ตัวเลือกที่เลือกบนฟิลด์ **รูปแบบใบแจ้งหนี้** บนหน้า **ข้อเสนอใบแจ้งหนี้** เพื่อพิมพ์ใบแจ้งหนี้โดยละเอียดหรือใบแจ้งหนี้แบบสรุป

- เลือกรายการธุรกรรมที่จะรวมไว้ในส่วนนี้บนแท็บ **ธุรกรรมที่มีอยู่** แล้วเลือก **รวมธุรกรรม** เพื่อย้ายไปยังแท็บ **ธุรกรรมที่เลือก**
- เลือก **ย้ายขึ้น** และ **ย้ายลง** เพื่อเปลี่ยนลำดับของส่วนต่างๆ
- เลือก **ตัวอย่างก่อนพิมพ์** เพื่อดูตัวอย่างใบแจ้งหนี้ที่จัดรูปแบบ

### <a name="print-management"></a>การจัดการการพิมพ์

การจัดการการพิมพ์ใช้ไฟล์รายงานที่แตกต่างกันเพื่อพิมพ์ ระบุปลายทาง และปรับแต่งข้อความส่วนท้ายสำหรับใบแจ้งหนี้ คุณสามารถตั้งค่าการจัดการการพิมพ์ได้ที่ระดับโมดูล อย่างไรก็ตาม การตั้งค่าเหล่านี้สามารถถูกแทนที่ได้สำหรับลูกค้า สัญญา หรือข้อเสนอใบแจ้งหนี้ที่เฉพาะเจาะจง ในการเข้าถึงฟังก์นนี้บนหน้า **ข้อเสนอใบแจ้งหนี้โครงการ** ให้เลือก **พิมพ์** > **การจัดการการพิมพ์**

การตั้งค่าการจัดการการพิมพ์จะแสดงเป็นมุมมองทรี โดยแต่ละระดับโหนดจะแสดงเอกสารที่มีอยู่เพื่อปรับ คุณสามารถมอบหมายงานพิมพ์แบบกำหนดเองที่ระดับโมดูล ลูกค้า สัญญา หรือเอกสารข้อเสนอใบแจ้งหนี้ ในการแก้ไขงานพิมพ์เอกสารต้นฉบับ ให้ขยายโหนดที่ต้องการ แล้วเลือก **รายการเดิม** ในฟิลด์ **รูปแบบรายงาน** เลือกรูปแบบรายงานที่จะใช้สำหรับการพิมพ์ คุณสามารถใช้รูปแบบรายงานที่กำหนดเองได้โดยใช้ [กรอบการจัดการเอกสารทางธุรกิจ](/dynamics365/fin-ops-core/dev-itpro/analytics/er-business-document-management)

## <a name="post-invoice-proposals"></a>ลงรายการบัญชีข้อเสนอใบแจ้งหนี้

หลังจากตรวจสอบและแก้ไขใบแจ้งหนี้แล้ว และรายการข้อเสนอใบแจ้งหนี้เป็นที่น่าพอใจ ให้ตรวจสอบยอดรวมใบแจ้งหนี้และภาษีขาย ในกลุ่ม **รายละเอียด** เลือก **ผลรวม** และจากนั้น เลือก **ลงรายการบัญชี** เพื่อลงรายการบัญชีใบแจ้งหนี้

หากต้องการดูใบแจ้งหนี้ก่อนลงรายการบัญชี ให้ล้างกล่องกาเครื่องหมาย **กำลังลงรายการบัญชี** **Pro forma** จะถูกพิมพ์ลงบนใบแจ้งหนี้เพื่อแสดงว่าเป็นใบแจ้งหนี้ตัวอย่าง ในการพิมพ์ใบแจ้งหนี้ ให้เลือกกล่องกาเครื่องหมาย **พิมพ์ใบแจ้งหนี้**

นอกเหนือไปจากหน้า **ข้อเสนอใบแจ้งหนี้** ยังสามารถลงรายการบัญชีข้อเสนอใบแจ้งหนี้ได้โดยเรียกใช้งานตามงวด **ลงรายการบัญชีข้อเสนอใบแจ้งหนี้** หากต้องการค้นหางานนี้ ให้ไปที่ **การจัดการโครงการและการบัญชี** > **เป็นระยะ** > **ใบแจ้งหนี้โครงการ** > **ลงรายการบัญชีข้อเสนอใบแจ้งหนี้**

หน้านี้แสดงข้อเสนอใบแจ้งหนี้ทั้งหมดที่พร้อมสำหรับการลงรายการบัญชี คุณสามารถจัดกำหนดการลงรายการบัญชีข้อเสนอใบแจ้งหนี้ได้โดยเลือก **ชุดงาน** ตั้งค่า **พารามิเตอร์การประมวลผลชุดงาน** เป็น **ใช่** และตั้งค่าการเกิดซ้ำของการประมวลผลชุดงานโดยเลือก **การเกิดขึ้นประจำ**
